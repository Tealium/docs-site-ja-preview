ファイルインポート設定

この記事では、ファイルインポートデータソースを設定する方法について説明します。

始める前に、設定プロセス中にアップロードするためのサンプルCSVファイル（1,000列未満）を作成してください。このサンプルファイルは、列マッピングを自動的に検出するために使用されます。

正しくインポートするためには、CSVファイルが正しくフォーマットされている必要があります。ファイル形式についての情報は、インポート用のCSVファイルを準備するを参照してください。

1. ファイルインポートデータソースを追加する

  1. Sources > Data Sourcesに移動します。
  2. + Add Data Sourceをクリックします。
  3. Categoriesの下でFile Importをクリックし、File Importプラットフォームを選択します。
  4. Nameフィールドに、ファイルタイプに関連する一意の名前を入力します。
  5. Continueをクリックします。

2. サンプルファイルをアップロードする（オプション）

ファイルインポート設定でサンプルCSVファイルを使用している場合、Upload Sample File画面でそれをアップロードします。サンプルファイルの行1の列名は自動的に検出され、Column Mappings画面のエントリを事前に設定するために使用されます。

このステップをスキップしてファイルの列を手動でリストする場合は、Continueをクリックして直接Map Columns画面に移動します。

サンプルファイルをアップロードするための次の手順を完了します：

  1. Upload Fileをクリックし、アップロードしたいサンプルCSVファイルを選択します。
    Sample Fileプレビュー画面では、ファイルの内容がテーブル形式で表示されます。
  2. サンプルファイルをスクロールして、列とデータが正しいことを確認します。検出された列名は、テーブルの下のCSV形式でリストされ、Map Columns画面に事前に入力されます。
  3. サンプルファイルに問題がある場合は、Removeをクリックしてエラーを修正するか、別のファイルを試してみてください。
  4. サンプルファイルの内容を確認し終えたら、Continueをクリックします。

3. 列マッピングを設定する

Map Columnsテーブルを使用して、事前に設定された列ラベルをイベント属性にマッピングするか、ファイルの列ラベルを手動で入力してマッピングします。CSVファイルの各行はイベントとして処理されます。

イベント属性にマッピングされていない列は無視されます。CSVファイルに存在しないマッピングされた列はスキップされます。

ファイルの列ラベルをイベント属性にマッピングするための次の手順を完了します：

  1. Event Specificationドロップダウンリストを使用して、ファイル内のデータに一致する仕様を選択します。カスタムのイベント属性セットを使用するには、Customを選択します。
    この選択は、インポートプロセス中のtealium_event属性の値を決定します。Customを選択した場合、tealium_eventはimportedに設定されます。それ以外の場合は、対応する仕様名（例：purchase）に設定されます。

    選択した仕様を変更すると、列マッピングテーブルがリセットされます。

  2. サンプルファイルをアップロードしていない場合は、ファイルの列ラベルをMap Columnsテーブルに入力します。各CSV列ラベルは一意でなければなりません。

  3. ドロップダウンリストから対応するイベント属性を選択します。
    列に日付/時間の値が含まれている場合は、対応する属性を選択し、Set Formatをクリックします。Date/Time Formatスライドアウトで、適切な日付/時間の形式を選択するか、カスタム形式を入力し、Doneをクリックします。

  4. すべての対応するイベント属性のマッピングが完了したら、Continueをクリックします。

4. 訪問者ID属性をマッピングする

ファイルデータをAudienceStreamで使用するには、データを訪問者ID属性にマッピングします。訪問者IDを表すマッピングされたイベント属性を選択し、それを対応する訪問者ID属性にマッピングします。

AudienceStreamのVisitor ID Mappingはデフォルトで有効になっています。Visitor ID Mappingを無効にすると、訪問者のステッチングにエラーが発生する可能性があります。AudienceStreamのVisitor ID Mappingについての詳細は、File Import Data Sourceを使用した訪問者識別を参照してください。

5. ファイル転送サービスを設定する

自分のファイル転送サービスを使用する場合は、進行する前に接続詳細を準備しておいてください。

既存のファイルサービス設定を選択するか、新しいものを作成します。

  • すでにファイルサービス設定を作成している場合は、既存のファイルサービス設定を完了するセクションを参照してください。
  • ファイルサービス情報を変更するには、設定名の隣にある編集アイコンをクリックします。
  • 新しい設定を追加するには、新しいファイルサービス設定を追加するセクションを参照してください。

既存のファイルサービス設定を完了する

ファイルサービス設定を完了または編集するには、次の手順を完了します：

  1. Choose File Service Configurationドロップダウンリストから既存の設定を選択します。

  2. File Name Prefixを入力します。例えばcustomersなど。ファイル名のプレフィックスは、ファイル内のデータをインポートする際にどのファイルインポートデータソースを使用するかを識別するために使用されます。この例では、プレフィックスはcustomers_VERSION.csvというタイトルのCSVファイルを作成するために使用されます。

    ファイル名のプレフィックスは、複数のファイルインポートデータソース間で一意でなければなりません。例えば、customersとcustomers-ordersというプレフィックスを使用すると、同じ初期プレフィックスcustomersを共有しているため、予期しない結果が発生する可能性があります。

  3. （オプション）ファイル処理の効率を向上させるために、GZIP形式を使用してファイルを圧縮します。ファイルが圧縮されている場合は、Compression形式としてGZIPを選択します。

  4. （オプション）ファイルの安全な処理を確保するために、PGP暗号化を使用してファイルを暗号化します。ファイルが暗号化されている場合は、Encryption形式としてPGPを選択します。

    圧縮と暗号化の両方を行うファイルを処理する場合、ファイルはまず圧縮され、次に暗号化される必要があります。以前にマッピングされたファイルの新しいバージョンは、元のファイルと同じ暗号化と圧縮の方法を使用する必要があります。
    File Importの圧縮についての詳細は、File Import Data Sourcesの暗号化と圧縮を参照してください。

  5. Enable File Service Processingセクションで、File Importのサービス処理を有効または無効にするためのトグルを使用します。

    • ONに設定すると、ファイルサービスがチェックされ、ファイルが10分ごとに処理されます。
    • ファイルの処理を開始する準備ができていない場合は、OFFに設定します。後でサービスを有効にすることができます。

  6. ContinueをクリックしてSummaryタブに移動します。

新しいファイルサービス設定を追加する

新しいファイル転送サービス設定を追加するには、次の手順を完了します：

  1. **+**アイコンをクリックして新しいファイルサービス設定を追加します。
  2. Service Configuration画面で、設定の名前を入力し、使用するファイルサービスのタイプを選択します。
  3. 選択したサービスにアクセスするために必要な認証情報を入力します。
    • Tealium S3 Bucket - 必要な認証情報は自動的に生成されます。
    • My SFTP Connection - Authentication Methodを選択します。Password、Upload Private Key File、Generate Key Pairがサポートされている認証方法です。
      • Passwordを選択した場合は、パスワードを入力し、Test File Source Connectionをクリックします。
      • Upload Private Key Fileを選択した場合は、リストからファイルを選択し、Test File Source Connectionをクリックします。
      • Generate Key Pairを選択した場合は、Generateをクリックし、Test File Source Connectionをクリックします。
  4. （オプション）Test Connectionをクリックしてファイルサービスの接続をテストします。
  5. Saveをクリックします。

6. サマリー

この最終ステップでは、サマリーを表示し、必要な修正を行い、プロフィールを保存して公開します。

  1. Event AttributeとVisitor ID mappingsを表示します。
  2. 設定を編集するには、Previousをクリックして変更を加えたいステップに戻ります。
  3. Finishをクリックして設定画面を終了します。新しいデータソースは現在、Data Sources Dashboardにリストされています。
  4. Save/Publishをクリックして変更を保存し、公開します。

インポート活動のステータスを確認するには、データソースを展開し、StatusタブをクリックしてData Sources Dashboardに移動します。詳細については、インポートステータスの表示を参照してください。

ファイルインポートデータソース設定の編集

ファイルインポートデータソース設定を変更するには、Data Sources Dashboardに移動し、変更したい設定の隣にある編集アイコンをクリックします。Data Sources Settings画面から、列マッピング、訪問者IDマッピング、およびファイルサービス設定を編集することができます。
