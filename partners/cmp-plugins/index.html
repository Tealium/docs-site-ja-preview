CMP プラグイン

Tealium iQ Tag Management で同意管理プラットフォーム (CMP) を統合して、管理対象のタグのプライバシー設定を実行する方法について説明します。

CMP プラグインは、拡張マーケットプレースにおけるベンダー別のソリューションで、Tealium iQ のインストールと CMP のインスト―ルを統合を容易にするものです。

CMP プラグインには、以下のメリットがあります:

  • 2 つのソリューションを統合する時間を減らすための簡単なインポート ツールであること。
  • CMP からの同意ステータスに基づくタグのトリガを強固に実行すること。
  • CMP から同意データをサーバー側に配信すること。

仕組み

CMP プラグインは、CMP で構成されたデータ処理サービスと Tealium iQ で構成されたタグとの間でマッピングを作成するものです。プラグインは、Tealium iQ に CMP 設定をインポートし、一または複数のタグに各サービス名をマッピングすることを容易にするヘルパー ツール、通常はブックマークレットを提供します。

マッピングの設定が完了しサイトにデプロイされると、管理対象のタグは、CMP とユーザーのインタラクションに基づきトリガされます。

CMP 統合は、以下の部分に分かれます:

  • サービス マッピング サービス マッピングは、CMP サービス名を、管理対象のタグに関連付けます。リンク済みのタグは、関連する CMP サービス名が同意された後でないとトリガされません。
  • データレイヤー統合 CMP からの同意ステータス変数は、各トラッキング コールの データレイヤーオブジェクト に追加されます。変数名は、 <cmp name>_<variable name>形式で表示されます。たとえば、Usercentrics からの同意済みサービス名の配列は、 usercentrics_services_with_consentとなります。
    • <cmp name>_services_with_consent - 同意済みサービス名の配列。
    • <cmp name>_consent_type - explicit または implicit 同意種類。

要件

Tealium iQ Tag Management と統合するには、CMP で構成される各データ処理サービスに関して、以下の同意ステータス情報が利用可能になっていなければなりません。これらは、通常は、ページ内でグローバル JavaScript 変数または方法として、以下のとおりアクセスされます。

  • サービス名 - 同意が適用されたサービスに対応する一意の名称。
  • 同意ステータス - サービスに対応する同意のステータスであり、 true は、同意が付与済みであることを示し、 false は、同意が未付与であることを示します。
  • 同意メソッド種類 - 同意が取得されたメソッド (implicit または explicit)。

Tealium への同意データの送信

クライアント側での同意の収集と実行に加えて、CMP 統合は、同意ステータスを、サーバー側コネクタを使用する他のベンダーにも送信することができます。

お客様の CMP Webhook コールバックをサポートしている場合には、アップデートを HTTP API エンドポイントに送信することによって、Tealium との統合を拡張することができます。こうしたイベントは、サーバー側で処理されますが、Tealium EventStream、AudienceStream、および DataAccess を用いる他のベンダーにおいてアクティベートすることができます。

CMP が Webhooks をサポートしていない場合には、Tealium に連絡して、担当のプロダクト チーム、エンジニアリング チームと連携して、統合をフォーマライズしてください。
