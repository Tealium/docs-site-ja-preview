データレイヤー

各モジュールによって提供される、共通データレイヤー値とデータの管理について説明します。

カスタム データ

各イベントにおいて一般データレイヤー値が必要になることは、一般的なことです。各トラッキング コールにこうした値を追加する必要がなくなるようにするには、すべてのトラッキング コールに適用する永続性データレイヤー値に tealium.dataLayer メソッドを用います。値が保存され次第、これらの値は、トラッキング対象の各イベントに自動的に含められます。

メソッド (tealium.dataLayer により提供されるもの) は、カスタム値の読み込みと書き出しが可能ですが、これには、 モジュール データへのアクセス権がありません。

値の設定と取得

データレイヤーの値の設定、取得、削除のためには、いくつかのユーティリティ メソッドがあります。カスタム データレイヤーの値を設定し取得するメソッドは、型付きですが、データレイヤーに保存された変数の型が不明の場合には、型付きでないメソッドが使用可能です。

たとえば、文字列値を設定するには、 putString()を呼び出します:

tealium.dataLayer.putString("my_string", "my_string_value")

型の指定がない値を取得するには、 get()を呼び出します:

tealium.dataLayer.get("my_string")

データ型

データレイヤーは、以下のデータ型をサポートします:

  • Boolean
  • String
  • Integer
  • Long
  • Double
  • JSONObject
  • JSONArray
  • Array<Boolean>
  • Array<String>
  • Array<Integer>
  • Array<Long>
  • Array<Double>

長い数値をを設定するには、 putLong()を呼び出します。

val launchTime = System.currentTimeMillis()
tealium.dataLayer.putLong("launch_time", launchTime)

文字列の配列を設定するには、 putStringArray()を呼び出します。

val lastThreeProductsViewed = arrayOf("SKU-1", "SKU-2", "SKU-3")
tealium.dataLayer.putStringArray("last_three_products", lastThreeProductsViewed)

データの有効期限

データレイヤー値には、有効期限があり、有効期限が過ぎるとデータレイヤーから削除されます。カスタム データレイヤー値を設定すると、任意で、値の有効期限を設定できます。デフォルト有効期限は、現在のセッションです。

使用可能な有効期限のオプションは、次のとおりです。

Type 値 説明
Session (default) Expiry.SESSION 現在のセッションの終了時に有効期限が切れる
Forever Expiry.FOREVER アプリがインストールされている間は有効期限が切れない
Custom Duration Expiry.afterTimeUnit(time, units) 次の時間単位を使って指定した期間が経過した後に有効期限が切れる: NANOSECONDS, MICROSECONDS, MILLISECONDS, SECONDS, MINUTES, HOURS, DAYS

たとえば、特定の値について決して有効期限が切れないように設定するには:

val userId = "my_id"
tealium.dataLayer.putString("user_id", userId, Expiry.FOREVER)

現在のセッションの終了時に有効期限が切れるように明示的に値を設定するには (デフォルトの作動):

val sessionProductViews = 10
tealium.dataLayer.putInt("product_views", sessionProductViews, Expiry.SESSION)

  1. 日経過後に有効期限が切れるように値を設定するには:

    val frequentVisitor = tealium.dataLayer.getBoolean("frequent_visitor") ?: false

tealium.dataLayer.putBoolean("frequent_visitor",
    frequentVisitor,
    Expiry.afterTimeUnit(1, TimeUnit.DAYS))

例

使用可能なメソッドの完全なリストについては、 クラス: DataLayer (データレイヤー)を参照してください。

データレイヤーに変動が含まれるかどうかをチェックするには、 contains()を呼び出します。

tealium.dataLayer.contains("my_string")

特定の値を削除するには、 remove()を呼び出します。

tealium.dataLayer.remove("my_string")

データレイヤーにおけるすべてのキーの名前をリストにするには keys():

tealium.dataLayer.keys()

データレイヤーにおける変数の数を取得するには、 count()を呼び出します。

tealium.dataLayer.count()

変数の有効期限を取得するには、 getExpiry()を呼び出します。

tealium.dataLayer.getExpiry("my_string")

すべてのデータレイヤー変数を取得するには、 all()を呼び出します。これによって mapOf()が返されます。

tealium.dataLayer.all()

すべてのデータレイヤー変数を削除するには、 clear()を呼び出します。

tealium.dataLayer.clear()

lifecycle_totalsleepcount アプリがインストール後にバックグラウンドに移行した総回数（アプリが削除された場合のみリセット） 400
lifecycle_totalwakecount インストール後の起動とスリープ解除両方の総回数（アプリが削除された場合のみリセット） 563
lifecycle_type ライフサイクルコールの種類。 launch, wake, sleep
lifecycle_updatelaunchdate バージョンアップデート検出後、最初のスリープ解除/起動のGMTタイムスタンプ 2014-09-08T18:10:01Z
lifecycle_wakecount 現バージョンのアプリの起動とスリープ解除両方の総数（アップデートされた場合にリセット） 29
