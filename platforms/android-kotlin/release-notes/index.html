リリースノート

リリースノート

以下は、Android Kotlin ライブラリのリリースごとの変更点をまとめたリストです。完全なリストについては、 変更ログ を参照してください。

Tag Management 1.1.2 (2022 年 2 月)

  • Tag Management 1.1.2
    • セルフホスト JavaScript ファイルを用いている Tealium iQ アカウント用のセッション カウントを無効にできるように sessionCountingEnabled フラッグを TealiumConfig に追加。

1.3.1 (2022 年 2 月)

  • Core 1.3.1
    • WebView 作成の失敗に対応する安定性に関する修正
    • 接続の例外の取り扱いに対応する安定性に関する修正
    • InApp Purchase 自動トラッキング 1.0.0 リリース。

1.3.0 (2021 年 10 月)

  • Core 1.3.0
    • Java ライブラリから再読み込みされる変数の脱漏:
      • デフォルト: tealium_random および was_queued
      • DeviceCollector - device_battery_percent および device_ischarging
      • TimeCollector -timestamp_epoch
    • DataLayer パフォーマンス改善と、 JSONArray オブジェクトの保存に関する追加のサポート
    • 日付について ISO 8601 を基準としたフォーマットで取得するようにしました
    • Kotlin、Coroutines、Android Target Version、AGP 7、Java 11、MockK、Robolectric について依存関係のアップデート
    • バグ修正:
      • Opaque URI を DeepLinkHandler により適切に取り扱うようにしました
    • 未使用コードを非推奨にしました
  • AutoTracking 1.0.0
    • スクリーン ビュー イベントを自動的にトラッキングするための新たなアクティビティ トラッキング モジュール
  • AdIdentifier 1.1.0
    • 新たな AppSet 識別子をサポート
    • Core に合わせて依存関係を更新。
  • CollectDispatcher 1.1.0、TagManagementDispatcher 1.1.0
    • シリアライゼーション (直列化) の使用を標準化JsonUtils
    • Core に合わせて依存関係を更新。
  • CrashReporter 1.1.0、HostedDataLayer 1.1.0、InstallReferrer 1.1.0、Lifecycle 1.1.0、Location 1.1.0、Media 1.1.0、RemoteCommandDispatcher 1.1.0
    • Core に合わせて依存関係を更新。
  • VisitorService 1.1.0
    • 訪問者の更新時に用いる Tealium Profile 名をオーバーライドするための構成オプションの追加:config.overrideVisitorServiceProfile
    • シリアライゼーション/デシリアライゼーションの性能改善
    • Core に合わせて依存関係を更新。
    • バグ修正:
      • 日付が 0 となったときの日付解析の例外

1.2.8 (2021 年 10 月)

  • Core 1.2.8
    • 環境から生じるログ レベルのオーバーライドを可能にするために TealiumConfig.logLevel を追加。
    • Consent Manager によりトリガーされる各イベントのペイロードに consent_last_updated タイムスタンプを追加。
    • ペイロード キー定数をアクセス簡便化のために Dispatch.Keys.XXX に移行。
    • バグ修正
      • app_name をアプリ レベルの非ローカライズ バージョンに戻しました。
      • スレッディングにより生じたイベント ルーター クラッシュを修正。
      • screen_title をデフォルトで TealiumView ペイロードに再追加し、コンテクスト データに提供されないときには viewName に設定
      • Asset として tealium-settings.json のロードを試行するときにおける必要なログを削除。
      • セッション中に複数のディープ リンクが生じる場合について、以前のディープ リンク パラメータを新規のものを追加する前にデータレイヤーから削除しました。
  • CollectDispatcher 1.0.6、HostedDataLayer 1.0.4、RemoteCommandDispatcher 1.0.5、TagManagement 1.0.7
    • 新たな Dispatch.Keys.XXX (Core 1.2.8 以降にのみに設定) を使用するためのイベント キー リファレンスの更新。

1.2.7 (2021 年 9 月)

  • Core 1.2.7
    • DeviceData モジュールからのデータレイヤーに device_logical_resolution を追加。

1.2.6 (2021 年 7 月)

  • Core 1.2.6
    • 独自の一意識別子の使用をサポートするために TealiumConfig.existingVisitorId を追加。
    • Consent Logging のためのプロファイルのオーバーライドをサポートするために TealiumConfig.consentManagerLoggingProfile を追加。
    • Consent Logging イベントを、有効なすべての Dispatchers を通じてルーティングしました。
    • データレイヤー変数 app_build および app_versionの値を更新。
  • Tag Management Dispatcher 1.0.6
    • Consent Logging プロファイルのオーバーライドをサポート。
  • Collect Dispatcher 1.0.5
    • Consent Logging プロファイルのオーバーライドをサポート。
  • Visitor Service 1.0.5
    • バグ修正: 保存された long 値をデシリアライゼーションしました。

1.2.5 (2021 年 5 月)

  • HttpClient の問題と、同意記録の失敗を修正。
  • IO dispatcher に HttpClient の実行を移すことにより非同期のパフォーマンスを改善。

1.2.4 (2021 年 4 月)

  • Core 1.2.4

    • 新たなデータレイヤー変数 enabled_modules および enabled_modules_versionsを追加。いずれも、使用中のモジュールの保存リストと、そのバージョン番号 (バージョンの取得に必要なすべてのモジュールの最新バージョン) を含む文字列の配列です。

    モジュール名についても、Swift に一致させるために「UPPER_SNAKE_CASE」から「PascalCase」に変更済みです。

    • 新たな構成オプション remoteApiEnabled を加えて、RemoteCommands モジュールと TagManagement モジュールの使用時における remote_api イベントの無効化を許可。
    • 新たな構成オプション overrideCollectProfile を加えて、 tealium_profile の値のオーバーライドを、イベント ペイロードと、 TealiumConfig オブジェクトで用いられるものと異なる Tealium profile にイベントを送信する場合に許可。
    • バグ修正: application/json と x-www-form-urlencoded のコンテンツ種類の両方について HttpRemoteCommandを用いるときのサポートを追加。
    • バグ修正: 特定のコレクションが正しくエンコードされない問題を修正。

1.2.3 (2021 年 3 月)

  • Core 1.2.3
    • Consent Management: カスタム同意ポリシーの設定に関するオプションを追加。
    • 新たな device_language、 os_name を DeviceCollector モジュールに追加。
  • Crash Reporter 1.0.2
    • CrashReporter モジュールのキーを更新: crash_exception_name を crash_nameに変更し、 crash_exception_cause を crash_causeに変更。

1.2.2 (2021 年 3 月)

  • Core 1.2.2
    • イベントおよびビュー ペイロードに request_uuid を追加
  • TagManagement Dispatcher 1.0.4、RemoteCommand Dispatcher 1.0.3
    • バグ修正: HttpRemoteCommand を実行のためにバックグランド スレッドにルーティングしました

1.2.1 (2021 年 3 月)

  • Proguard および Consumer Rules の改訂。パブリック クラスの不足分を追加し、消費者ルールを改訂
  • Core 1.2.1、Collect Dispatcher 1.0.3、TagManagement Dispatcher 1.0.3、Install Referrer 1.0.2、Lifecycle 1.0.3、Location 1.0.3、VisitorService 1.0.3、RemoteCommand Dispatcher 1.0.2、Hosted DataLayer 1.0.2、Crash Reporter 1.0.1、Ad Identifier 1.0.1

1.2.0 (2021 年 1 月)

  • 同意管理
    • 同意の有効期限: 提供された同意の有効期限を設定できるようにしました
    • 非推奨の consentManagerEnabled 構成プロパティ。有効な ConsentPolicy を設定することで、Consent Manager を自動的に有効化
    • バグ修正: 同意コレクターのロジックの誤り
  • 訪問者 ID: 新たなメソッドによりデバイス用の訪問者 ID を手動で再生成
    • Proguard: 当初の Proguard ルールに、生成したバイナリおよび消費者 Proguard ルールを追加
  • AdIdentifier 1.0.0
    • AdIdentifier モジュールの最初のリリース: 使用可能な場合は、各 Dispatch に AdIdentifier を追加
  • CrashReporter 1.0.0
    • CrashReporter モジュールの最初のリリース: クラッシュを生じさせる未捕捉の例外を記録し、アプリ再起動時にクラッシュ データを次のディスパッチに送信
  • Collect Dispatcher 1.0.2、Hosted DataLayer 1.0.1、InstallReferrer 1.0.1、Lifecycle 1.0.2、Location 1.0.2、RemoteCommand Dispatcher 1.0.1、TagManagement Dispatcher 1.0.1、Visitor Service 1.0.2
    • Proguard - 当初の Proguard ルールに、生成したバイナリおよび消費者 Proguard ルールを追加

1.1.0 (2020 年 11 月)

  • ユーザーの特定イベントの間隔を報告するための時間付きイベントのサポート
  • 新たな device_model および device_manufacturer データ キーで、 DeviceCollectorが提供するもの
  • Github Actions を用いてマスターへの PR へのユニットの自動化とテストの実装
  • Collect Dispatcher 1.0.1
    • Collect URL のオーバーライドの追加
    • テスト カバレッジの拡大
  • Hosted Datalayer 1.0.0
    • テスト カバレッジの拡大
  • Lifecycle 1.0.1
    • バグ修正: スリープ イベントに関する自動トラッキング キーの不足
    • テスト カバレッジの拡大
    • Location 1.0.1
    • バグ修正: 周辺ジオフェンスの自動追加に関する問題
    • テスト カバレッジの拡大
  • Visitor Service 1.0.1
    • テスト容易性目的での非ブレイキング変更
    • テスト カバレッジの拡大

0.1.2 (2020 年 10 月)

  • 複数のインスタンスの Tealium のサポート
    • Tealium インスタンスのコンストラクションに関するブレイキング変更の導入
  • 修正: Android の旧バージョンに関する接続問題

0.1.1 (2020 年 9 月)

  • Maven 依存関係の修正
  • Dispatch ペイロードのログの追加

0.1.0 (2020 年 7 月)

  • 最初のリリース
