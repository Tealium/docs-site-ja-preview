ANDROID KOTLIN

TealiumConfig

Tealium for Android (Kotlin) で提供されているTealiumConfig クラスおよびメソッドに関するリファレンス ガイド。

クラス：TealiumConfig

TealiumConfig クラスは、メインの Tealium クラス用の設定オプションを設定するメソッドを提供します。以下の文書は、一般的に使用される Kotlin の TealiumConfig クラスのメソッドとプロパティをまとめたものです。個々のモジュールでは、TealiumConfig クラスにモジュール独自の拡張機能も提供します (有効な場合)。これは、関連のモジュールの独自の文書で文書化されます。

メソッド/Property 説明
adobeVisitorAuthState 訪問者に関する現在の認証状況を設定します。
adobeVisitorCustomVisitorId 訪問者 ID が初期化時に既知であれば、既知の訪問者 ID を Adobe ECID に設定します。ECID が将来変更になった場合は、 linkECIDToKnownIdentifier を呼び出して、訪問者 ID を更新します。
adobeVisitorDataProviderId ECID を既知の ID とリンクさせるときに必要となる、Adobe が提供するデータ プロバイダー ID を設定します。
adobeVisitorExistingEcid 新規の ECID をリクエストせず、以前のインストールからの既存の ECID (Adobe SDK など) を設定して用います。
adobeVisitorOrgId @ 部分を含めて、Adobe 組織 IDを含む文字列。
adobeVisitorRetries ECID を用いずにトラッキング コールが送信されることになる、リスクエストの失敗までに認められる無効な訪問者 ID による再試行の回数。
autoTrackingBlocklistFilename ローカル ブロック リスト ファイルの名前を設定します。
autoTrackingBlocklistUrl リモート ブロック リスト ファイルの URL を設定します。
autoTrackingCollectorDelegate 自動トラキング アクティビティのために実行されるグローバル デリゲートを設定します。
autoTrackingMode AutoTracking モジュール用のトラッキング モードを設定します。
consentExpiry ユーザーの同意選択の有効期限を設定します。
consentManagerEnabled 同意管理機能を有効化/無効化するためのオプションの Boolean 。
consentManagerPolicy 遵守する同意管理機能がどれかを決定するための、オプションの ConsentPolicy 。
consentManagerLoggingEnabled 同意変更の記録を有効化/無効化にするための、オプションの Boolean 。
consentManagerLoggingUrl 同意記録の更新についてターゲットに関する、オプションの String 。
consentManagerLoggingProfile Tealium Collect からのデータの送信先のプロファイルをオーバーライドするための、オプションの String 。
deepLinkTrackingEnabled ディープ リンクの自動トラッキングを有効化または無効化します。
existingVisitorId tealium_visitor_idに保存されている中から、独自の一意識別子を設定します。
hostedDataLayerMaxCacheTimeMinutes ホスト型データレイヤーデータの有効期限を設定します。
hostedDataLayerEventMappings データレイヤー ID の検索時にホスト型データレイヤーモジュールが使用するイベント マッピングを設定します。
logLevel 記録する情報の量を制御するログ レベル プロパティを設定します。
remoteAPIEnabled remote_api イベントを有効化または無効化します。DispatchQueue モジュールが使用されている場合に RemoteCommands モジュール用に必要です。
sessionCountingEnabled Tealium iQ accountsアカウント用のセッション カウントを無効化します。Tealium JavaScript ファイルをセルフホストしている場合にこれを用います
TealiumConfig TealiumConfigのインスタンスを作成するコンストラクタ メソッド。

adobeVisitorAuthState

訪問者に関する現在の認証状況を設定します。

例:

config.adobeVisitorAuthState = AdobeAuthState.AUTH_STATE_AUTHENTICATED
config.adobeVisitorAuthState = AdobeAuthState.AUTH_STATE_LOGGED_OUT
config.adobeVisitorAuthState = AdobeAuthState.AUTH_STATE_UNKNOWN

adobeVisitorCustomVisitorId

訪問者 ID が初期化時に既知であれば、既知の訪問者 ID を Adobe ECID に設定します。ECID が将来変更になった場合は、 linkECIDToKnownIdentifier を呼び出して、訪問者 ID を更新します。

config.adobeVisitorCustomVisitorId = "20381482060927465156359999806251989655"

adobeVisitorDataProviderId

ECID を既知の ID とリンクさせるときに必要となる、Adobe が提供するデータ プロバイダー ID を設定します。

config.adobeVisitorDataProviderId = "01"

adobeVisitorExistingEcid

新規の ECID をリクエストせず、以前のインストールからの既存の ECID (Adobe SDK など) を設定して用います。

例:

config.adobeVisitorExistingEcid = "20381482060927465156359999806251989655"

adobeVisitorOrgId

@ 部分を含めて、Adobe 組織 IDを含む文字列。組織 ID が提供されない場合は、モジュールは初期化されず、訪問者 ID が取得されません。トラキング コールは、Adobe ECID がなくとも通常とおり継続します。

config.adobeVisitorOrgId = STRING_VALUE

例:

config.adobeVisitorOrgId = "1A2A111A111150AA0A110A12@AdobeOrg"

adobeVisitorRetries

ECID を用いずにトラッキング コールが送信されることになる、リスクエストの失敗までに認められる無効な訪問者 ID による再試行の回数。

例:

config.adobeVisitorRetries = 5

autoTrackingBlocklistFilename

AutoTracking モジュールとともに用いるため。

自動トラッキングの対象から除外する活動のリストを含む JSON 形式のブロック リスト ファイルの名前を設定します。ファイルは、Android assets ディレクトリから入手可能とする必要があります。

config.autoTrackingBlocklistFilename = "autotracking-blocklist.json"

autoTrackingBlocklistUrl

AutoTracking モジュールとともに用いるため。

自動トラッキングの対象から除外する活動のリストを含む JSON 形式のブロック リスト ファイルの URL を設定します。

config.autoTrackingBlocklistUrl = "https://example.com/autotracking-blocklist.json"

autoTrackingCollectorDelegate

AutoTracking モジュールとともに用いるため。

AutoTracking モジュールによりトラッキングされる活動のために実行されるグローバル デリゲートを設定します。特別な活動に含まれるデータをカスタマイズするには、このデリゲートを編集します。

config.autoTrackingCollectorDelegate = myCollectorDelegate

autoTrackingMode

AutoTracking モジュール用のトラッキングのレベルを設定します。

  • FULL
    @Autotracked(track=false)によって付加されるものを除き、すべてのアクティビティをトラッキングします。
  • ANNOTATED
    @Autotrackedによって付加されるアクティビティのみをトラッキングします。

  • NONE
    (アプリのデバッグまたはデベロップメント ビルトのために) すべての自動トラッキングを無効化します。

    config.autoTrackingMode = AutoTrackingMode.FULL

Collectors

コレクターは、デバイスから補足情報を収集し、データレイヤーにそれを添付して、Tealium Customer Data Hub にそれを転送するモジュールです。コレクターの中にはコア ライブラリに含まれるものもあれば、オプションで、別のモジュールとしてインストールされるものもあります。

以下の表では、利用可能なコレクターを列挙しています。デフォルト コレクターは、コレクター名に「*」印を付して注記しています。

Collector Name TealiumConfig のリファレンス
AdobeVisitor Collectors.AdobeVisitor
AppData* Collectors.AppData
Connectivity* Collectors.Connectivity
Device Collectors.Device
Lifecycle Collectors.Lifecycle

このモジュールは、 TealiumConfig collectors プロパティを用いて有効化または無効化されます。

お使いのコレクターの既存リストに以下の例を追加します。

val config = TealiumConfig(collectors = mutableSetOf(Collectors.AdobeVisitor))

consentExpiry

ユーザーの同意選択の有効期限を設定します。Tealium の初期化の前にこのプロパティを設定します。ユーザーの同意選択のデフォルト有効期限は、次のとおりです。

  • CCPA: 395 日
  • GDPR: 365 日

以下では、同意有効期限を 90 日に設定しています。

config.consentExpiry = ConsentExpiry(90, TimeUnit.DAYS)

consentManagerEnabled

ConsentManagerモジュールを有効にするかどうかを設定します。consentManagerPolicyを通じてポリシーを設定することも必要です。

config.consentManagerEnabled = true  // enable

consentManagerPolicy

アプリ内で遵守するべき ConsentPolicy を設定します。このポリシーは、ディスパッチャーが関連の法的要件に従って SDK を通じてキューに登録されてドロップされているかどうかを制御します。

config.consentManagerPolicy = ConsentPolicy.GDPR

consentManagerLoggingEnabled

同意設定の変更がリモート サーバーに送付して記録される必要があるかどうかを設定します。

config.consentManagerLoggingEnabled = true

consentManagerLoggingProfile

Tealium Collect からのデータの送信先のプロファイルをオーバーライドします。

config.consentManagerLoggingProfile = "us-mobile"

consentManagerLoggingUrl

同意設定の変更を記録するときに用いる URL を設定します。

config.consentManagerLoggingUrl = url

deepLinkTrackingEnabled

標準ディープ リンク (Facebook などのソースからアプリへのリンクや、QR トレースなど) の自動トラッキングを有効化または無効化します。デフォルトでは有効になっています。

config.deepLinkTrackingEnabled = true

existingVisitorId

tealium_visitor_idに関するデータレイヤー値を設定します。貴社が貴社の独自の一意識別子を持っているときには、これを用います。

config.existingVisitorId = "8243d23b56d5488571027547bc72d93"

hostedDataLayerMaxCacheTimeMinutes

ホスト型データレイヤーデータの有効期限を設定します。このプロパティが設定されていない場合は、ホスト型データレイヤーアイテムの有効期限はデフォルトで 7 日間になっており、この期間が満了次第、CDN から再ダウンロードされます。

hostedDataLayerMaxCacheTimeMinutes = 30

ホスト型データレイヤーの更新頻度が高い場合は、ネットワーク リクエストが増えるために、設定する値が低いと、バッテリーの寿命に影響することがあります。

hostedDataLayerEventMappings

ホスト型データレイヤーモジュールがデータレイヤー ID を取得するときに使用するホスト型データレイヤーキー (ルックアップ変数) を設定します。このキーは、ディスパッチの tealium_event キーの値です。

hostedDataLayerEventMappings = mapOf("pdp" to "product_id")

logLevel

記録されている情報の量を制御するためののログ レベル プロパティを設定します。このプロパティを設定することで、 environment プロパティ (TealiumConfig内) に由来するデフォルト値がオーバーライドされます。

ログ レベルを、次の値のうちのいずれかで設定します。

ログレベル ログ アクティビティの説明
DEV アプリケーションをデバッグするために用いられるデバッグ レベルのイベント (開発環境用で推奨)。
QA アプリケーションの進捗を表示する通知イベント。
PROD 重要なエラーや失敗などのエラー イベント。
SILENT ログがありません。 
config.logLevel = LogLevel.DEV

LogCat を用いて Android Studio の開発ログを表示します。Android デバイスに接続して、LogCat アプリを用いてプロダクション ログを表示します。

remoteAPIEnabled

remote_api イベントを有効化または無効化します。DispatchQueueモジュールが使用されている場合にRemoteCommandsモジュールに必要です。デフォルトはfalseです。

config.remoteAPIEnabled = true

sessionCountingEnabled

Tealium iQ accounts アカウント用のセッション カウントを無効化します。Tealium JavaScript ファイルをセルフホストしている場合にこれを用います。デフォルトは trueとなっています。

config.sessionCountingEnabled = false

TealiumConfig

TealiumConfig のインスタンスを作成するコンストラクタ メソッド。

val config = TealiumConfig(
      application,
      account,
      profile,
      environment);

TealiumConfig インスタンスは、次のパラメータを使用して設定されます。

パラメータ 型 説明 例
application Application アプリケーションインスタンス applicationObj
account String Tealiumアカウントの名前 "companyXYZ"
profile String Tealiumプロファイルの名前 "main"
environment String Tealium環境の名前 Environment.PROD

以下は例です。

// Assuming execution is within Application.onCreate()
val config = TealiumConfig(this, "ACCOUNT_NAME", "PROFILE_NAME", Environment.PROD)

// Add required Dispatchers
config.dispatchers.addAll(setOf(CollectDispatcher, TagManagementDispatcher))

// Add required Modules
config.modules.add(VisitorService)

// Automatic tracking mode
config.autoTrackingMode = AutoTrackingMode.FULL // (FULL, ANNOTATED, NONE)

timedEventTriggers

時間付きイベントを自動でトラッキングするための EventTrigger オブジェクトのリスト。

val config = TealiumConfig(…).apply {
  timedEventTriggers = mutableListOf(
     EventTrigger.forEventName(
       start_event,
       stop_event,
       timed_event_name)
  )  
}

EventTrigger のパラメータは、以下のとおりです。

パラメータ 型 説明
start_event String 時間付きイベントを開始するための TealiumEvent の名前
stop_event String 時間付きイベントを停止するための TealiumEvent の名前
timed_event_name String? (オプション) 時間付きイベントの名前 (デフォルト: "start_event::stop_event")
