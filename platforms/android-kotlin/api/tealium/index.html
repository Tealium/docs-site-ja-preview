ANDROID KOTLIN

Tealium

Tealium for Android (Kotlin) で提供されている Tealium クラスおよびメソッドに関するリファレンス ガイド。

クラス：Tealium

Tealium クラスは、スクリーン ビューやイベントをトラッキングするメソッドを提供します。以下の文書は、一般的に使用される Kotlin の Tealium クラスのメソッドとプロパティをまとめたものです。個々のモジュールでは、 Tealium クラスにモジュール独自の拡張機能も提供可能です（有効な場合）。

メソッド/Property 説明
cancelTimedEvent() 時間付きイベント用のタイマーをキャンセルします。
consentManager ユーザー同意設定を設定するために Consent Manager にアクセスします。
dataLayer デバイスの永続性ストレージにアクセスします。データレイヤー内のアイテムは各イベントに追加されます。
endTraceVisitorSession() セッション イベントの終了を確認するために訪問者セッションをリモートで終了します。
events Tealium SDK 全体で数多くのイベントにリスナー クラスを提供できるようになります。
joinTrace() 現在のセッション用のデータレイヤーに、供給されたトレース ID を追加します。
leaveTrace() トレース ID を削除してトレースを終了します。
linkEcidToKnownIdentifier 現在の ECID を既知のセカンダリ ID にリンクします。
modules システム内のモジュールへのアクセス権を提供します。
resetVisitor() 次のトラッキング コールで新規のものを取得して、訪問者 ECID をリセットします。
sendQueuedDispatches() DispatchValidators を再評価するリクエストをします。
session 現在のセッションに関する情報を提供します。
startTimedEvent() 所与の名前を用いて時間付きイベントを開始します。
stopTimedEvent() timed_event トラッキング コールをトリガーする時間付きイベント用のタイマーを停止します。
Tealium Tealium のインスタンスを作成するコンストラクタ メソッド。
track() スクリーン ビューやイベントをトラッキングするメソッド。
visitor AdobeVisitor インスタンスの全部を返します。
visitorId データレイヤーに保存されている現在の訪問者 ID。

cancelTimedEvent()

以前に開始した時間付きイベントをキャンセルします。時間付きイベントがトラッキングされていません。

tealium.cancelTimedEvent(name: "TIMED_EVENT_NAME")
パラメータ 型 説明
name String 時間付きイベントの名前

consentManager

現在の同意ステータスとカテゴリを設定して、現在のポリシーの実施状況を確認できるようになります。同意管理 ガイドで、詳細を参照してください。

tealium.consentManager.userConsentStatus = ConsentStatus.CONSENTED

dataLayer

キーと値のペアに関するデバイスの永続性ストレージを提供し、同時に、有効期限の設定を可能にします。各エントリは、 track() メソッドを用いて送信する各ディスパッチにも追加されます。データ管理 ガイドで、詳細を参照してください。

tealium.dataLayer.putString("key", "value", Expiry.FOREVER)

endTraceVisitorSession()

リモートで訪問者セッションを終了します。SDK セッションを終了せず、セッション ID をリセットしません。

Tealium["INSTANCE_NAME"]?.endTraceVisitorSession()

events

SDK から有益な情報を受け取るために、リスナー クラスを Tealium インスタンスに提供します。

以下の例は、同意設定の変更についてリスナーを登録するものです。リスナーには、新規の同意設定と、実施中の現行ポリシーが提供されます。 com.tealium.core.messagingにて入手可能な追加のリスナー。

tealium.events.subscribe(object: UserConsentPreferencesUpdatedListener {
    override fun onUserConsentPreferencesUpdated(userConsentPreferences: UserConsentPreferences, policy: ConsentManagementPolicy) {
        // take action on consent changes
    }
})

joinTrace()

現在のセッション用のデータレイヤーに、供給されたトレース ID を追加します。

Tealium["INSTANCE_NAME"]?.joinTrace(traceId)
パラメータ 型 説明
traceId String トレース ID

leaveTrace()

トレース ID を削除してトレースを終了します。

Tealium["INSTANCE_NAME"]?.leaveTrace()

linkEcidToKnownIdentifier()

現在の ECID を既知のセカンダリ識別子 (電子メール アドレスまたはその他の内部 ID など) にリンクします。

linkEcidToKnownIdentifier(
  knownId: String,
  adobeDataProviderId: String,
  authState: Int?,
  adobeResponseListener: ResponseListener<AdobeVisitor>?
  )
パラメータ 型 説明
knownId String 既知の識別子。
adobeDataProviderId String Adobe データ プロバイダー識別子。
authState Int 認証された状態。
adobeResponseListener ResponseListener<AdobeVisitor> Adobe レスポンス リスナー。

例:

var tealium: Tealium?
...

tealium.adobeVisitorApi?.linkEcidToKnownIdentifier(
  "myidentifier",
  "123456",
  AdobeAuthState.AUTH_STATE_AUTHENTICATED,
  null
  )

modules

SDK 用のモジュール マネージャーへのアクセス権を提供します。これまでは個別のモジュールへのアクセス権が提供されていました。

tealium.modules.getModule(VisitorService::class.java)?.delegate = myDelegate

resetVisitor()

次のトラッキング コールで新規のものを取得して、訪問者 ECID をリセットします。

resetVisitor()

例:

var tealium: Tealium?
//...
tealium?.adobeVisitorApi?.resetVisitor()

sendQueuedDispatches()

DispatchValidator により多くの理由からディスパッチのキューが登録されます。このメソッドを用いて再評価するリクエストを行うことができます。これにより、イベント バッチング制限が無視されますが、他の DispatchValidators を遵守します。たとえば、接続がなかった場合で、依然として接続がない場合には、キューが登録されたディスパッチについて、キューが登録されたままになります。

tealium.sendQueuedDispatches()

session

現在のセッションに関するすべてのデータを含めます。

val sessionId = tealium.session.id

startTimedEvent()

所与の名前を用いて時間付きイベントを開始します。同じイベント名でこのメソッドを改めて呼び出した場合でも、そのイベントが終了していない、またはキャンセルされていない場合には、無視されます。イベント開始時間に永続性が付与されていません。

任意のデータがイベント名とともに渡された場合は、 stopTimedEvent() コールによりタイマーが停止したときにトラッキング コールに追加されます。

tealium.startTimedEvent(name: "TIMED_EVENT_NAME", mapOf("custom_value" to "custom_key"))
パラメータ 型 説明
name String 時間付きイベントの名前
mapOf("custom_key" to "custom_value") Map (オプション) データレイヤー内でトラッキングされるキーと値のペアのオブジェクトのデータ

stopTimedEvent()

timed_event トラッキング コールをトリガーする時間付きイベント用のタイマーを停止します。

tealium.stopTimedEvent(name: "TIMED_EVENT_NAME")
パラメータ 型 説明
name String 時間付きイベントの名前

Tealium

Tealium のインスタンスを作成するコンストラクタ メソッド。

val config = TealiumConfig(key, config);
パラメータ 型 説明 例
key String 新しいTealiumインスタンス名 "abc123"
config Tealium.Config 新しいインスタンスの構成 tealConfigObj 
// Assuming execution is within Application.onCreate()
val config = TealiumConfig(this, "ACCOUNT_NAME", "PROFILE_NAME", Environment.PROD)
val tealium = Tealium.create("main", config)

モジュールは、バックグラウンド スレッドで初期化されますので、Tealium オブジェクトの作成後は直接利用できないことがあります。インスタンスが利用可能になり次第、後日必要となる設定を追加するために完了ブロックを追加します。

val tealium = Tealium.create("main", config) {
    events.subscribe(object : VisitorUpdatedListener {
        override fun onVisitorUpdated(visitorProfile: VisitorProfile) {
            Logger.dev("--", "VisitorProfile updated: $visitorProfile")
        }
    })
}

track()

スクリーン ビューやイベントなどの、システム内のイベントをトラッキングするために用いられるメソッド。

tealium.track(TealiumView("SCREEN_NAME", mapOf("key" to "context value")))

visitor

AdobeVisitor インスタンスの全部を返します。

var tealium: Tealium?
//...
val visitor = tealium.adobeVisitorApi?.visitor?.let { visitor ->
    val ecid = visitor.experienceCloudId
    val nextRefresh = visitor.nextRefresh
    val blob = visitor.blob
    val region = visitor.region
    val idSyncTTL = visitor.idSyncTTL
}

visitorId

訪問者 ID に関する現在の文字列。データレイヤー内に保存されますが、必要があればオーバーライドされることがあります。

val visitorId = tealium.visitorId
