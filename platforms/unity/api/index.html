UNITY

APIリファレンス

Tealium for Unityで提供されているクラスとメソッドに関するリファレンスガイド。

クラス：Tealium

以下は、Unity ライブラリで一般的に使用される Tealium クラスのメソッドをまとめたものです。

メソッド 説明
AddRemoteCommand() リモートコマンドをリモートコマンドマネージャに追加します。
AddToDataLayer() 永続性データレイヤーにデータを追加します。
GetFromDataLayer() データレイヤーからの指定値を取得します。
GetConsentCategories() ユーザーの同意済みカテゴリを取得します。
GetConsentStatus() ユーザーの同意ステータスを取得します。
GetVisitorId() 現在の訪問者 ID を取得します。
Initialize() 構成パラメータを用いて Tealium を初期化します。
JoinTrace() 所与の ID でトレースに参加します。
LeaveTrace() アクティブなトレースから退出し、訪問者セッションを終了します。
RemoveFromDataLayer() AddToDataLayer() を使用して以前設定された永続性データを削除します。
RemoveRemoteCommand() リモートコマンドをリモートコマンドマネージャから削除します。
SetConsentCategories() ユーザーの同意カテゴリを設定します。
SetConsentExpiryListener() 同意の有効期限が過ぎたリスナーまたはコールバックを設定します。
SetConsentStatus() ユーザーの同意ステータスを設定します。
SetVisitorServiceListener() 訪問者サービスのリスナーまたはコールバックを設定します。
Terminate() Tealium インスタンスを無効化し破棄します。
Track() ユーザーによるゲームとのインタラクション、またはゲーム ビューをトラッキングします。

AddRemoteCommand()

リモートコマンドをリモートコマンドマネージャに追加します。

TealiumUnityPlugin.AddRemoteCommand(id, callback);
パラメータ 型 説明 例
id string タグ構成からのコマンド ID の名前。 "test_command"
callback Action<string> リモート コマンドからレスポンスを受信した後に実行されるコールバック関数。コールバックは、タグマッピングからのキーと値のペアのペイロードを返します。 （例を参照）

例:

TealiumUnityPlugin.AddRemoteCommand("firebase", (payload) => {

  string? eventName = (string)payload["firebase_event_name"];
  Dictionary<string, object>? eventProperties =
      (Dictionary<string, object>)payload["firebase_event_properties"];

  if (eventName != null) {
    analytics().logEvent(eventName, eventProperties);
  }
});

AddToDataLayer()

所与の有効期限内に、永続性データ ストレージに永続性データを追加します。

TealiumUnityPlugin.AddToDataLayer(data, expiry);
パラメータ 型 説明 例
data Dictionary<string, object> キーと値のペアから成るJSONオブジェクト。ここで、キーは文字列で、値は単一文字列または文字列の配列になります。 new Dictionary<string, object> {{"KEY", "VALUE"}}
expiry Expiry データを永続化させる期間 (長さ)。 Expiry.Forever

Collectors

コレクター (Collectors) は、デバイスから補足情報を収集し、データレイヤーにそれを添付して、Tealium Customer Data Hub にそれを転送するモジュールです。コレクターの中にはコア ライブラリに含まれるものもあれば、オプションで、別個のモジュールとしてインストールされるものもあります。

以下の表では、利用可能なコレクターを列挙しています。デフォルト コレクターは、コレクター名に「*」印を付して注記しています。

Collector Name TealiumConfig リファレンス
AppData* Collectors.AppData
Connectivity* Collectors.Connectivity
Device Collectors.Device
Lifecycle Collectors.Lifecycle
VisitorService Collectors.VisitorService

ConsentExpiry

同意設定の有効期限を定義します。

ConsentExpiry(time, unit)
パラメータ 型 説明 例
time Number 有効期限が満了するまでの期間。 90
unit TimeUnit 有効期限が満了するまでの期間の単位。 TimeUnit.Days

例:

new ConsentExpiry(90, TimeUnit.Days)

TimeUnit

以下は、期間の単位の値です。

値 説明
ConsentExpiry.Minutes 期間単位 (分)。
ConsentExpiry.Hours 期間単位 (時間)。
ConsentExpiry.Months 期間単位 (月)。
ConsentExpiry.Days 期間単位 (日)。

ConsentPolicy

遵守するべき同意ポリシーを定義します。同意ポリシーが TealiumConfig オブジェクトについて定義されていない場合は、consent manager (同意マネージャー) が無効化されます。

値 説明
ConsentPolicy.GDPR GDPR 同意ポリシー。
ConsentPolicy.CCPA CCPA 同意ポリシー。

例:

ConsentPolicy.GDPR

Dispatchers

Dispatchers (ディスパッチャー) は、データレイヤーからデータを送信させ、これを Tealium エンドポイントに送信するモジュールです。現在使用可能なディスパッチャーは次のとおりです。

Dispatcher Name TealiumConfig リファレンス
Collect Dispatchers.Collect
RemoteCommands Dispatchers.RemoteCommands
TagManagement Dispatchers.TagManagement

少なくとも 1 つのディスパッチャーが必要です。ディスパッチャーが 1 つも指定されていない場合は、データはデータレイヤーに送信されません。

Expiry

カスタム データの有効期限を定義します。

値 説明
Session 現在のアクティブなセッションの有効期限。
UntilRestart アプリが再起動するまで。
Forever アプリの有効期限については、アプリが削除されるまで。

例:

Expiry.session

GetFromDataLayer()

永続性データレイヤーの指定キーの値を取得し、コールバック関数の形式で返します。

TealiumUnityPlugin.GetFromDataLayer(key);
パラメータ 型 説明
key string データレイヤーから取得するべきキー。
callback 想定される型に変換する必要がある object 型を返します。 データレイヤーキーからの値。

例:

string? stringValue =
    (string)TealiumUnityPlugin.GetFromDataLayer("test_string");

GetConsentCategories()

ユーザーの同意済みカテゴリを取得します。

TealiumUnityPlugin.GetConsentCategories();
Return Type 説明
List<ConsentCategories> ユーザーが同意した対象の ConsentCategories のリスト。

例:

List<string> caetgories = TealiumUnityPlugin.GetConsentCategories();
TealiumLogger.Log("Consent Categories: ");
caetgories.ForEach(category => TealiumLogger.Log(category.Value + " "));

GetConsentStatus()

ユーザーの同意ステータスを取得します。

TealiumUnityPlugin.GetConsentStatus();
Return Type 説明
ConsentStatus ユーザーの同意ステータス。

例:

TealiumLogger.Log("Consent Status: "
    + TealiumUnityPlugin.GetConsentStatus().Value);

GetVisitorId()

ユーザーの訪問者 ID を取得します。

TealiumUnityPlugin.GetVisitorId()
Return Type 説明
string Tealium 訪問者 ID。

例:

TealiumLogger.Log("Visitor Id: " + TealiumUnityPlugin.GetVisitorId())

Initialize()

他のメソッドを呼び出す前にTealiumを初期化します。オプションとして、Tealium の初期化が完了した後に実行される Action<bool> コールバックを渡します。

TealiumUnityPlugin.Initialize(config, callback);
パラメータ 型 説明
config TealiumConfig Tealium 構成パラメータ。
callback Action<bool> Tealium の初期化が成功した場合に true を返し、エラーがあった場合に false を返します。

例:

private TealiumConfig config =
      new TealiumConfig("tealiummobile",
      "demo",
      TealiumEnvironment.DEV,
      new List<Dispatchers> {
        Dispatchers.TagManagement,
        Dispatchers.Collect,
        Dispatchers.RemoteCommands },
      new List<Collectors> {
        Collectors.AppData,
        Collectors.DeviceData,
        Collectors.Lifecycle,
        Collectors.Connectivity },
      consentPolicy: ConsentPolicy.GDPR,
      consentExpiry: new ConsentExpiry(10, TimeUnit.Minutes));

TealiumUnityPlugin.Initialize(config, success => {
    if (success) {
        TealiumLogger.Log("TealiumUnityPlugin Initialized");
        TealiumUnityPlugin.SetConsentStatus(ConsentStatus.Consented);
        TealiumUnityPlugin.SetConsentExpiryListener(() =>
            TealiumLogger.Log("Consent Expired!!"));
    } else {
        TealiumLogger.Log("TealiumUnityPlugin Failed to Initialize");
    }
});

JoinTrace()()

指定されたIDでトレースに参加します。Tealium Customer Data Hubでの、 トレース 機能の詳細については、こちらから参照してください。

TealiumUnityPlugin.JoinTrace(id);
パラメータ 型 説明 例
id string CDH から取得されたトレース ID。 abc123xy

LeaveTrace()

アクティブなトレースから退出し、訪問者セッションを終了します。トレースは、アプリのセッション中、このメソッドが呼び出されるまでアクティブなままになります。

TealiumUnityPlugin.LeaveTrace();

LogLevel

ログ (記録) されている情報の量を制御するログ レベル プロパティを、以下の値のいずれかに設定します。

値 説明
LogLevel.Dev アプリケーションの進捗を表示する情報イベント。
LogLevel.Qa アプリケーションをデバッグするために用いられるデバッグ レベルのイベント。
LogLevel.Prod 重要なエラーと失敗などのエラー イベント。
LogLevel.Silent ログがありません (デフォルト)。

例:

LogLevel.Dev

SetConsentCategories()

ユーザーの同意カテゴリを設定します。カテゴリを設定するには、文字列の配列を渡します。デフォルトは、 ConsentStatus が .Consentedに設定されるまで空の配列です。同意ステータスが .Consented であって、 SetConsentCategories() メソッドを使用したカテゴリの指定がなされていない場合は、すべてのカテゴリが設定されています。

TealiumUnityPlugin.setConsentCategories(categories);
パラメータ 型 説明 例
categories List<ConsentCategories> ユーザーの同意カテゴリの配列 [ConsentCategories.email, ConsentCategories.personalization]

例:

TealiumUnityPlugin.SetConsentCategories(new List<ConsentCategories>(){ConsentCategories.Analytics, ConsentCategories.Email});

ConsentCategories

値 説明
Analytics Analytics 同意カテゴリー。
Affiliates Affiliates 同意カテゴリー。
DisplayAds Display Ads 同意カテゴリー。
Email 同意カテゴリーを電子メールで送信します。
Personalization Personalization 同意カテゴリー。
Search 検索同意カテゴリー。
Social ソーシャル同意カテゴリ。
BigData ビッグ データ同意カテゴリー。
Mobile モバイル同意カテゴリ。
Engagement エンゲージメント同意カテゴリー。
Monitoring モニタリング同意カテゴリー。
Crm CRM 同意カテゴリ。
Cdp CDP 同意カテゴリ。
CookieMatch Cookie 対応同意カテゴリー。
Misc その他同意カテゴリ。

SetConsentExpiryListener()

ユーザーの同意設定が ConsentExpiryに従って有効期限が切れた後に実行されるコールバックを定義します。Tealium が完全に初期化されたことを確認するために、初期化コールバックでリスナーを定義することを推奨します。

TealiumUnityPlugin.SetConsentExpiryListener(callback);
パラメータ 型 説明
callback Action 同意の有効期限が切れた後に実行するためのコード。

例:

TealiumUnityPlugin.SetConsentExpiryListener(()
    => TealiumLogger.Log("Consent Expired!!"));

SetConsentStatus()

ユーザーの同意ステータスを設定します。デフォルトは .Unknownです。

TealiumUnityPlugin.SetConsentStatus(status);
パラメータ 型 説明 例
status ConsentStatus ユーザー同意ステータス。 ConsentStatus.Consented

例:

TealiumUnityPlugin.SetConsentStatus(ConsentStatus.Consented);

ConsentStatus

値 説明
.Consented 同意済み同意ステータス。
.NotConsented 同意なし同意ステータス。
.Unknown 不明同意ステータス。

SetVisitorServiceListener()

訪問者プロファイルが更新されたときに実行されるコールバックを定義します。コールバック レスポンスでは、更新された VisitorProfile が提供されます。

VisitorServiceモジュールは、Tealium Customer Data Hubのデータレイヤーエンリッチメント機能を実装します。

Tealium AudienceStream のライセンスがある場合に、訪問者プロファイルを使用してモバイル アプリケーションのユーザー エクスペリエンスを強化するには、このモジュールを使用することをお勧めします。AudienceStreamのライセンスがない場合は、訪問者プロファイルが返されないため、このモジュールを使用することは推奨されません。

TealiumUnityPlugin.setVisitorServiceListener(callback);
パラメータ 型 説明
callback Action<Dictionary<string, object>> 更新された訪問者プロファイルが返された後に実行するためのコード。

例:

TealiumUnityPlugin.setVisitorServiceListener((profile) => {
  string? serializedProfile = JsonConvert.SerializeObject(profile);
  if (serializedProfile != null) {
      TealiumLogger.Log(serializedProfile);
  }
});

RemoveFromDataLayer()

TealiumUnityPlugin.AddToDataLayer() を使用して以前設定された永続性データを削除します。

TealiumUnityPlugin.RemoveFromDataLayer(keys);
パラメータ 型 説明 例
keys List<string> キー名の配列。 new List<string>(){"foo", "bar"}

RemoveRemoteCommand()

リモートコマンドをリモートコマンドマネージャから削除します。

TealiumUnityPlugin.RemoveRemoteCommand(id);
パラメータ 型 説明 例
id string 削除するコマンド ID の名前。 test_command

例:

TealiumUnityPlugin.RemoveRemoteCommand("firebase");

Track()

ユーザーによるゲームとのインタラクション、またはゲーム ビューをトラッキングします。

TealiumUnityPlugin.Track(dispatch);
パラメータ 型 説明 例
dispatch TealiumDispatch イベント名とデータレイヤーを使用しての Tealium ディスパッチ。 TealiumEvent("button_click")

ゲーム イベントをトラッキングするには、 TealiumEvent オブジェクトを作成し、これを track() メソッドに渡します。

TealiumEvent event = TealiumEvent('GAME_EVENT_NAME', new Dictionary<string, object> {{"KEY", "VALUE"}});
TealiumUnityPlugin.Track(event);

ゲーム ビューをトラッキングするには、 TealiumView オブジェクトを作成し、これを track() メソッドに渡します。

TealiumView view = new TealiumView("VIEW_NAME", new Dictionary<string, object> {{"KEY", "VALUE"}})
TealiumUnityPlugin.Track(view);

Terminate()

Tealium ライブラリを無効化し、すべてのモジュール設定を削除します。必要な場合に再有効化するには、新たな Tealium インスタンスを作成します。

TealiumUnityPlugin.Terminate();

インターフェイス: TealiumDispatch

トラッキング対象のディスパッチのタイプを定義するインターフェイス。次のクラスによって TealiumDispatchが実装されます。

クラス: TealiumEvent

ユーザーによるゲームとのインタラクション、またはゲーム ビューをトラッキングするには、 TealiumEvent(eventName, dataLayer) のインスタンスを Track() メソッドに渡します。 TealiumEvent は、 tealium_event としてトラッキング コールに表示されるイベント名と、オプションのデータ ディクショナリで構成されます。

TealiumEvent(viewName, dataLayer)
パラメータ 型 説明 例
eventName string ゲーム イベントの名前。 level_completed
dataLayer Dictionary<string, object> (オプション) イベントとともに送付されるデータ。 new Dictionary<string, object> {{"user_id", "gamer123"}, {"level": 2}, {"total_points": 17500}}

例:

TealiumEvent event = TealiumEvent('EVENT_NAME',
    new Dictionary<string, object> {{"KEY", "VALUE"}});
TealiumUnityPlugin.Track(event);

クラス: TealiumView

ゲームでスクリーン ビューをトラッキングするには、 TealiumView(viewName, dataLayer) のインスタンスを Track() メソッドに渡します。 TealiumView は、 tealium_event としてトラッキング コールに表示されるビュー名と、オプションのデータ ディクショナリで構成されます。

TealiumView(viewName, dataLayer)
パラメータ 型 説明 例
viewName string ゲームでのビューまたはスクリーンの名前。 login_view
dataLayer Dictionary<string, object> (オプション) イベントとともに送付されるデータ。 new Dictionary<string, object> {{"customer_id", "abc123"}}

例:

TealiumView view = new TealiumView("VIEW_NAME",
    new Dictionary<string, object> {{"KEY", "VALUE"}})
TealiumUnityPlugin.Track(view);
