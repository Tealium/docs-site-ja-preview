NATIVESCRIPT

Tealium

Tealiumクラスは、すべてのモジュールのメインAPIエントリポイントとして機能します。

クラス：Tealium

以下の文書は、一般的に使用される NativeScript プラグインの Tealium クラスのメソッドとプロパティをまとめたものです。

メソッド/Property 説明
addRemoteCommand() Adds a remote command to the remote command manager
addData() Adds data to persistent data storage
consentCategories Gets or sets the consent categories of a user
consentStatus Gets or sets the consent status of a user
getData() Retrieves data from the data layer
initialize() 構成パラメータを用いて Tealium を初期化します
joinTrace() Joins a trace with the given ID
leaveTrace() Leaves active trace session
removeData() Remove persistent data that has been previously set using addData()
removeRemoteCommand() リモートコマンドをリモートコマンドマネージャから削除します
removeVisitorServiceListener() Removes the visitor service listener/callback
setVisitorServiceListener() Defines the visitor service listener/callback
terminateInstance() Disables the Tealium library and removes all module references
track() Track an event or screen view
visitorId 現在の訪問者 ID を取得します

addRemoteCommand()

リモートコマンドをリモートコマンドマネージャに追加します。

Tealium.addRemoteCommand(id, callback);
パラメータ 型 説明 例
id String タグ構成からのリモート コマンド ID の名前 "test_command"
callback Function リモートコマンドから応答を受信すると実行されるコールバック関数。コールバックは、タグマッピングからのキーと値のペアのペイロードを返します。 （例を参照）

例:

Tealium.addRemoteCommand('mycommand', (result: any): void => {
	console.log(result["payload"]["command_id"]);            // logs "mycommand"
	console.log(result["payload"]["my_remote_command_key"]); // logs value for key "my_remote_command_key"
});

addData()

所与の 有効期限中、永続性データ ストレージに永続性データを追加します。

Tealium.addData(data, expiry);
パラメータ 型 説明 例
data Object キーと値のペアのマップ。ここで、キーは文字列で、値は単一文字列または文字列の配列になります {"persistent_key2" : "persistent_val2"}
expiry Expiry データを永続化する期間 (長さ) Expiry.forever

例:

let data: Map<string, any> = new Map([['test_session_data', 'test']]);
data.set('my_test_value', 1);
data['my_test_value'] = 1;
Tealium.addData(data, Expiry.session);

Expiry

有効期限のあるプロパティの設定に関する有効期限を定義します。

使用可能な有効期限オプションは次のとおりです。

値 説明
Expiry.session (default) 現在のセッションの終了時に有効期限が切れます
Expiry.forever アプリがインストールされている間は有効期限は決して切れません
Expiry.untilRestart デバイスが再起動されたときに有効期限が切れます

consentCategories

ユーザーの同意カテゴリを取得または設定します。

ユーザーの同意カテゴリを設定には:

Tealium.consentCategories = categories

ユーザーの同意カテゴリを取得するには:

let categories = Tealium.consentCategories
パラメータ 型 説明 例
categories ConsentCategories[] ユーザーの同意カテゴリの配列 [ConsentCategories.email, ConsentCategories.personalization]

例:

Tealium.consentCategories = [ConsentCategories.analytics, ConsentCategories.email];

使用可能な同意カテゴリは次のとおりです。

値 説明
analytics 分析
affiliates 加盟企業
displayAds ディスプレイ広告
email Eメール
personalization パーソナライゼーション
search 検索
social ソーシャル
bigData ビッグデータ
mobile モバイル
engagement エンゲージメント
monitoring モニタリング
crm CRM
cdp CDP
cookieMatch Cookie照合
misc Misc (雑)

consentStatus

ユーザーの同意ステータスを取得または設定します。デフォルトは、変更されるまで .unknown です。

ユーザーの同意ステータスを設定するには:

Tealium.consentStatus = status;

ユーザーの同意ステータスを取得するには:

let status = Tealium.consentStatus
パラメータ 型 説明 例
status ConsentStatus ユーザーの同意ステータス ConsentStatus.consented

例:

Tealium.consentStatus = ConsentStatus.consented;

使用可能な同意ステータスは次のとおりです。

値 説明
.consented 同意あり
.notConsented 同意なし
.unknown 不明

getData()

データレイヤーから、 any 型としてのデータを取得します。

let data = Tealium.getData(key);
パラメータ 型 説明 例
key String 取得するべきデータのキー "KEY" 
let data: Tealium.getData("KEY")

initialize()

他のメソッドを呼び出す前にTealiumを初期化します。

Tealium.initialize(config);
パラメータ 型 説明 例
config TealiumConfig Tealium 構成パラメータ （例を参照）

例:

let config: TealiumConfig =
{
	account: 'ACCOUNT',
	profile: 'PROFILE',
	environment: TealiumEnvironment.dev,
	dispatchers: [Dispatchers.Collect,
	 			  Dispatchers.TagManagement,
	  			Dispatchers.RemoteCommands],
	collectors: [Collectors.AppData,
				 Collectors.DeviceData,
				 Collectors.Lifecycle,
				 Collectors.Connectivity],
	consentLoggingEnabled: true,
	consentPolicy: ConsentPolicy.gdpr,
	visitorServiceEnabled: true
};

Tealium.initialize(config);

joinTrace()

指定されたIDでトレースに参加します。Tealium Customer Data Hubでの、 トレース 機能の詳細については、こちらから参照してください。

Tealium.joinTrace(id);
パラメータ 型 説明 例
id String のデータ ソース キーから取得されたトレース ID abc123xy

例:

Tealium.joinTrace("abc123xy");

leaveTrace()

トレースは、アプリのセッション中、 leaveTrace() メソッド (以前に参加したトレースから退出し、訪問者セッションを終了) が呼び出されるまでアクティブなままになります。

Tealium.leaveTrace();

removeData()

Tealium.addData() を使用して以前設定された永続性データを削除します。

Tealium.removeData(keys);
パラメータ 型 説明 例
keys String[] キー名の配列 ["foo", "bar"]

例:

Tealium.removeData(["foo", "bar"]);

removeRemoteCommand()

リモートコマンドをリモートコマンドマネージャから削除します。

Tealium.removeRemoteCommand(id);
パラメータ 型 説明 例
id String 削除するコマンドIDの名前 "test_command"

例:

Tealium.removeRemoteCommand("test_command");

setVisitorServiceListener()

訪問者プロファイルが更新されたときに実行されるコールバックを定義します。コールバック レスポンスでは、更新された VisitorProfile が提供されます。

VisitorServiceモジュールは、Tealium Customer Data Hubのデータレイヤーエンリッチメント機能を実装します。

Tealium AudienceStream のライセンスがある場合に、訪問者プロファイルを使用してモバイル アプリケーションのユーザー エクスペリエンスを強化するには、このモジュールを使用することをお勧めします。AudienceStreamのライセンスがない場合は、訪問者プロファイルが返されないため、このモジュールを使用することは推奨されません。

Tealium.setVisitorServiceListener(callback);
パラメータ 型 説明 例
callback Function 更新された訪問者プロファイルが返され次第、実行するためのコード （例を参照）

以下の例は、訪問者のオーディエンスとバッジをコンソールに記録するものです。

Tealium.setVisitorServiceListener(profile => {
    console.log(profile["audiences"]);
    console.log(profile["badges"]);
});

terminateInstance()

Tealium ライブラリを無効化し、すべてのモジュール設定を削除します。必要な場合に再有効化するには、新たな Tealium インスタンスを作成します。

Tealium.terminateInstance();

track()

TealiumEvent または TealiumView のいずれかのディスパッチによってイベントまたはスクリーン ビューをトラッキングします。

Tealium.track(dispatch);

パラメータ 型 説明 例
dispatch TealiumDispatch イベント/ビュー名とデータレイヤーを使用しての Tealium ディスパッチ TealiumEvent("button_click", 新規 Map([["button_name", "Submit"]]))

TealiumView

スクリーン ビューをトラッキングするには、 TealiumView(viewName, data) のインスタンスを track() メソッドに渡します。 TealiumView は、 tealium_event としてトラッキング コールに表示されるビュー名と、オプションのデータ ディクショナリで構成されます。

let tealView = new TealiumView("VIEW_NAME", new Map([["key", "value"]]));
Tealium.track(tealView);

TealiumEvent

非ビュー イベントをトラッキングするには、 TealiumEvent(eventName, data) のインスタンスを track() メソッドに渡します。 TealiumEvent は、 tealium_event としてトラッキング コールに表示されるイベント名と、オプションのデータ ディクショナリで構成されます。

以下は例です。

let tealEvent = new TealiumEvent("EVENT_NAME", new Map([["key", "value"]]));
Tealium.track(tealEvent);

visitorId

現在の訪問者 ID を取得します。

let visitorId = Tealium.visitorId;
TealiumDispatch

トラッキングの対象のディスパッチのタイプ、およびイベントとデータレイヤーの名前またはタイトルを定義するインターフェイス。
