GOOGLE TAG MANAGER

データレイヤー

Google Tag Manager マーケットプレイスでの Collect タグ用のビルトイン (自動) データレイヤー変数について説明します。

データレイヤーには、トラッキングされるページの基本情報を収集するビルトイン変数が含まれています。これらの変数には、Cookie、ページの標準 DOM 変数、および読み込まれる構成に関する Tealium 固有の変数があります。

詳細は、使用可能なデータレイヤー変数の種類に関するページを参照してください。

標準的なページデータ

以下の変数は、ウェブ ページで使用できる標準 JavaScript プロパティから生成されます。

表示されるサンプル値は、次のページ URL に基づいています。

http://www.example.com/path/file.html?param1=value1#hash=fragment

変数 説明 例
dom.domain URLの完全ドメイン。ソース: location.hostname www.example.com
dom.hash URLのハッシュフラグメント（文字#を除く）。ソース: location.hash hash=fragment
dom.pathname URLのパス (クエリ パラメータとドメインを除く)。ソース: location.pathname "/path/file.html"
dom.query_string URLの完全クエリ文字列。ソース: location.search param1=value1
dom.referrer 前のページのURL。ソース: document.referrer
dom.title <title> タグのテキスト。ソース: document.title "Page Title"
dom.url ページの完全URL。ソース: document.URL http://www.example.com/ PATH/FILE.html?param1=VALUE#hash=FRAGMENT
dom.viewport_height ブラウザのビューポートの高さ。ソース: window.innerHeight or document.documentElement.clientHeight 1320
dom.viewport_width ブラウザのビューポートの幅。ソース: window.innerWidth or document.documentElement.clientWidth 1278

Cookie

次の変数は、 TEALという名称の単一の Cookie に保存され維持されます。それぞれが TEAL オブジェクトの変数です。

変数 説明 例
tealium)_session_event_number 現在のセッションのイベント数。 2
tealium_session_number このユニーク訪問者のセッション数。 1
tealium_session_id セッションの一意識別子。 1628542077230
tealium_visitor_id 訪問者の一意識別子。文字数は 40 字以上です。 016297481...

次は、document.cookie 形式の TEAL Cookie の例です。これは、 $ という文字で区切られた変数値を示しています。

TEAL=v:927b2ebc092a70110329356772472215264145f9fa1$t:1628543914615$s:1628542077230%3Bexp-sess$sn:1$en:3

Tealiumデータ

変数 名前 例
tealium_account アカウント名 sandbox
tealium_datasource データソースキー abc123
tealium_environment 公開環境 prod
tealium_event Tealiumイベント名 my_event
tealium_library_name ライブラリ名 tealium.js
tealium_library_version ライブラリバージョン 5.0.3
tealium_profile アカウントプロファイル main
tealium_random 乱数 7782219645308327
tealium_timestamp_epoch 現在のUNIXタイムスタンプ（秒単位） 1522956509
tealium_timestamp_local 現地時間タイムスタンプ 2018-04-05T12:28:29.019
tealium_timestamp_utc UTCタイムスタンプ 2018-04-05T19:28:29.019z
tealium_cookie_domain TEAL Cookie が設定されるドメイン example.com
tealium_domain ウェブサイトのドメイン example.com

データレイヤー変数の平坦化の方法

Google Tag Manager イベントは、Tealium Customer Data Hub 仕様を満たすために Tealium 属性名に平坦化されます。イベント データは、Tealium EventStream から受け取ります。

tealium_event の値は、 event の値 (dataLayer.push コール内のもの) です。Google Tag Manager は、デフォルト ページ ビューをイベント名としての gtm.js に設定します。デフォルトをオーバーライドするには、Tealium Event 構成設定を、イベント名を含む変数に設定します。

属性は、自動的には追加されません。この属性を、Tealium EventStream に追加する必要があります。

Eコマース

以下の購入イベントの例は、クライアントサイド E コマースと引きイベントが平坦化する仕組みを示しています。

dataLayer.push({
  'ecommerce': {
    'purchase': {
      'actionField': {
        'id': 'T12345',    // Transaction ID. Required for purchases and refunds.
        'affiliation': 'Online Store',
        'revenue': '35.43',
        'tax':'4.90',
        'shipping': '5.99',
        'coupon': 'SUMMER_SALE'
      },
      'products': [{
        'name': 'Triblend Android T-Shirt',     // Name or ID is required.
        'id': '12345',
        'price': '15.25',
        'brand': 'Google',
        'category': 'Apparel',
        'variant': 'Gray',
        'quantity': 1,
        'coupon': ''
       },
       {
        'name': 'Donut Friday Scented T-Shirt',
        'id': '67890',
        'price': '33.75',
        'brand': 'Google',
        'category': 'Apparel',
        'variant': 'Black',
        'quantity': 1
       }]
    }
  }
});

平坦化した後、購入イベントが、次のようになります:

"ecommerce.purchase.actionField.affiliation": "Online Store"
"ecommerce.purchase.actionField.coupon": "SUMMER_SALE"
"ecommerce.purchase.actionField.id": "T12345"
"ecommerce.purchase.actionField.revenue": "35.43"
"ecommerce.purchase.actionField.shipping": "5.99"
"ecommerce.purchase.actionField.tax": "4.90"
"ecommerce.purchase.products.brand": [ "Google", "Google" ]
"ecommerce.purchase.products.category": [ "Apparel", "Apparel" ]
"ecommerce.purchase.products.coupon": [ "", "" ]
"ecommerce.purchase.products.id": [ "12345", "67890" ]
"ecommerce.purchase.products.name": [ "Triblend Android T-Shirt", "Donut Friday Scented T-Shirt" ]
"ecommerce.purchase.products.price": [ "15.25", "33.75" ]
"ecommerce.purchase.products.variant": [ "Gray", "Black" ]
"ecommerce.purchase.products.quantity": [ "1", "1" ]

フラットナー (flattener) は、キーを作成するためにドット表記を使用しており、値は、購入イベントからのすべての値の配列に設定されます。

イベント

以下の dataLayer.push コマンドは、「イベント」がデータレイヤーに含まれるときにトリガーされます。

window.dataLayer.push({
  event: "test_number",
  type_number: 5,
  type_string: 'hello',
  type_object: { someKey: 'someValue' },
  type_array: [1,2,3,4],
  type_function: function() { return 'hello'; },
  type_boolean: true
});

Tealium EventStream に現れるデータレイヤーは、以下のように表示されます。type_object のみ平坦化され、機能が削除されます。

{
    "event": "test_number",
    "type_number": 5,
    "type_string": "hello",
    "type_object.someKey": "someValue",
    "type_array": [1, 2, 3, 4],
    "type_boolean": true
}
