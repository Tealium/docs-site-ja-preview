仕組み

モバイル向けリモート コマンド機能について説明します。

リモート コマンドは、ネイティブ アプリ内の機能をトリガーするための、クライアントサイドのソリューションです。

リモートコマンドは次の2通りの方法で使用します。

  • ベンダー統合 Tealium は、ベンダーのネイティブ API を実装するプリビルトのリモート コマンド モジュールを提供します。リモートコマンド統合のリストを参照してください。
  • カスタムコマンド 自分のネイティブ カスタム コマンドを定義し、Tealium iQ タグ管理でカスタム リモートコマンド タグを使用してこれをトリガーします。

ベンダー統合

ベンダー統合は、事前にビルドされたリモートコマンドモジュールであり、ベンダーSDKをインストールしたり、ベンダーAPI用のネイティブコードハンドラーを定義したりします。

ベンダー統合の実装には、以下が必要です。

  • ベンダー リモート コマンド モジュール アプリのビルド スクリプトで、ベンダー SDK を Tealium リモート コマンド モジュールに置換します。リモートコマンドモジュールにより、ベンダーSDKのインストールとセットアップが処理されます。
  • 構成 (次のいずれを選択します)
    • JSON ファイル ローカルに読み込まれた、またはリモートでホスティングされる JSON ファイルで、ベンダー設定、データ マッピング、イベント トリガーを含むもの。
    • リモート コマンド タグ ベンダーの API に関する構成オプションを提供する iQ Tag タグ管理のタグ (タグ管理モジュールとともに用いる場合)。

リモート コマンド ベンダー統合のリストを参照してください。

リモート コマンドは、特定のベンダー SDK リリースに構成されます。これは、Tealium リモート コマンド コードと、ベンダー SDK との間の非適合性を避けるためです。ベンダー SDK 機能の更新版が必要な場合は、Tealium サポートに連絡して、アップグレードの支援を受けてください。

JSON ファイル

JSON ファイルを使用してベンダー統合を構成します。このファイルの読み込みを、アプリ内にローカルに行うか、またはリモートでホスティングされている URL から行います。各ベンダー統合のために JSON テンプレート ファイル が利用可能です。

マッピング

JSON テンプレート ファイルには、3 つの主なセクション (config、 mappings、および commands) があります。

  • config このセクションは、ベンダー SDK 初期化オプション (クライアント ID、API キー、またはログ レベルなど) を設定します。たとえば、次のものは、アプリケーション ID を設定します。
{
      "config": {
        "applicationid": "appid123"
      }
}
  • mappings このセクションは、データ マッピングを定義します。以下のマッピングは、データレイヤー変数 product_id をベンダー パラメータ param_item_idに割り当てます。
{
      "mappings": {
        "product_id": "param_item_id"
      }
}

ベンダー パラメータは、ドット表記付きの接頭辞を使用して、ぺイロード内の個別のオブジェクトのマッピングをスコープとすることができます。この接頭辞は、オブジェクトを定義し、 . の右にある値がオブジェクトのプロパティになります。たとえば、以下のマッピングは、 product_id を param_item_id 変数 (event オブジェクトのもの) に割り当てます。リモート コマンド パーサ (parser) は、 event オブジェクトを、そのプロパティの 1 つの param_item_id によって作成します。

{
      "mappings": {
        "product_id": "event.param_item_id"
      }
}

オブジェクト接頭辞は、一般的には event.、 product.、または purchase.ですが、カスタム オブジェクト名にすることができます。JSON テンプレート ファイル を参照して、ベンダー統合のために使用されるオブジェクト接頭辞を確認してください。

  • commands このセクションは、Tealium イベント名をベンダーのイベントにマッピングします。: の左の値は、Tealium イベント名であり、右の値は、ベンダー メソッドに関する Tealium ラッパーです。次の例では、Tealium インベント cart_add は、ベンダー メソッド logeventにマッピングされています。
{
      "commands": {
        "cart_add": "logevent"
      }
}

例

以下は、JSON 構成ーの例です。

{
    "config": {
        "analytics_enabled": "true",
        "session_timeout_seconds": "30",
        "log_level": "max",
        "minimum_seconds": "100",
        ...
    },
    "mappings": {
        "event_title": "event_name",
        "product_brand": "event.param_item_brand",
        "product_category": "event.param_item_category",
        "product_id": "event.param_item_id",
        "product_list": "event.param_item_list",
        "product_location_id": "event.param_item_location_id",
        "product_name": "event.param_item_name",
        "product_variant": "event.param_item_variant",
        ...
    },
    "commands": {
        "launch": "config",
        "cart_add": "logevent",
        ...
    }
}

読み込みオプション

JSON 構成ファイルの読み込みには、ローカル ファイルとして行うか、または、ホスト型 URL から行うかの、2 つのオプションがあります。

    iOS (Swift) アプリに保存されたローカル構成ファイルを使用するには:

    let config = TealiumConfig(account: "ACCOUNT",
        profile: "PROFILE",
        environment: "ENVIRONMENT")
config.dispatchers = [Dispatchers.TagManagement, Dispatchers.RemoteCommands]
config.remoteAPIEnabled = true

tealium = Tealium(config: config) { _ in
    guard let remoteCommands = self.tealium?.remoteCommands else {
        return
    }
  	let facebook = FacebookRemoteCommand(type: .local(file: "facebook"))
	remoteCommands.add(facebook)
}

    ホスト型データレイヤーを用いて、URL としてアクセス可能な構成ファイルをホスティングします。

    iOS (Swift) アプリで、ホスト型構成ファイルを使用するには:

    let config = TealiumConfig(account: "ACCOUNT",
        profile: "PROFILE",
        environment: "ENVIRONMENT")
config.dispatchers = [Dispatchers.TagManagement, Dispatchers.RemoteCommands]
config.remoteAPIEnabled = true

tealium = Tealium(config: config) { _ in
    guard let remoteCommands = self.tealium?.remoteCommands else {
        return
    }
  	let facebook = FacebookRemoteCommand(type: .remote(url: "https://tags.tiqcdn.com/dle/ACCOUNT/PROFILE/facebook.json"))
	remoteCommands.add(facebook)
}

    Android (Kotlin) アプリに保存されたローカル構成ファイルを使用するには:

    val config = TealiumConfig(application,
        "ACCOUNT", "PROFILE",
        Environment.DEV,
        dispatchers = mutableSetOf(Dispatchers.RemoteCommands));
var tealium = Tealium.create(TEALIUM_MAIN, config) {
  val facebook = FacebookRemoteCommand(this);
  remoteCommands?.add(facebook, filename = "facebook.json");
}

    ホスト型データレイヤーを用いて、URL としてアクセス可能な構成ファイルをホスティングします。

    Android (Kotlin) アプリで、ホスト型構成ファイルを使用するには:

    val config = TealiumConfig(application,
        "ACCOUNT", "PROFILE",
        Environment.DEV,
        dispatchers = mutableSetOf(Dispatchers.RemoteCommands));
var tealium = Tealium.create(TEALIUM_MAIN, config) {
  val facebook = FacebookRemoteCommand(this);
  remoteCommands?.add(facebook, remoteUrl = "https://tags.tiqcdn.com/dle/ACCOUNT/PROFILE/facebook.json");
}

    同意管理

    ベンダー SDK が 同意管理をサポートしていて、クライアントがベンダー SDK について同意の管理を行う場合は、ベンダー SDK がデフォルでオプトアウトに設定されていることを確認します。Tealium SDK に保存された同意ステータスに基づき、ユーザーがオプトインしている場合は、このベンダーのために Tealium iQ リモート コマンド タグを使用します。

    カスタム コマンド

    アプリで iQ タグ管理を使用しているときは、カスタム リモート コマンドを使用すると、非表示のウェブビューからネイティブ コード ブロックをトリガーできます。この機能は、非表示のウェブビュー内に保持されているデータをさらに処理できるよう、ネイティブ コードに戻す必要がある場合に使用されます。

    リモート コマンドでは、アプリ内で JavaScript タグによってメソッドをトリガーする方法として、カスタム URL スキームが使用されます。レンダリングされていないウェブビューのタグはアプリ リクエストを送信し、これは、ネイティブ コードまたはベンダーの SDK をトリガーするリモート コマンド ハンドラーによって処理されます。

    コンポーネント

    リモートコマンドには、コマンド名とデータのペイロードがあります。

    • コマンド コマンド名、またはコマンド ID。アプリ内のネイティブ コードに登録されています。
    • ペイロード ネイティブ アプリに渡され、ハンドラーのレスポンス コールバックで requestPayload という名称のオブジェクトとして受信されるデータ。

    リモート コマンドをビルド時にネイティブ アプリで定義します。カスタム リモート コマンド タグでは、デバイス上で事前定義済みのコードのみ実行されます。

    ネイティブコード

    ネイティブ コードでは、 add() 関数が、リモート コマンド ハンドラーを登録します。コールバック関数 response は、 response.payloadによってペイロードにアクセスできます。以下の例では、参照されるペイロード変数に、iQ タグ管理で構成され、対応するデータレイヤー変数があります。

      tealium = Tealium(config: config) { [weak self] _ in
	let remoteCommand = RemoteCommand(commandId: "myRemoteCommand", description: "") { response in
		guard let payload = response.payload,
	    let myVariable = payload["myVariable"] as? String else {
			return
		}
		print(myVariable)
	}
	self.tealium?.remoteCommands?.add(remoteCommand)
}

      val remoteCommand = object : RemoteCommand("sample", "testing RCs") {
    override fun onInvoke(response: Response) {
        Logger.dev(BuildConfig.TAG, "ResponsePayload for webView RemoteCommand ${response.requestPayload}")
    }
}
val tealium = Tealium.create(instanceName, config) {
    remoteCommands?.add(webViewRemoteCommand)
}

      ベンダー統合用のモジュールを、お使いのプラットフォーム用のビルドスクリプトに追加します。モジュールはTealiumVendorXYZの形式で命名されます。例：TealiumBraze。

      インストールするには、ベンダーの依存関係をTealiumのリモートコマンドの依存関係に置換します。たとえば、以下の行を削除します。

      pod "VendorXYZ-iOS-SDK"

      次に、以下の行を追加します。

      pod "TealiumVendorXYZ"

      使用ケース: 購入アンケート

      以下の手順は、ユーザーが購入を完了した後に、お使いのアプリでアンケートをトリガーするためのリモート コマンドの使用法を示しています。

      1. アンケートへの参加をユーザーに求めるメッセージを表示するネイティブ コードを作成します。ここで、メッセージと URL は、メソッド パラメータとして (レンダリングされていないウェブビューから) 渡されます。
      2. コード ブロックをリモート コマンドとして Tealium API に登録します。
      3. iQ タグ管理で、カスタム リモート コマンド タグを追加して、ネイティブ コードに登録されている同じコマンドにコマンド ID を設定します。
      4. survey_titleやsurvey_urlなどの特定の変数をアプリに渡すようにデータマッピングを設定します。
      5. ネイティブ コードで、レスポンス ペイロードで返された変数を使用して、ユーザーにアンケートを表示します。

      こうした手順を完了した後、アプリを再デプロイすることなく、iQ タグ管理の拡張機能を使用してアンケートのタイトル、URL、ロジックを構成できます。

      リモートコマンドタグ

      リモート コマンドがカスタム リモート コマンド タグのインスタンスとして以下の設定でセットアップされた iQ タグ管理を使用します。

      • コマンドID コマンド名（ネイティブメソッドaddRemoteCommandIDで使用されているのと同じ名前）。
      • 読み込みルール リモートコマンドをトリガーするタイミングを定めたルール。
      • マッピングされた変数 リモートコマンドに含めるデータ。マッピングされたデータレイヤー変数は、ネイティブ コード コールバックの response.requestPayload オブジェクトで利用できます。

      リモート コマンド タグの例

      以下は、Swift でトラッキングされたイベントの例です。

      tealium?.track(
    title: "My Screen",
    data: ["tealium_event": "my_event", "my_variable": "my_value"]
);

      タグ セットアップは、次のように定義されています。

      • コマンドID myRemoteCommand
      • 読み込みルール IF tealium_event EQUALS my_event
      • マッピングされた変数 my_variable -> myVariable
