REMOTE COMMANDS

仕組み

リモートコマンドは、Tealium iQ Tag Management内のタグを使用してネイティブアプリ内のコードをトリガーするための、クライアントサイドのソリューションです。

リモートコマンドにより、ネイティブアプリのコードが持つ力と、Tealium iQ Tag Managementの利便性とタグ管理を統合できます。リモートコマンドでは、アプリ内でJavaScriptタグによってメソッドをトリガーする方法として、カスタムURLスキームが使用されます。レンダリングされていないウェブビュー内のタグによりカスタムリクエストが送信され、それをアプリが受信します。これらのリクエストは、カスタムネイティブコードまたはベンダーSDKをトリガーするリモートコマンドハンドラーによって処理されます。

リモートコマンドは次の2通りの方法で使用します。

  • ベンダー統合 ベンダーAPIを実装するネイティブリモートコマンドモジュールと、iQ Tag Management内の対応するベンダータグを使用しているベンダーSDK用に、事前にビルドされたソリューション。リモートコマンド統合のリストを参照してください。
  • カスタムコマンド ネイティブコードで定義され、iQ Tag Managementでカスタムリモートコマンドタグにより構成された、自分のカスタムコマンド。

はじめに

リモートコマンドを使用すると、Tag Managementをアプリで使用するときに、非表示のWebビューからネイティブコードブロックをトリガーできます。この機能は、非表示のWebビュー内に保持されているデータをさらに処理できるよう、ネイティブコードに戻す必要がある場合に使用されます。

たとえば、非表示のWebビュー内のCookieまたはlocalStorageに保存されている値にアクセスする必要がある場合や、ユーザーによる購入後にアンケートを開始するためにアプリの一部をパーソナライズする必要がある場合が当てはまります。

購入後のアンケートを設定する手順は次のとおりです。

  1. アンケートへの参加をユーザーに求めるメッセージを表示できるネイティブコードを作成します。ここで、アンケートのメッセージとURLはメソッドパラメータとして渡されます（非表示のWebビューから取得されます）。
  2. このコードブロックをリモートコマンドとしてTealium APIに登録します。
  3. iQ Tag Managementで、カスタムリモートコマンドタグを追加して、ネイティブコードに登録されている同じコマンドにコマンドIDを設定します。
  4. survey_titleやsurvey_urlなどの特定の変数をアプリに渡すようにデータマッピングを設定します。
  5. ネイティブコードで、応答ペイロードで返された変数を使用して、ユーザーにアンケートを表示します。

アプリを再デプロイすることなく、iQ Tag Managementの拡張機能を使用してアンケートのタイトル、URL、ロジックを制御できるようになりました。

コンポーネント

リモートコマンドには、コマンド名とデータのペイロードがあります。

  • コマンド コマンド名、またはコマンドID。アプリ内のネイティブコードとiQ Tag Management内のタグに登録されています。
  • ペイロード タグからネイティブアプリに渡されるデータ。iQ Tag Managementでデータマッピングとして構成され、アプリ内でハンドラーのレスポンスコールバックのrequestPayloadオブジェクトとして受信されます。

リモートコマンドはビルド時にネイティブアプリで定義されている必要があります。リモートコマンドタグでは、デバイス上で定義済みのコードのみ実行されます。

ネイティブコード

ネイティブコードでは、addRemoteCommandIDというヘルパー関数が、リモートコマンド用のハンドラーを登録します。コールバック関数responseBlockは、response.requestPayload内のペイロードにアクセスします。ここで参照されるペイロード変数には、iQ Tag Managementで構成され、対応するデータ層変数があるはずです。

TealiumHelper.shared.tealium.addRemoteCommandID(
    "myRemoteCommand",
    description: "",
    targetQueue: DispatchQueue.main,
    responseBlock: { response in
        guard let myVariable = response.requestPayload["myVariable"] as? String else {
            return
        }
    }
)

カスタムリモートコマンドタグ

iQ Tag Managementでは、リモートコマンドはカスタムリモートコマンドタグのインスタンスとして、以下の設定でセットアップされます。

  • コマンドID コマンド名（ネイティブメソッドaddRemoteCommandIDで使用されているのと同じ名前）。
  • タグ配信ルール リモートコマンドをトリガーするタイミングを定めたルール。
  • マッピングされた変数 リモートコマンドに含めるデータ。マッピングされたデータ層変数は、ネイティブコードコールバックのresponse.requstPayloadオブジェクトで利用できます。

リモートコマンドタグの例：

トラッキングされたイベント：

tealium?.track(
    title: "My Screen",
    data: ["tealium_event": "my_event", "my_variable": "my_value"]
);

タグのセットアップ：

  • コマンドID myRemoteCommand
  • タグ配信ルール IF tealium_event EQUALS my_event
  • マッピングされた変数 my_variable -> myVariable

ベンダー統合

ベンダー統合は、事前にビルドされたリモートコマンドモジュールであり、ベンダーSDKをインストールしたり、ベンダーAPI用のネイティブコードハンドラーを定義したりします。

リモートコマンド統合の実装には、以下が必要です。

  • ベンダーリモートコマンドモジュール アプリのビルドスクリプトで、ベンダーSDKをリモートコマンドモジュールに置換します。リモートコマンドモジュールにより、ベンダーSDKのインストールとセットアップが処理されます。
  • ベンダーリモートコマンドタグ iQ Tag Managementで、ベンダー統合用のリモートコマンドタグを追加します。目的のベンダーメソッドを実装するには、タグ設定とデータマッピングを構成します。

ネイティブコード

ベンダー統合用のモジュールを、お使いのプラットフォーム用のビルドスクリプトに追加します。モジュールはTealiumVendorXYZの形式で命名されます。例：TealiumBraze。

インストールするには、ベンダーの依存関係をTealiumのリモートコマンドの依存関係に置換します。例：

// Remove this line
pod "VendorXYZ-iOS-SDK"

// Add this line
pod "TealiumVendorXYZ"

リモートコマンドタグ

iQ Tag Managementでは、ベンダー統合用のリモートコマンドタグが提供されています。リモートコマンドタグは、通常のウェブサイトのタグのように使用します。各リモートコマンドタグは、ベンダーのSDKメソッド用のイベントマッピングを提供します。イベントマッピングは、アプリのモジュールでリモートコマンドとして定義されています。
