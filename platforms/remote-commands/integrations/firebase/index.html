REMOTE COMMANDS/INTEGRATIONS

リモートコマンド：Firebase

AndroidおよびSwift/iOSにおけるFirebaseのTealiumリモートコマンド統合です。

要件

  • 次のいずれかのモバイルライブラリ:
    • Tealium for Android-Kotlin (1.0.0+)
    • Tealium for Android-Java (5.9.0+ for Firebase 2.0.0+ or <5.9.0 for previous versions)
    • Tealium for iOS-Swift
  • 次のいずれかのリモートコマンド連携:
    • Firebase Remote Command JSON File (Requires Android-Kotlin 1.0.0+ or iOS-Swift 2.1.0+)
    • Firebase Remote Command tag in Tealium iQ Tag Management

動作

Firebase連携には、下記の３つのアイテムが必要となります:

  1. FirebaseのネイティブSDK
  2. Fireabaseモジュールを内包するリモートコマンドのモジュール
  3. JSON設定ファイル、もしくは（Tealiumの）イベントをFirebaseネイティブの機能呼び出しに変換する「リモートコマンド」タグ

モバイルアプリにFirebaseリモートコマンドのモジュールを追加すれば、特定のベンダーに依存するコードをアプリに追記することなく、必要となるFirebaseのライブラリを自動的に追加してビルドします。依存関係マネージャを利用してインストールすれば、Fireabase SDKを個別にインストールする必要はありません。

リモートコマンドには「JSON設定ファイル」と「iQタグマネージメントによるマッピング」の２つのオプションがあります。我々が推奨するのはJSON設定ファイルでの連携で、これはリモートもしくはローカルに連携したいベンダー別に設定ファイルを保存しておくという方法です。iQタグマネージメントによる連携を希望する場合、連携したいベンダーのリモートコマンドタグを（タグマネージメントの管理画面で）追加します。詳細については ベンダー統合を参照願います。

インストール

依存関係マネージャの利用

JSON設定ファイルを利用していない場合、依存関係マネージャによるインストールを推奨します。

Tealium iOS（Objective-C）ライブラリを使用している場合は、手動インストール方法を使用してください。CocoaPodsおよびCarthageオプションは、Tealium iOS（Swift）ライブラリを使用している場合にのみ使用可能です。

TealiumFirebaseリモートコマンドをCocoaPodsかCarthageでインストールします。

    Swiftパッケージマネージャを使ってiOSにFirebaseリモートコマンドをインストールする方法:

    1. XcodeのプロジェクトでFile > Swift Packages > Add Package Dependencyを選択します。
    2. リポジトリのURLを入力します。 https://github.com/tealium/tealium-ios-firebase-remote-command.
    3. バージョンのルールを設定します。一般的にはUp to Next Major Versionが推奨されています。現在のTealiumFirebaseのバージョンが一覧に表示されない場合、Swiftパッケージマネージャのキャッシュをクリアします。
    4. TealiumFirebaseモジュールを選択してインストールします。そして、 Frameworks > Libraries & Embedded Contentで、Xcodeプロジェクトにあるアプリのターゲットを追加します。

    Tealium Swiftライブラリからモジュールを追加したい場合は、Swiftパッケージマネージャの手順に従います。

    Cocoapodを利用してFirebaseリモートコマンドをインストールする方法:

    1. Podfileに次の行があれば削除:

      pod "Firebase/Analytics"

    2. Podfileに次の依存を追加:

      pod "TealiumFirebase"

      TealiumFirebaseのpodには、次のTealiumSwiftパッケージへの依存が含まれます:

      'tealium-swift/Core'
'tealium-swift/RemoteCommands'
'tealium-swift/TagManagement'

    3. TealiumHelperファイルなどTealiumクラスやFireabaseリモートコマンドを利用するファイルに、TealiumSwiftとTealiumFirebaseモジュールをインポートします。

    Carthageを利用してFirebaseリモートコマンドをインストールする方法:

    1. Cartfileからtealium-swiftを削除します。tealium-swiftへの依存は、TealiumFirebaseに含まれています。

    2. Cartfileに次の行があれば削除:

      binary "https://dl.google.com/dl/firebase/ios/carthage/FirebaseAnalyticsBinary.json"

    3. Cartfileに下記の依存を追加:

      github "tealium/tealium-ios-firebase-remote-command"

    Carthageの--use-xcframeworksフラグを利用している場合、TealiumFirebaseモジュールをxcframeworkとしてビルドできません。Carthageの公式なXcode 12での対策が推奨されます。

    Mavenを利用してAndroidにFirebaseリモートコマンドをインストールする方法:

    1. Tealium for Android (Kotlin)もしくはTealium for Android (Java)を事前にインストールします.

    2. build.gradleファイルに下記の依存を追加:

      dependencies {
implementation 'com.tealium.remotecommands:firebase:2.0.0'
}

    マニュアルでのインストール (iOS)

    マニュアルによるFirebaseリモートコマンドのインストールにはTealiumのSwiftライブラリが必要です。FireabaseリモートコマンドをiOSのプロジェクトに追加する方法は下記の通りです。

    1. Firebase SDKを事前にインストールしておきます。

    2. Tealium iOSのFirebaseリモートコマンドをレポジトリからクローンして、プロジェクトのSourcesフォルダにドラッグします。

    3. remoteAPIEnabled設定のフラグにtrueをセットします。

    初期化

    全てのライブラリで、初期化時にFirebaseモバイルリモートコマンドの登録が必要になります。

    Android (Kotlin)

    JSON設定ファイルもしくはリモートコマンドタグを使って、TealiumのKotlinライブラリでリモートコマンドを初期化します。

      下記のコードは、ローカルファイルを利用してJSONリモートコマンドを実行します:

      val config = TealiumConfig(application,
        "ACCOUNT",
        "PROFILE",
        Environment.DEV,
        dispatchers = mutableSetOf(Dispatchers.RemoteCommands));
var tealium = Tealium.create(TEALIUM_MAIN, config) {
    val firebase = FirebaseRemoteCommand(application);
  // register the command
  remoteCommands?.add(firebase, filename = "firebase.json");
}

          トラッキング

          アプリにおける標準的なインタラクションでトリガーされるイベントは自動的に計測されます。Fireabaseが自動で計測するイベントについてはリンクを参照願います。

          Firebase Mobile Remote Command Tag setupも合わせて確認します。

          Google アナリティクスとTealiumの同意管理

          Google アナリティクス for Firabase （GA4）は、アプリの利用状況やエンゲージメントに関する洞察を提供するアプリの計測サービスです。このサービスは、Firebaseの機能全体を統合し、FirebaseSDKで定義された最大500種類のイベントに関するレポートを無制限に提供します。

          FirebaseのAPIは一般に公開されていないため、Firebaseにデータを送信するにはAndroidもしくはiOSに対応したネイティブSDKを使います。TealiumのリモートコマンドはFirebaseのSDKを呼び出します。すでに実装されたデータレイヤーとトラッキングコールを利用すれば、アプリがAndroid StoreやApple App Storeにリリースした後であっても、Firebase SDKの挙動が調整可能です。

          詳細はGoogle Analytics for Firebase Mobile Integration Guideを参照願います。

          同意オプションの共有

          ネイティブの画面からWebViewに同意オプションを共有する方法は3種類あります。ここで説明するWebView連携は、まずページが汎用的な作りであること、そしてアプリからアクセスした場合に限定されます。

          その1: Cookieの利用
          Tealium iQと有効化された同意管理により、TealiumはCookieをモバイルWebViewに残します。クライアント端末は、ページを読み込む前に当該Cookieから同意オプションを参照します。

          その2: APIからWebviewの連携
          リスナーイベントを設定し、ユーザの同意設定変更を検知したら、APIを呼び出してWebViewのCookieを書き換えます。

          その3: クエリーストリングの利用
          WebViewページにTealium iQの同意管理を利用している場合、WebViewを呼び出すURLに同意設定をクエリーストリングに含めておきます。それにより、Tealium iQのプリローダースコープの機能拡張で設定情報を読み込み、同意管理のCookieを再生成できます。

          FAQ

          GoogleがFirebase SDKを更新した場合は？

          メジャーなAPIに変更がないという前提で、FireabaseのSDKを更新することに問題はありません。TealiumはFirebaseのAPI機能の更新を積極的に観察しており、Firabaseリモートコマンドに必要な更新を行っています。

          Firebase SDKは、初期状態でイベント計測を自動実行します。ユーザから同意を得る前の計測を停止するには？

          Firebase SDKにて、firebase_analytics_collection_enabledの設定をfalse (初期値はtrue)にセットします。ユーザが計測に同意しているなら、Firebaseリモートコマンドを使ってtrueにセットし直すことも可能です。

          下記のステップでユーザの操作例を解説します。

          1. ユーザがアプリを起動します。
          2. Firebaseでのデータ収集は停止しています。(setAnalyticsCollectionEnabled(false))
          3. ユーザが計測に同意すると、Tealiumに同意のイベントが送信されます。
          4. TealiumはFirebase SDKに計測開始のメッセージを送ります。(firebase_analytics_enabled = true)
          5. ユーザの操作がGoogle アナリティックスで計測されます。

          どの同意（カテゴリ）がアプリで承諾されたか確認する方法は？

          Tealium SDKにAPIリクエストを送信することで、ユーザが特定のカテゴリを承認したかどうかを確認できます。

          Tealiumでは同意情報を記録していません。アプリ内に保存されます。

          APIを使って特定のタグのみ無効にすることは可能か？

          いいえ、それはできないです。同意はカテゴリレベルでオプトイン・オプトアウトされます。
