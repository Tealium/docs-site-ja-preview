JAVASCRIPT

設定

utag.jsの動作を調整するために使用可能な設定について説明します。

仕組み

utag.jsの動作の多くは、utag.cfgオブジェクト内の設定によって制御されます。これらの設定によって制御されるデフォルトの動作は、utag_cfg_ovrdという名前の新しいオブジェクトを使用してオーバーライドされます。このオブジェクトを使用するには、utag.jsを読み込む前にページコードで設定するか、または[Pre Loader]をスコープとするJavaScriptコード拡張機能内で以下のように設定します。

window.utag_cfg_ovrd = window.utag_cfg_ovrd || {};

設定

次の表は、utag.jsの動作を調整するために使用できるパラメータ設定をまとめたものです。

Setting 説明
consentPeriod Consent Managementの保持期間
dom_complete DOM完了時（読み込み）イベントまでタグを遅延
domain Cookieドメインのオーバーライド
load_rules_ajax ページ読み込み後にタグ配信ルールを無効化（レガシー）
load_rules_at_wait 拡張機能後にタグ配信ルールを実行（レガシー）
lowermeta 小文字のメタタグデータ（レガシー）
lowerqp 小文字のクエリ文字列パラメータと値（レガシー）
noload utag.jsの無効化
noview 最初のトラッキングの無効化（単一ページアプリケーション）
nocookie Cookieのオプトアウト
readywait DOM対応までutag.jsを遅延
session_timeout セッションのタイムアウト
waittimer タイマによるタグの遅延

consentPeriod

Consent Managementの保持期間 デフォルトの90日間を任意の日数でオーバーライドします。（デフォルト：設定なし、ただし90と等価）

window.utag_cfg_ovrd.consentPeriod = 60;

dom_complete

DOM完了時（読み込み）イベントまでタグを遅延 DOM対応時ではなくドキュメント読み込みイベント時にタグをトリガーします。また、[DOM Ready]をスコープとするすべての拡張機能を抑制します。（デフォルト：false）

window.utag_cfg_ovrd.dom_complete = true;

domain

Cookieドメインのオーバーライド utag_main Cookieが設定されるドメインを設定します。Cookieを受け付けないルートドメイン上のサイトに対して役立ちます。たとえば、amazonaws.comなどです。（デフォルト値location.hostnameの最上位ドメイン）

window.utag_cfg_ovrd.domain = "mysite.amazonaws.com";

load_rules_ajax

ページ読み込み後にタグ配信ルールを無効化（レガシー） 同じページ読み込み内でutag.viewおよびutag.linkを呼び出すたびにタグ配信ルールを再処理するかどうかを制御します。新しくトリガーされたタグ配信ルールにより、ページに新しいタグが読み込まれる可能性があります。この設定は、4.2xより古いバージョンのutag.jsでの動作を再現する場合に使用します。（デフォルト：true）

window.utag_cfg_ovrd.load_rules_ajax = false;

load_rules_at_wait

拡張機能後にタグ配信ルールを実行（レガシー） 拡張機能後にタグ配信ルールを評価します。古いバージョンのutag.jsで、utag.jsの読み込み後にデータ層オブジェクトに値が入力されるようなインストール環境に対して使用されます。新しいバージョンのutag.jsでは、この設定の代わりに拡張機能の実行順序を利用できます。（デフォルト：false）

window.utag_cfg_ovrd.load_rules_at_wait = true;

lowermeta

小文字のメタタグデータ（レガシー） メタデータの変数と値をすべて小文字にします。たとえば、utag.data['meta.author']などです。この設定は、4.2xより前のバージョンのutag.jsでの動作を再現します。（デフォルト：false）

window.utag_cfg_ovrd.lowermeta = true;

lowerqp

小文字のクエリ文字列パラメータと値（レガシー） クエリ文字列の変数と値をすべて小文字にします。たとえば、utag.data['qp.refid']などです。この設定は、4.2xより前のバージョンのutag.jsでの動作を再現します。（デフォルト：false）

window.utag_cfg_ovrd.lowerqp = true;

noload

utag.jsの無効化 すべてのタグの読み込みを抑制します。[Pre Loader]拡張機能の後、utag.jsの実行が停止します。[DOM Ready]をスコープとする拡張機能は、引き続き実行されます。これは通常、[Prevent Tag Load]の[Publish Configuration]設定を使用して調整されます。（デフォルト：0）

window.utag_cfg_ovrd.noload = true;

noview

最初のトラッキングの無効化（単一ページアプリケーション） utag.jsによって呼び出される自動ページトラッキングイベントを抑制します。一般に、単一ページアプリケーションサイトで使用されます。（デフォルト：false）

window.utag_cfg_ovrd.noview = true;

nocookie

Cookieのオプトアウト このオプションは、サイトを訪問するユーザーが明示的にすべてのCookieをオプトアウトした場合のみ設定します。（デフォルト：false）

このオプションを使用すると、訪問者およびセッションの数が増加し、すべての訪問が単一ページセッションのように見えます。

このオプションを設定すると、データ値を保持拡張機能で設定されたCookieや訪問者/セッションCookieを含めて、_すべて_のCookieがutag.jsによって保存されなくなります。また、Cookieが使用できないとき、変数ut.visitor_id、tealium_visitor_id、cp.utag_main_v_idに新しいタイムスタンプが設定されます。

window.utag_cfg_ovrd.nocookie=true

readywait

DOM対応までutag.jsを遅延 ブラウザからDOM対応信号が受信されるまで、utag.jsのすべての動作を停止します。[Pre Loader]をスコープとする拡張機能は、utag.jsが読み込まれても引き続き実行されます。（デフォルト：0）

window.utag_cfg_ovrd.readywait = true;

session_timeout

セッションのタイムアウト 現在のセッションを有効期限切れにするまで待つ時間（ミリ秒単位）を設定します。これは通常、[Session Timeout]の[Publish Configuration]設定を使用して調整されます。（デフォルト値：1800000（30分））

// set timeout to 900000 milliseconds (15 minutes)
window.utag_cfg_ovrd.session_timeout = 900000;

waittimer

タイマによるタグの遅延 DOM対応イベント後にタグを読み込むまで待つ時間（ミリ秒単位）を設定します。（デフォルト：設定なし）

window.utag_cfg_ovrd.waittimer = 1000;
