GETTING STARTED

ユーザーの位置情報とジオフェンシング

モバイル向け位置追跡機能とジオフェンシング機能について説明します。

サポートされているプラットフォーム

位置追跡とジオフェンシングは以下のプラットフォームでサポートされています。

  • Android
  • iOS

位置追跡

位置追跡は、ユーザーのデバイスの地理的位置（緯度と経度）を特定する機能です。位置追跡を可能にするには、アプリにアクセス許可を与える必要があります。

位置情報は継続的に更新される可能性があります。これによって情報の精度は高くなりますが、バッテリーの消費量が多くなります。

継続的なトラッキングが無効になっている場合または使用不可の場合は、明示的なメソッド呼び出しを使用して、最後に認識されたユーザーの位置情報を取得できます。このアプローチはバッテリーの消費量が少なく抑えるものであり、リアルタイムの位置情報の監視を必要としないアプリで使用できます。

以下の変数は、位置追跡が使用される場合にデータ層に追加されます。これらの変数は各トラッキングコールに自動的に含められます。

変数名 型 説明 例
latitude String 最後に記録されたユーザーの位置の緯度 "32.9072"
longitude String 最後に記録されたユーザーの位置の経度 "-117.2367"
location_accuracy String 精度設定（"high"、"low"など） "high"

ジオフェンシング

ジオフェンシングは、ユーザーの位置の認識と_ジオフェンス_とのユーザーの近接性を組み合わせた位置追跡サービスです。ジオフェンスは、緯度、経度、および半径で指定される、目標物を囲む仮想境界です。

ジオフェンシングは、3つの遷移状態を使用して、ユーザーのジオフェンスとのやりとりを追跡します。これらの遷移状態は、Tealiumライブラリでトラッキングコールをトリガーします。次の遷移状態が監視されます。

Transition Tealiumイベント 説明
Geofence entered tealium_event="geofence_entered" ユーザーがジオフェンスに侵入しました
Geofence exited tealium_event="geofence_exited" ユーザーがジオフェンスを退出しました
Dwelled tealium_event="geofence_dwell"（Androidのみ） ユーザーが定義された期間、ジオフェンス内に滞留しています

ジオフェンシングの詳細は ジオフェンスファイルで定義され、初期化時にアプリに提供されます。

ジオフェンシングの遷移状態中は、次の変数が追跡されます。

変数名 型 説明 例
tealium_event String Tealiumのトラッキング対象ジオフェンスイベント："geofence_entered"、"geofence_exited"、または"geofence_dwell"（Androidのみ） "geofence_entered"
geofence_name String ジオフェンス地域の名前 "Tealium San Diego"
geofence_transition_type String ジオフェンス遷移イベントのタイプ "geofence_entered"、"geofence_exited"、または"geofence_dwell"（Android）
movement_speed (iOS only) String ユーザーの瞬間速度（メートル/秒で測定） "1.0"
location_timestamp (iOS only) String ユーザーがジオフェンス地域に進入/退出したときに記録された日付/時刻（GMT） "2020-01-28 16:29:46 +0000"

ファイル形式

アプリの初期化中にジオフェンスファイルが提供されると、ジオフェンシングが有効になります。ジオフェンスファイルには、目標物の座標と、JSON形式で追跡する遷移状態が含まれます。

ジオフェンスファイルを作成するときは、ファイルにgeofences.jsonという名前を付けます。ジオフェンスは、次の表に示す変数を使用して、キーと値のペアとして記述されます。

[{
  "name": "Geofence_Name",
  "latitude": 0.000,
  "longitude": 0.000,
  "radius": 100,
  "expire_after": -1,
  "trigger_on_enter": true,
  "trigger_on_exit": false,
  "minimum_dwell_time": 0 },
 {
    ...
 }]

各ジオフェンスは以下のプロパティで定義されます。

パラメータ 型 説明 例
name String 一意の文字列としてのジオフェンス名。アンダースコアで区切られた単語からなります。スペース、絵文字、UTF-8エンコードでサポートされていない文字は使用しないでください。 Geofence_Name
latitude Number ジオフェンスの中心の緯度座標。-90.0～90.0の10進値。 0.000
longitude Number ジオフェンスの中心の経度座標。-180.0～180.0の10進値。 0.000
radius Number ジオフェンスの半径値（メートル単位）。推奨される値：100以上 100
expire_after Number ジオフェンスの有効期間（ミリ秒単位）。（無期限にするには-1に設定します） -1
trigger_on_enter Boolean ユーザーがエリアに進入したときにジオフェンスがトラッキングイベントをトリガーするかどうかを定義します。 true
trigger_on_exit Boolean ユーザーがエリアから退出したときにジオフェンスがトラッキングイベントをトリガーするかどうかを定義します。 true
minimum_dwell_time Number ユーザーがジオフェンスに進入して滞留したときに、トラッキングイベントをトリガーする時間（ミリ秒単位）。ユーザーがジオフェンスの入退出を継続した場合にイベントが繰り返しトリガーされるのを防ぐため、大半のユースケースで推奨されています。（trigger_on_enterがtrueに設定されている場合は、0に設定します） 0
