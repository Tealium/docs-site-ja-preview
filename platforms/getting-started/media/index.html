メディア トラッキング

モバイル向けのメディア トラッキングについて説明します。

メディア トラッキングは、アプリ内のストリーミング メディア イベント (ビデオやオーディオ) をトラッキングする機能です。ストリーミング メディア イベントには、セッション、チャプター、広告や、プレイヤー イベント (再生、シーク、一時停止など) が含まれます。Media モジュールをインストールして、お使いの iOS や Android アプリにこうしたトラッキング機能を追加し、メディア トラッキング メソッドを実装します。

仕組み

Media モジュールは、ユーザーのストリーミング メディア エクスペリエンス全体のトラッキングをサポートします。これには、セッションの開始と終了、広告ブレークの開始と終了や、メディア プレイヤーとの各インタラクション (再生、一時停止、シークなど) が含まれます。

Media モジュールは、以下の階層の要素によりトラッキングを構成します。

メディア セッション メディア セッションは、他の要素のすべてについてのコンテナであり、それは、ストリーミング エクスペリエンスに関する一般情報を定義します。メディア セッションは、トラッキングのための必須要素です。

チャプター チャプターは、コンテンツ内でブレークがなされるときにセッションを分割する、オプションとしてのセグメントであり、オーディオブックの章や、映画のシーンなどがこれにあたります。

広告ブレーク 広告ブレークは、一または複数の広告についてのオプションとしてのコンテナです。広告ブレークは、通常、セッションの開始時、中間、終了時に置かれます。

広告 広告は、広告ブレーク内においてなされるオプションとしての要素です。

メディア セッションの構成の例：

└── Media Session
    ├── Ad Break 1
    │   ├── Ad 1
    │   └── Ad 2
    ├── Chapter 1
    ├── Ad Break 2
    │   ├── Ad 3
    │   └── Ad 4
    └── Chapter 2

インストール

まず、サポートされているプラットフォームのうちの一つについて Media モジュールをインストールします。

  • Android (Kotlin)
  • iOS (Swift v2.x)

初期化

Media モジュールをインストールした後、 MediaContent オブジェクトを初期化します:

    val mediaContent: MediaContent = MediaContent(
  name = "What is the Tealium Customer Data Hub?",
  trackingType = TrackingType.MILESTONE,
  streamType = StreamType.DVOD,
  mediaType = MediaType.VIDEO,
  qoe = QoE(1500),
  duration = 130
)

    let media = MediaContent(
  name: "What is the Tealium Customer Data Hub?",
  trackingType: .milestone,
  streamType: .vod,
  mediaType: .video,
  qoe: QoE(bitrate: 1500),
  duration: 130
)

    TrackingType

    トラッキング タイプは、Tealium イベントとしてトラッキングするメディア イベントがどれかを定めます。(トラッキング タイプの概要を参照。)

    次の定数のいずれかを用いて、 TrackingTypeを設定します。

      Type プロパティ 説明
      完全プレイバック FULL_PLAYBACK インターバル イベントを除き、すべてのメディア イベントをトラッキングします。
      - インターバル INTERVAL 標準的なメディア イベントをトラッキングし、10 秒ごとにピンを実行して、コンテンツが再生されているかどうかをチェックします。
      マイルストーン MILESTONE 標準的なメディア イベントをトラッキングし、コンテンツの再生された割合 (%) を測定します。
      インターバル マイルストーン INTERVAL_MILESTONE インターバル イベントとマイルストーン イベントの双方をトラッキングします。
      サマリー SUMMARY セッション終了時に単一のサマリー (再生されたコンテンツや広告の時間の計算など) をデータレイヤーに送信します。
      Type プロパティ 説明
      完全プレイバック fullPlayback インターバル イベントを除き、すべてのメディア イベントをトラッキングします。
      - インターバル interval 標準的なメディア イベントをトラッキングし、10 秒ごとにピンを実行して、コンテンツが再生されているかどうかをチェックします。
      マイルストーン milestone 標準的なメディア イベントをトラッキングし、コンテンツの再生された割合 (%) を測定します。
      インターバル マイルストーン intervalMilestone インターバル イベントとマイルストーン イベントの双方をトラッキングします。
      サマリー summary セッション終了時に単一のサマリー (再生されたコンテンツや広告の時間の計算など) をデータレイヤーに送信します。

        値 説明

        AUDIOBOOK オーディオ ブックのストリーム タイプ。
        CUSTOM カスタム ストリーム タイプ。
        DVOD オンデマンド デジタル ビデオのストリーム タイプ。
        LINEAR リニア (1 次元) コンテンツのストリーム タイプ。
        LIVE ライブ コンテンツのストリーム タイプ。
        PODCAST ポッドキャストのストリーム タイプ。
        SONG 曲のストリーム タイプ。
        UGC ユーザー生成コンテンツのストリーム タイプ。
        VOD オンデマンド ビデオのストリーム タイプ。
        値 説明
        aod オンデマンド オーディオのストリーム タイプ。
        audiobook オーディオ ブックのストリーム タイプ。
        custom カスタム ストリーム タイプ。
        dvod オンデマンド デジタル ビデオのストリーム タイプ。
        linear リニア (1 次元) コンテンツのストリーム タイプ。
        live ライブ コンテンツのストリーム タイプ。
        podcast ポッドキャストのストリーム タイプ。
        song 曲のストリーム タイプ。
        ugc ユーザー生成コンテンツのストリーム タイプ。
        vod オンデマンド ビデオのストリーム タイプ。

        MediaType

        MediaType 定数は、以下のとおりです。

          Property 説明
          ALL オーディオとビデオを含む、すべてのメディア タイプ。
          AUDIO オーディオ ストリーム向けのメディア タイプ。
          VIDEO ビデオ ストリーム向けのメディア タイプ。
          Property 説明
          all オーディオとビデオを含む、すべてのメディア タイプ。
          audio オーディオ ストリーム向けのメディア タイプ。
          video オーディオ ストリーム向けのメディア タイプ。

          State

          プレイヤー状態トラッキングは、プレイバック中のビデオ プレイヤーのモード (全画面モード、字幕、音声のミュートなど) をキャプチャします。プレイヤー状態は、データレイヤー変数 media_player_state に設定されます。

          プレイヤー状態の変動のトラッキングの例:

            開始値を設定するには、メソッド updatePlayerState()を呼び出します。

            tealium?.media?.updatePlayerState(STATE_CONSTANT_VALUE)

            開始値を設定するには、 playerState 変数を設定します。

            mediaSession.playerState = STATE_CONSTANT_VALUE

            次の定数のいずれかを用いて、 Stateを設定します。

              値 説明
              CLOSED_CAPTION 字幕状態のタイプ。
              FULLSCREEN 全画面状態のタイプ。
              IN_FOCUS イン フォーカス状態のタイプ。
              MUTE ミュート状態のタイプ。
              PICTURE_IN_PICTURE ピクチャー イン ピクチャー状態のタイプ。
              値 説明
              closedCaption 字幕状態のタイプ。
              fullscreen 全画面状態のタイプ。
              inFocus イン フォーカス状態のタイプ。
              mute ミュート状態のタイプ。
              pictureInPicture ピクチャー イン ピクチャー状態のタイプ。

              Quality of Experience (QoE)

              メディア セッションの Quality of Experience (QoE) 。キロバイト/秒 (kpps) 単位のビットレートに設定されます。メディア プレイバック中に現在のビットレート値が入手できる場合は、 qoe 変数として、 MediaContent オブジェクトの初期化時にそれを提供します。Quality of Experience (QoE) を設定すると、 media_bitrate_change tealium イベントがトリガーされます。

              Quality of Experience (QoE) のトラッキングの例:

                Quality of Experience (QoE) ビットレート値を設定するには、 updateQOE() メソッドを呼び出します。

                tealium?.media?.updateQOE(BITRATE_VALUE)

                Quality of Experience (QoE) ビットレート値を設定するには、 bitrate 変数を設定します。

                mediaSession.bitrate = BITRATE_VALUE

                メタデータ

                メタデータは、オブジェクトに関する追加の情報を設定するパラメータです。メタデータをオブジェクトに送信すると、こうしたキーと値のペアもデータレイヤーに設定されます。メタデータを受け付けるオブジェクトは、以下のとおりです: MediaContent、 Chapter、 QoE。

                メタデータが複数のオブジェクト内で設定可能であることから、同一のメタデータ キーを複数回設定することができます。データレイヤー内でのメタデータ キーの抵触を解決するには、複数のオブジェクトからのメタデータを、オブジェクト階層 ( MediaContent > Chapter > QoE) を用いて、データレイヤーにマージします。

                たとえば、 feed_type を MediaContent と Chapterの双方に設定する場合は、 Chapter メタデータからの値は、最終データレイヤーにおいて MediaContent メタデータからの値をオーバーライドします。

                メタデータの設定の例:

                  val mediaContent: MediaContent = MediaContent(
    // ...
    metadata = mutableMapOf("episode" to "2", "network" to "YouTube")
)

                  let media = MediaContent(
    // ...
    metadata: ["episode": "2", "network": "YouTube"]
)

                  ビデオ メタデータ キー

                  以下は、一般的なビデオ メタデータ キーの例です。

                  変数 説明
                  authorized ユーザーは、認証されるか、または認証されません。
                  episode エピソードの名前または番号。
                  asset_id TMS ID。これは、1 つのビデオ コンテンツについての業界標準 ID です。
                  daypart 時間帯 (午前、プライムタイムなど)。
                  feed_type フィード タイプ (HD、SD、リージョナルなど)。
                  first_air_date メディアが最初に配信された日付。
                  first_digital_date メディアが最初にデジタルで利用可能になった日付。
                  genre ジャンル。
                  media_mvpd マルチチャンネル ビデオ プログラミング ディストリビュータ。通常は、ケーブル ネットワークや衛星ネットワーク (Cox、DirecTV、Sky など)。
                  network ネットワーク。
                  originator ショー オリジネーター。
                  rating レーティング。
                  season シーズンの番号または名前。
                  show ショーの名前。
                  show_type ショーのタイプ。
                  stream_format ストリームの形式: 0 は、部分的なコンテンツであり、 1 は、完全なエピソードです。

                  オーディオ メタデータ キー

                  以下は、一般的なオーディオ メタデータ キーの例です。

                  変数 説明
                  album アルバム名。
                  artist アーティスト名。

                  publisher 音楽出版社。
                  record_label レコード レベル名。
                  station ラジオ局。

                  トラッキング タイプの概要

                  トラッキング タイプは、トリガーされる Tealium イベントがどれであるかを定めます。トリガーするイベントのレベルに基づき、トラッキング タイプを選択します。

                  たとえば、全トラッキング メソッドを実装して、サマリー トラッキングを用いる場合は、 media_session_end イベントのみがトリガーされます。

                  完全プレイバック イベント

                  完全プレイバック トラッキングを用いるには、インターバル イベントを除くすべてのメディア イベントをトラッキングします。

                  トラッキングされるイベントには、以下が含まれます:

                  • セッション イベント:
                    • media_session_start
                    • media_session_resume
                    • media_session_end
                  • 広告ブレーク イベント:
                    • media_adbreak_start
                    • media_adbreak_end
                  • 広告イベント:
                    • media_ad_start
                    • media_ad_end
                    • media_ad_click
                    • media_ad_skip
                  • チャプター イベント:
                    • media_chapter_start
                    • media_chapter_skip
                    • media_chapter_end
                  • バッファ イベント:
                    • media_buffer_start
                    • media_buffer_end
                  • プレイヤー イベント:
                    • media_play
                    • media_pause
                    • media_seek_start
                    • media_seek_end
                  • その他のイベント
                    • media_bitrate_change

                  インターバル イベント

                  インターバル トラッキングを用いて、繰り返しインターバル時にメディア プレイヤーのステータスをトラッキングします。このタイプを用いるときは、10 秒ごとに ping() メソッドが呼び出され、コンテンツが再生中であるかどうかをチェックし、データレイヤーで tealium_event を media_interval に設定します。

                  トラッキングされるイベントには、以下が含まれます:

                  • セッション イベント:
                    • media_session_start
                    • media_session_resume
                    • media_session_end
                  • プレイヤー イベント:
                    • media_play
                    • media_pause
                  • インターバル イベント:
                    • media_interval

                  マイルストーン イベント

                  コンテンツの一定割合 (%) が再生済みであるときにマイルストーン トラッキングを使用します。各マイルストーン パーセントに達したときに、 sendMilestone() メソッドが呼び出され、データレイヤーで tealium_event を media_milestone に設定します。

                  トラッキングされるマイルストーン パーセントは、以下のとおりです。

                  Media % Played media_milestoneの値
                  8.0 - 12.0 10%
                  23.0 - 27.0 25%
                  48.0 - 52.0 50%
                  73.0 - 77.0 75%
                  88.0 - 92.0 90%
                  97.0 - 100.0 100%

                  トラッキングされるイベントには、以下が含まれます:

                  • マイルストーン イベント:
                    • media_milesetone
                  • セッション イベント:
                    • media_session_start
                    • media_session_resume
                    • media_session_end
                  • プレイヤー イベント:
                    • media_play
                    • media_pause
                    • media_seek_start
                    • media_seek_end
                  • 広告ブレーク イベント:
                    • media_adbreak_start
                    • media_adbreak_end

                  インターバル マイルストーン イベント

                  インターバル マイルストーン イベントを用いて、 インターバル イベントと マイルストーン イベントの両方をトラッキングします。

                  サマリー イベント

                  サマリー トラッキングを用いて、セッション終了時に単一のサマリー イベントを送信します。セッション サマリー イベントには、再生されたコンテンツや広告の時間の計算が含まれます。

                  サマリー イベントには、以下が含まれます:

                  • セッション イベント:
                    • media_session_start
                    • media_session_end

                  トラッキングされる メディア サマリー変数 のすべての完全なリストを参照してください。

                  トラッキング

                  MediaContent オブジェクトを初期化した後、以下のメソッドを用いて、メディア インタラクションをトラッキングします。

                  メディア セッション

                  メディア セッションは、チャプター、広告ブレーク、広告、メディア プレイヤー イベントを含むメディア コンテンツの全体です。

                  メディア セッションのトラッキングを開始するには、 startSession() メソッドを MediaContent を使用して呼び出します。メディア セッションのトラッキングを停止するには、 endSession() メソッドを呼び出します。

                  セッションの開始は、 play() イベントを暗示しません。

                  例:

                    tealium?.media?.startSession(mediaContentObject)
//...
tealium?.media?.endSession()

                    let mediaSessionObject = tealium?.media?.createSession(
      from: mediaContentObject)                                
mediaSessionObject.startSession()
// ...
mediaSessionObject.end()

                        チャプター

                        チャプターは、コンテンツ内のブレーク (オーディオブックの章や、映画のシーンなど) を表す、メディア セッション内のオプションとしてのセグメントです。

                        メディア セッションでチャプター セグメントをトラッキングするには、 Chapter オブジェクトを作成して、これを startChapter() メソッドに渡します。チャプターのトラッキングを停止するには、 endChapter() メソッドを呼び出します。

                        チャプターのトラッキングの例:

                          val chapterObject = Chapter(
  name = "What is the Tealium Customer Data Hub?",
  duration = 114
)
tealium?.media?.startChapter(chapterObject)
// ...
tealium?.media?.endChapter()

                          let chapterObject = Chapter(
  name: "What is the Tealium Customer Data Hub?",
  duration: 114
)
mediaSession.startChapter(chapterObject)
// ...
mediaSession.endChapter()
