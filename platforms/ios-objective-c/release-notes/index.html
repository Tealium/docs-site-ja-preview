リリースノート

リリースノート

以下は、iOS ライブラリのリリースごとの変更点をまとめたリストです。

5.8.3 (2022 年 2 月)

影響の小さい変更点

  • セルフホスト JavaScript ファイルを用いている Tealium iQ アカウント用のセッション カウントを無効にできるように sessionCountingEnabled フラッグを TEALConfiguration に追加。
  • デバイスに言語設定ができないクラッシュの可能性を修正。

5.8.2 (2021 年 8 月)

影響の小さい変更点

  • 添付アプリの .ipa ファイルのアーカイブの完了を妨げる Lifecycle XCFramework ファイルの問題を修正
  • Carthage は、中間の JSON ファイルを必要とすることなく Github repo から直接 XCFrameworks を使用できるようになりました (インストールを参照)。既存の Cartfile リファレンスがあれば、その更新が必要です。
  • CocoaPods では、XCFramework ファイルを使用するようになりました。

5.8.1 (2021 年 6 月)

影響の大きい変更点

  • Cookie の同期問題を回避するために用いるためのに WKProcessPool インスタンス (Tealium Tag Management ウェブビュー用 TealiumConfig オブジェクト上のもの) を設定するオプションを追加。

5.8.0 (2021 年 2 月)

影響が中程度の変更点

  • platform 変数の値を小文字に変更し、モバイル セッション タイマーを訂正。以前は、ウェブと同様にセッションが測定されていました (一貫性したアクティビティ 30分)。platform 変数を小文字に変更することで、Customer Data Hub は、受信データをモバイル セッションとして認識するようになり、セッション所要時間が 2 分に短縮されました。

大文字と小文字を区別する platform 値 (iOS と ios) のチェックのために読み込みルールまたは属性ルールが構成されている場合は、読み込みルールを更新します。

影響の小さい変更点

  • ビルドの警告項目を解決し、非推奨の API を更新

5.7.0 (2020 年 8 月)

影響の大きい変更点

  • utag.js 内のセッション カウント ロジックを使わずに、ネイティブ セッション カウントに対するサポートを追加。これにより、セッション カウントの正確性が増し、一部のエッジ ケースが修正されました。こうしたエッジ ケースでは、バックグラウンド アクティビティが新たなセッションをトリガーして、セッション カウントが想定よりも高くなっていました。

⚠️ 重要事項 ⚠️ タグ管理 (Tag Management) がモバイル公開設置で有効化された場合は、関連するアカウント内で mobile.html テンプレートも更新する必要があります。このアクションが実施されない場合は、Tealium 内に重複セッションが記録されます。テンプレートを更新する方法について 詳しく読む。

5.6.6 (2020 年 1 月)

影響の小さい変更点

  • バグ修正。removeRemoteCommandID上のログ形式を更新。

5.6.5（2019年11月）

影響の小さい変更点

  • 揮発性データ変数を簡単に取得して使用できるよう、tealium_session_idを追加。

5.6.4（2019年10月）

影響の小さい変更点

  • "iPhone10,4"など、モデルのApple内部識別子を含む追加のdevice_type変数を追加。

5.6.3（2019年10月）

影響の小さい変更点

  • バグ修正。TEALDispatchService配列の反復処理中にディスパッチサービスを安全に削除できるようにdestroyInstanceForKeyメソッドを更新。

5.6.2（2019年10月）

影響の小さい変更点

  • device、model_name、model_variant変数を作成するためのプル元のデバイスとモデルを、最新のiPad第7世代モデルを含むように更新。
  • デバイスリストで一致項目が見つからない場合に基本モデル名（iPhone 10、4など）の使用にフォールバックするロジックをいくつか追加。

5.6.1（2019年10月）

影響の小さい変更点

  • Carthageのインストールが失敗する原因となっていたDevicesOnlyフレームワークフォルダを削除。#99

5.6.0（2019年10月）

影響の大きい変更点

  • AppleがiOS 8.0以降でUIWebViewを非推奨にしたため、UIWebViewを完全に削除してWKWebViewのみをサポートするようにしました。利便性を損なわないよう、また訪問者のセッションおよび分析履歴がそのまま残されるよう、UIWebViewのサポートは継続しました。ただし、ITMS-90809（非推奨APIの使用）によると、AppleはUIWebView APIを使用するアプリの申請の受け入れを停止します。UIWebViewの詳細については、こちらを参照してください。UIWebViewで使用していたCookieをWKWebViewに移行するWKHTTPCookieStoreが用意されています。これにより、ユーザーID/訪問者Cookieを新しい実装にシームレスに移行できます。

影響が中程度の変更点

  • 最新のAppleデバイス（X、XR、XS、11など）を含むようにデバイスリストを更新。また、デバイス名をハードコーディングする代わりにJSONファイルから読み込むように実装を更新しました。さらに、デバイスの詳細を指定する、model_name（iPhone 8、iPhone XR Max、iPhone 11 Proなど）とmodel_variant（"GSM"、"WiFi"、"model A1457, A1518, A1528"など）という2つの変数を追加しました。追加のモデル情報がない場合、標準のdevice変数は"iPhone 10,0"のようになります。

5.5.4～5.5.5（2019年8月）

影響の小さい変更点

  • ビュー階層が変化したときにTealiumの再初期化が必要にならないように、トラックコールのたびにビューへの再接続を行う機能が追加されました。独自のアプリでの検証が不可欠です。トラッキングコールの実行後に、プッシュ通知から起動されるビューなど、ビュー階層が変更されている可能性のあるアプリ内のさまざまな画面が表示されることを確認してください。

5.5.3（2019年7月）

影響の大きい変更点

  • WKWebView向けのUIView自動検出ロジックが改善されました。Webview構成内のUIViewインスタンスを手動で渡す必要がないので、複雑なビュー階層を伴うほとんどのケースで改善効果が得られます。独自のアプリでの検証が不可欠です。トラッキングコールの実行後に、プッシュ通知から起動されるビューなど、ビュー階層が変更されている可能性のあるアプリ内のさまざまな画面が表示されることを確認してください。

5.5.2（2019年7月）

影響の小さい変更点

  • Webviewへのコールごとに行っていたトレースID Cookieのチェックがクラッシュを引き起こしている可能性があったので、このチェックは廃止されました。UIWebViewではうまく機能していましたが、WKWebViewで問題を引き起こしていたようです。その他のTealiumインストール検証およびデバッグ手法については、Tealiumモバイルアプリのデバッグを参照してください。

5.5.1（2019年12月）

影響の小さい変更点

  • clientVolatileDataSourcesではなくvolatileDatasourcesCopyを直接使用するようcaptureTimeDataSourcesForTypeメソッドを修正。これにより、addToVolatileDataSourcesメソッドを別のスレッドから安全にコールできるようになりました。

5.5.0（2019年4月）

影響の大きい変更点

  • タグ管理でUIWebViewではなくWKWebViewがデフォルトで使用されるようになりました。アップグレードの前に、タグ管理に関するドキュメントをよくお読みください。

影響の小さい変更点

  • WKWebViewの更新および変更内容を紹介する新しいサンプルアプリが追加されました

4.xから5.xへの移行

5.xで追加された機能

  • Collect、Compact、Fullの各ライブラリを1つのライブラリに統合
  • リモートで構成可能なディスパッチサービス
  • マルチインスタンスをサポート
  • ディスパッチを監視または編集するためのデリゲート 

削除された4.xの機能

  • ライフサイクルトラッキング
  • UIイベントの自動トラッキング
  • Mobile Companion 

個別に追加される単独のモジュールとして利用可能です。

データレイヤーの相違点

Version 5.xでは現在、4.xライブラリから利用可能なデータレイヤー変数のサブセットを使用しています。ただし、以前の変数はすべて、今後5.xのオプションのクラスや更新で使用できるようデータレイヤーで提供されます。 

APIの相違点

バージョン5.xでは、バージョン4.xと区別するために、まったく異なる一連のAPIを使用しています。 2つの主要な相違点は次のとおりです。

  • シングルトンインスタンスのコールに代わり、インスタンスキー（任意の文字列）を必要とするMultitonインスタンスが使用されています。SDKは、このキーを使用してライブラリの特定のインスタンスを参照します。
  • 単一のトラックコールに代わり、イベントコールおよびビュー変更コールが使用されています

Objective-Cの例

// version 4.x & prior
[Tealium trackCallType:TealiumEventCall
    customData:@{@"testKey":@"testValue"}
    object:nil];

// version 5.x
[Tealium instanceForKey:@"testInstanceID"] trackEventWithTitle:@"buttonClick"
    dataSources:@{@"testKey":@"testValue"}];
[Tealium instanceForKey:@"testInstanceID"] trackViewWithTitle:@"mainView"
    dataSources:@{@"testKey":@"testValue"}];
// Note:  An event or view title is required, but dataSources are still optional

さらに、iOS 9（2015年9月）の時点でuseHttpsオプションは不要になっています。AppleはすべてのURL通信について、TLS 1.2以上がサポートされたHTTPSを必須としています（アプリの開発者によって明示的に上書きされていない場合）。

Appleの公式ドキュメント（「アプリのトランスポートセキュリティ」セクション）を参照してください。

Tealium iQ Tag Managementのセットアップの相違点

バージョン5.xでは、新しいリモート構成オプションが利用できます。以前アカウントで4.xを使用していた場合は、以下を実施します。

  1. Tealium iQで、[Save/Publish]をクリックし、[Configure Publish Settings…]をクリックします

  2. [Mobile Library Publishing]をクリックします

  3. [+ Add Version]をクリックして、[Mobile SDK 5.0]を選択します

  4. [Add]、[Save]の順にクリックします

  5. 公開の詳細を入力し、更新された設定を公開します

Tag Managementはデフォルトでは_無効_になっています。ネイティブのモバイルアプリケーションからタグを実行するには、Tag Managementを有効にします。Tag Managementが有効な場合は、重複したリクエストを送信しないようにCollectをオフにします。CollectタグはTealium iQで追加できます。
