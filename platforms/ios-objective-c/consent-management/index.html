同意管理

iOS（Objective-C）向け同意管理の実装について説明します。

使用方法

このモジュールはCocoaPodsフレームワークビルドに自動的に含まれて有効になっているため、使用することが推奨されます。

サポートされるプラットフォームには、iOS、macOS、tvOS、watchOSが含まれます。

同意管理の詳細については、こちらを参照してください。

サンプルアプリ

Tealium Consent Managementを有効にする方法に精通していただけるよう、以下のサンプルアプリをダウンロードすることをお勧めします。

  • Objective-Cサンプルアプリ
  • Swiftサンプルアプリ

データレイヤー

次の変数は、同意管理が有効な場合に、各トラッキングコールのたびに送信されます。

変数名 説明 例
policy 同意を得る際に従ったポリシー（デフォルト：gdpr） gdpr
consent_status ユーザーの現在の同意ステータス ["Consented", "Not Consented"]
consent_categories ユーザーが現在同意しているカテゴリ @[@"analytics"]

使用事例

使用事例：簡単なオプトイン

[_tealium.consentManager setUserConsentStatus:Consented];

この例では、ユーザーに簡単な”オプトイン/オプトアウト”オプションを提供する方法を示しています。ユーザーが同意する場合、そのユーザーは自動的にすべてのトラッキングカテゴリにオプトインします。ユーザーが同意を取り消す場合、どのカテゴリもアクティブにならず、トラッキングコールは送信されません。

Tealiumのヘルパークラスで次のメソッドを定義し、アプリユーザーがトラッキングに同意したかトラッキングを拒否した場合に呼び出します。ユーザーがトラッキングに同意する場合、Consent Managerによって自動的にすべてのトラッキングカテゴリにオプトインします。

 

使用事例：グループオプトイン

[_tealium.consentManager setUserConsentStatus:Consented withUserConsentCategories:@[@"analytics", @"monitoring", @"big_data", @"mobile", @"crm"]];

この例では、カテゴリベースの同意モデルを示しています。ここでは、トラッキングカテゴリは、開発者が定義した少数の上位カテゴリにグループ化されます。たとえば、Tealiumの同意カテゴリ"analytics"、"monitoring"、"big_data"、"mobile"、"crm"を"performance"という1つのカテゴリにグループ化することができます。このようにすると、ユーザーにとって、カテゴリの包括リストから選択するよりも簡単になる可能性があります。これを、最小限の許可から最大限の許可まで（すべてのカテゴリ）のスライダーインターフェイスで表すこともできます。

 

使用事例：カテゴリベースのオプトイン

[_tealium.consentManager setUserConsentStatus:Consented withUserConsentCategories:@[@"analytics", @"personalization"]];

この例では、カテゴリベースの同意モデルを示しています。ここでは、ユーザーはカテゴリの包括的なリストから、明示的に各カテゴリを選択する必要があります。Consent Managerのデフォルト状態は"Unknown"です。これは、ユーザーが同意を提示するまで、ヒットをキューに追加します。ユーザーがなんらかのカテゴリに同意した場合、イベントはキューから取り出され、同意したカテゴリデータが各トラッキングコールに追加されます。

 

APIリファレンス

consentManager

このコードは、Tealiumの初期化時にConsent Managementを有効にする方法を示しています。

// TealiumHelper.h

@import TealiumIOS;
@interface TealiumHelper : NSObject<TealiumDelegate, TEALConsentManagerDelegate>
@property (nonatomic, strong) Tealium *tealium;
@end

// TealiumHelper.m

@implementation TealiumHelper

+ (instancetype _Nonnull)sharedInstance {
  static TealiumHelper *tealiumHelper = nil;
  static dispatch_once_t onceToken;
  dispatch_once(&onceToken, ^{
  tealiumHelper = [[self alloc] init];
  });

  return tealiumHelper;
}

- (instancetype)init {
  if (self = [super init]) {
    TEALConfiguration *configuration = [TEALConfiguration configurationWithAccount:@"ACCOUNT" profile:@"PROFILE" environment:@"ENVIRONMENT"];
    [configuration setLogLevel: TEALLogLevelDev];

    // enable Tealium Consent Manager
    [configuration setEnableConsentManager:YES];

    _tealium = [Tealium newInstanceForKey:@"INSTANCE" configuration:configuration];
    _tealium.delegate = self;

    // set optional Tealium Consent Manager Delegate
    _tealium.consentManagerDelegate = self;
    NSLog(@"%s, %@", __FUNCTION__, [TEALConsentManager consentStatusString:_tealium.consentManager.userConsentStatus]);
  }

  return self;
}

@end

enable
Consent Managerを有効にします。このメソッドを使用すると、Tealiumの初期化時ではなく、後でConsent Managerが有効になります。

[_tealium.consentManager enable];

お勧めのアプローチは、Tealiumの初期化時にConsent Managerを有効にすることです。 

isEnabled
Consent Managerが有効かどうかを確認します。

[_tealium.consentManager isEnabled];

disable
Consent Managerを無効にします。このメソッドを使用すると、Tealiumが完全に初期化された後でConsent Managerが無効になります。

[_tealium.consentManager disable];

isDisabled
Consent Managerが無効かどうかを確認します。

[_tealium.consentManager isDisabled];

consentManagerDelegate
TEALConsentManagerDelegateプロトコルに準拠する特定のクラスを登録します。

[_tealium setConsentManagerDelegate:(id<TEALConsentManagerDelegate>)];

setUserConsentStatus
ユーザーの同意ステータスを設定します。ユーザーがトラッキングを有効または無効にしたときに、アプリの同意管理設定画面から呼び出されるように設計されています。

[_tealium.consentManager setUserConsentStatus:Consented];

Consent Managerでは、ステータスがConsentedの場合は常に、使用可能なすべての同意カテゴリを含むように同意カテゴリのリストを設定します。このメソッドではカテゴリを選択的に設定することはできません。 

パラメータ 説明 例
TEALConsentStatus TEALConsentStatus列挙型の値 [Consented, NotConsented, Disabled]

setUserConsentCategories
ユーザーの同意カテゴリを設定します。アプリの同意管理設定画面から呼び出されるように設計されています。

[_tealium.consentManager setUserConsentCategories:@[@"analytics"]];

任意の数のカテゴリが設定されている場合に必ず同意ステータスをConsentedに設定します。

パラメータ 説明 例
NSArray 事前定義されたNSStringカテゴリの配列 @[@"analytics", @"big_data"]

setUserConsentStatus:withUserConsentCategories
1つのメソッドでユーザーの同意ステータスとユーザーの同意カテゴリを設定します。

[_tealium.consentManager setUserConsentStatus:Consented withUserConsentCategories:@[@"analytics"]];
パラメータ 説明 値の例
TEALConsentStatus TEALConsentStatus列挙型の値 [Consented, NotConsented]
NSArray 事前定義されたNSStringカテゴリの配列 @[@"analytics", @"big_data"]

userConsentStatus
ユーザーの現在の同意ステータスを返します。

[_tealium.consentManager userConsentStatus];

userConsentCategories
ユーザーの現在の同意カテゴリを返します。

[_tealium.consentManager userConsentCategories];

isConsented
ユーザーが同意しているかどうかを判断する便利なメソッド。

[_tealium.consentManager isConsented];

consentStatusString
人間が判読できるTEALConsentStatus列挙型の文字列の便利なメソッド。

[TEALConsentManager consentStatusString:_tealium.consentManager.userConsentStatus];

resetUserConsentPreferences
現在保存されている同意設定をすべてリセットします。カテゴリが何も設定されていないUnknown同意ステータスに戻します。

[_tealium.consentManager resetUserConsentPreferences];

setConsentLoggingEnabled
同意の記録機能を有効にします。この機能では、監査目的で同意ステータスの変化をすべてTealium Customer Data Hubに送信します。

[_tealium.consentManager setConsentLoggingEnabled:YES];
パラメータ 説明 値の例
TEALConsentLoggingEnabled 同意の記録を有効または無効にします。 [YES, NO]

isConsentLoggingEnabled
同意の記録機能の現在の状態を返します。

[_tealium.consentManager isConsentLoggingEnabled];

allCategories
サポートされている同意カテゴリの包括的なリストを返します。

[_tealium.consentManager allCategories];

TEALConfiguration

enableConsentManager
ライブラリが初めて起動したときに、ユーザーの初期同意ステータスを設定します。保存されている設定がある場合は、その設定でconfigオブジェクトに渡された設定がオーバーライドされます。

self.configuration.enableConsentManager = YES;

Consent Managerでは、ステータスがConsentedの場合は常に、使用可能なすべての同意カテゴリを含むように同意カテゴリのリストを設定します。このメソッドではカテゴリを選択的に設定することはできません。

userConsentStatus
ライブラリが初めて起動したときに、ユーザーの初期同意ステータスを設定します。保存されている設定がある場合は、その設定でconfigオブジェクトに渡された設定がオーバーライドされます。

self.configuration.userConsentStatus = Consented;
パラメータ 説明 値の例
TEALConsentStatus TEALConsentStatus列挙型の値 [Consented, NotConsented, Disabled]

userConsentCategories
ライブラリが初めて起動したときに、ユーザーの初期同意カテゴリを設定します。保存されている設定がある場合は、その設定でconfigオブジェクトに渡された設定がオーバーライドされます。

self.configuration.userConsentCategories = @[@"analytics", @"retargeting"];
パラメータ 説明 値の例
NSArray 事前定義されたNSStringカテゴリの配列 @[@"analytics", @"big_data"]

TEALConsentManagerDelegate

consentManagerDidChangeWithState
このメソッドは、Consent Managerのユーザーの同意ステータスが変更されるたびに呼び出されます。たとえば、ユーザーがトラッキングに同意したか、拒否したときです。

- (void)consentManagerDidChangeWithState:(TEALConsentStatus)consentStatus;

consentManagerDidUpdateConsentCategories
このメソッドは、Consent Managerのユーザーの同意カテゴリが更新されるたびに呼び出されます。

-(void)consentManagerDidUpdateConsentCategories:(NSArray * \_Nullable)categories;

consentManagerWillDropDispatch
このメソッドは、Consent Managerがトラッキングコールを中断するときに呼び出されます。たとえば、ユーザーがトラッキングを拒否したときです。

- (void)consentManagerWillDropDispatch:(TEALDispatch * \_Nullable)dispatch;

consentManagerWillQueueDispatch
このメソッドは、Consent ManagerがUnknown状態のときに呼び出されます。ユーザーが同意するか、同意を拒否するまで、トラッキングコールはローカルでキューに追加されます。

- (void)consentManagerWillQueueDispatch:(TEALDispatch * \_Nullable)dispatch;

TEALConsentStatus

Unknown
Unknownステータスは、Consent Managerのデフォルト設定です。このステータスの場合、Consent Managerはローカルでイベントをキューに追加します。これは、断定的なConsented/NotConsentedステータスが設定されるまで続きます。

self.tealium.consentManager.userConsentStatus = Unknown

Consented
Consentedステータスは、ユーザーがトラッキングに同意している場合に設定されます。このステータスの場合、Consent Managerはすべてのトラッキングコールを通常通りに続行できます。

self.tealium.consentManager.userConsentStatus = Consented

NotConsented
NotConsentedステータスは、ユーザーがトラッキングを拒否している場合に設定されます。このステータスの場合、Consent Managerはすべてのトラッキングコールを中断し、SDKによるそれ以上の処理を停止します。

self.tealium.consentManager.userConsentStatus = NotConsented
