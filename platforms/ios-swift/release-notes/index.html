リリースノート

リリースノート

以下は、Tealium for Swift ライブラリのリリースごとの変更点をまとめた包括的なリストです。

2.6.3 (2022 年 3 月)

影響の小さい変更点

  • 公開設定が複数回取得される場合があるバグを修正。
  • dataLayer 変数で untilRestart の有効期限が検出されなくなるバグを修正。
  • 以下の LocationModule は非推奨として、それぞれのゲッターを推奨しました。
    • createdGeofences -> getCreatedGeofences(completion:)
    • lastLocation -> getLastLocation(completion:)
    • monitoredGeofences -> getMonitoredGeofences(completion:)

2.6.2 (2022 年 2 月)

影響の小さい変更点

  • セッション カウント ロジックがセッションの第 2 イベントでセッション ヒットを送信するものを修正。

2.6.1 (2022 年 2 月)

影響の小さい変更点

  • 列挙型に基づくエラーに関するログを改善。
  • セルフホスト JavaScript ファイルを用いている Tealium iQ アカウント用のセッション カウントを無効にできるように sessionCountingEnabled フラッグを TealiumConfig に追加。

2.6.0 (2022 年 1 月)

影響が中程度の変更点

  • アプリ内購入を自動的にトラッキングするための新たな TealiumInAppPurchase モジュールを追加。

影響の小さい変更点

  • リモート コマンドの名前とバージョンをデータレイヤーに追加。
  • iOS 15 内部クラッシュの回避策を isLowPowerModeEnabled に追加。
  • ライフサイクル イベントの処理について内部で変更。
  • 従来の Teealiym Objective-C ライブラリからの移行における非推奨の NSKeyedUnarchiver の内部使用を置き換え。
  • リモート設定のリクエストを最初に行うときに JSON リモート コマンドが重複する問題を修正。
  • gatherTrackDataにおける誤字を修正。

2.5.1 (2021 年 12 月)

影響が中程度の変更点

  • 静的ライブラリとしてコンパイルするとき (use_frameworks! を設定しないまま Cocoapods を使用したり、Tealium React Native ライブラリを使用するときなど) の Objective-C ファイルでの Swift コードのインポートを修正。

影響の小さい変更点

  • 自動ディープ リンク トラッキングが AppDelegate のディープ リンクのダイジェストすることを防止。これは、他の実装が、独自のディープ リンクを処理することができるようにするためです。
  • TagManagement モジュールと TealiumConfigに関するスレッドの問題を修正。

2.5.0 (2021 年 11 月)

影響の大きい変更点

  • AutoTracking モジュール が SwiftUI と UIKit をサポートできるようになりました。これは、クロスプラットフォーム (iOS、watchOS、tvOS および macOS) です。

影響が中程度の変更点

  • AutoTracking モジュール用の SPM サポートを追加。
  • SceneDelegate と SwiftUI のアプリ用のディープ リンクの自動処理を追加。
  • 自動ディープ リンク トラッキングを無効化するために false 値を用いて info.plist flag TealiumAutotrackingDeepLinkEnabled を追加。
  • 手動ディープ リンク トラッキングのサポートを追加。
    • SwiftUI: trackingAppOpenUrl() View モディファイアまたは TealiumAppTrackable コンテナ View を使用します。
    • UIKit: AppDelegate または SceneDelegate から handleDeepLink() を使用します。

影響の小さい変更点

  • メソッド Tealium.gatherTrackData()を用いて、すべてのデータレイヤーと Collector の変数を利用できるようにしました。
  • TealiumKeys を TealiumDataKeysに名称変更しました。これは、 gatherTrackData()によって使用するためです。
  • Codable のサポートを改善。このために、リリース ビルドの非コーダブル値を無視して、デバッグ ビルドでクラッシュを生じさせました。
  • メイン スレッドの Geofences (ジオフェンス) 読み込みの使用を減らしました。

既知の問題

  • 問題のリンティングにより、CocoaPods プラットフォームの iOS バージョンを 11 にバージョン アップしました (ただし、必要な場合は、ポストインストール スクリプトによって 9 にバージョンダウンしても安全です)。
  • Carthage が Xcode 13 によって警告を生成するようになりました (https://github.com/Carthage/Carthage/issues/3238)。

2.4.6 (2021 年 10 月)

影響の小さい変更点

  • HTTP リクエストを内部コマンドに送信する前に RemoteCommand を割り当て解除するときにクラックを生じさる可能性があるバグを修正。
  • データレイヤー変数が 30 分を超える長さのセッションでの永続化を妨げるバグを修正。
  • isLowPowerModeEnabled 通知の登録について、 batterySaver 機能が有効になっていない場合に、iOS 15 内部デッドロックでのクラッシュの可能性を防止するために、これを防止しました。
  • 同意を受け取った後にトラッキング コールのキューが処理されないバグを修正。

2.4.5 (2021 年 9 月)

影響の大きい変更点

  • Apple Watches でのトラッキングを妨げるバグを修正。
  • AppDelegate でのディープ リンク トラッキングを妨げるバグを修正。

影響が中程度の変更点

  • ライフサイクル モジュールによらずに Objective-C ライブラリからの移行を行うときにライフサイクル モジュールにクラッシュを生じさせるバグを修正。

影響の小さい変更点

  • メモリーの漏洩の可能性を回避するための構成内での visitorServiceDelegate および webViewDelegates の脆弱性についての言及。
  • 同一の commandIdを用いた 2 つのリモート コマンドを防止するためのチェック機能の追加。
  • 新たなキー deep_link_referrer_app および deep_link_referrer_url の追加。これは、ディープリンクが自動的に検出されてリファラーが現れるとき、または、メソッド Tealium.handleDeepLink(_:referrer:)に渡されるときに、追加されます。
  • ディープ リンクのクエリ パラメータの処理を改善。このために、同一セッション中に複数のディープ リンクを受け取る時に、旧クエリ パラメータを削除しました。
  • iPhone、iPad および Apple Watch の最新モデルを含むようにデバイス モデル名のリストを更新。

2.4.4 (2021 年 9 月)

影響が中程度の変更点

  • Android および Obj-C のライブラリと整合性が取れるように device_resolution と device_logical_resolution の形式を、 {height}x{width} から {width}x{height} に変更。

影響の小さい変更点

  • 失敗の割合を減らし、一部のイベントのトラッキングを防止するために、データのエンコードとデコードを改善。
  • device_logical_resolution をデータレイヤーに追加。
  • Tealium が管理するジオフェンスとアプリケーションの間の抵触の可能性を解消。
  • Tealium 初期化が完了する前に DeepLinks の処理ができるようにしました。
  • 設定における loggerType の変更がカスタム ロガー用の正しいロガーで反映されないバグを修正。
  • データレイヤーに渡さられる Date オブジェクトを ISO8601 形式の文字列に変換。

2.4.3 (2021 年 9 月)

  • 問題の公開によりバージョンをスキップ。

2.4.2 (2021 年 8 月)

影響の小さい変更点

  • IDFA および ATT の許可ステータスがランタイムを更新しない原因となったバグを最終的に修正。

2.4.1 (2021 年 7 月)

影響の小さい変更点

  • IDFA および ATT の許可ステータスがランタイムを更新しない原因となったバグを修正。
  • Objective-C ライブラリからの移行がデフォルトで無効となる原因となったバグを修正。

2.4.0 (2021 年 7 月)

影響が中程度の変更点

  • 不要なファイルのバックアップを防止するためのストレージの更新。

影響の小さい変更点

  • カスタム イベントの使用を通じてアプリ内のストリーミング メディアをトラッキングする Media モジュール を追加。
  • Remote HTTP Command から、従来の認証オプションを削除。

2.3.2 (2021 年 6 月)

影響が中程度の変更点

  • Application Storage を、 Caches ディレクトリの代わりに使用するために永続性ストレージ モジュールを更新。これによって、 iOS 14.7 移行へのアップグレード時における永続性データの削除が防止されます。

影響の小さい変更点

  • Connectivity モジュールについて、複数回のインスタンス化ではなく、 CTTelephonyNetworkInfo の 1 つのインタンスを使用するための更新。
  • UIWebView から WKWebViewへの Cookie の移行のために用いられる Cookie 移行ロジックを削除。
  • URL の検証に用いられる RegEx を更新。これによって、比較的一般的でない URL (localhost、ローカル IP や、ポート付きの URL など) が利用できるようになりました。

2.3.1 (2021 年 5 月)

影響の小さい変更点

  • 同意ポリシーを policyに戻すための軽微な修正。
  • decline_consent イベントがキューに登録されないまたはドロップされないことを確認。

2.3.0 (2021 年 4 月)

影響の大きい変更点

  • Cookie の同期問題を回避するために用いるためのに WKProcessPool インスタンス (Tealium タグ管理ウェブビュー用 TealiumConfig オブジェクト上のもの) を設定するオプションを追加。
  • データレイヤー値 call_type を削除し、 tealium_event_typeに置き換え。イベント タイプが標準トラックであって、スクリーン ビューでないときに、 tealium_event_type 値が event となりました (linkではなくなりました)。

読み込みルールまたは属性ルールが構成されていない場合は、call_type, update them to look for tealium_event_type instead. If they were checking for the value link on event types, update the condition to check for event を見つけるようにしました。

影響の小さい変更点

  • RemoteCommands モジュールにおける小規模メモリの漏洩を解決するための軽微な修正。
  • ライフサイクル モジュールがスリープ イベントおよびスリープ解除 イベント時に lifecycle_diddetectcrash=true を送信する点についての軽微な修正。
  • カスタム同意ポリシーがサポートされるようになりました。

2.2.2 (2021 年 1 月)

影響の小さい変更点

  • ユーザーの同意ステータスの有効期限を設定する機能と、同意の有効期限が切れたときにトリガーされるコールバックを追加。同意管理の詳細は、こちらから参照してください。
  • ユーザーの訪問者 IDをリセットするための公開メソッドを追加。
  • Tealium の初期化解除 (deinitialization) 時における Lifecycle リスナーの削除。

2.2.1 (2021 年 1 月)

影響の小さい変更点

  • 組み込みフレームワークとしての TealiumCore の削除。これは、依存関係として Carthage を使用する時に AppStore 送信エラーが生じる原因となっていたためです。
  • TealiumAttribution スキームを、 tealium-swift.xcodeproj の中に改めて追加。これにより、フレームワークが Carthage により構成できるようになります。

2.2.0 (2020 年 11 月)

新機能

  • 複数の指定イベントの間の時間をトラッキングする 時間付きイベント 機能を追加。

影響の小さい変更点

  • モジュール用のカスタム エンドポイントが設定できなくなる原因となっていたバグを修正。
  • 他のライブラリとの一貫性を保つために collectOverrideURL を overrideCollectURL に名称変更。
  • バッチ処理イベントについての、新たな URL オーバーライドのオプションを導入 overrideCollectBatchURL。
  • device_manufacturer をデータレイヤーに追加。

2.1.1 (2020 年 10 月)

新機能

  • Objective-C または Swift 1.x ライブラリからのアップグレードの場合に、自動的に 既存の永続性データを移行する ユーティリティを追加。
  • SKAdNetwork について、 TealiumAttribution モジュール内でのサポートを追加。

影響の小さい変更点

  • ATTrackingManager.AuthorizationStatus の値を device_tracking_authorization としてのディスパッチ ペイロードに追加 (TealiumAttribution モジュールが有効となっている場合)。

iOS 14 以降の Attribution (アトリビューション) および ATTransparency と SKAdNetworkのサポートの詳細は、こちらから参照してください。

2.1.0 (2020 年 10 月)

新機能

  • リモート コマンド機能を更新して、リモートまたはローカルでのホスト型 JSON 構成ファイルを使用するオプションを追加して、ウェブビューの要件を削除。Remote Command タグ オプションを用いる場合は、iQ Tag Management にて、ウェブビューが必要です。JSON 制御 リモートコマンドの詳細は、こちらから参照してください。
  • Approximate Location Tracking (iOS 14 以降) のサポートを追加。機能 requestTemporaryFullAccuracyAuthorization(purposeKey:) を TealiumLocation モジュールで装備。iOS 14 Location Updates (位置情報の更新) と、 暫定的な完全許可のリクエストの詳細は、こちらから参照してください

影響が中程度の変更点

  • platform 変数の値を小文字に変更し、モバイル セッション タイマーを訂正。以前は、ウェブと同様にセッションが測定されていました (一貫性のあるアクティビティ 30分)。platform 変数を小文字に変更することで、Customer Data Hub は、受信データをモバイル セッションとして認識するようになり、セッション所要時間が 2 分に短縮されました。

大文字と小文字を区別する platform 値 (iOS と ios) について読み込みルールまたは属性ルールで構成されたチェックがなされる場合は、読み込みルールを更新します。

影響の小さい変更点

  • origin: mobile をトラック ペイロードに追加

2.0.0 (2020 年 7 月)

新機能

  • Consent Manager モジュールに、新たな CCPA API を追加。Consent Manager ドキュメンテーションで詳細を確認してください。
  • ネイティブ ホスト型データレイヤーのサポート
  • Automatic Deep Link (自動ディープ リンク) トラッキング。これは、ユーザーがこれらのメソッドのいずれかからアプリを開いたときに、ディープ リンクの URL、 ユニバーサル リンクの URL、およびクエリ文字列パラメータをデータレイヤーに追加するものです。
  • Customer Data Hub UI から QR Code をスキャンして、Trace (トレース) を開始する機能
  • 新たな TealiumLogger を追加。これは、Apple 優先 OSLog と print 関数を比較して用いるものです。logType プロパティを TealiumConfig オブジェクトで設定することで、印刷を使用するオプションも残っています。Logging (ログ)の詳細は、こちらから参照してください。
  • 複数の補助的モジュールを、インストールを早く簡単にするために、コア ライブラリに統合。詳しくは モジュール セクション を参照してください。
  • track() メソッドについて、 Dispatch タイプ (View または Eventのもの) を受け付けるために更新。これによって、1 つの track メソッドをサポートすることができ、スクリーンとイベントのトラッキングが明らかになりました。詳細はこちらからご覧ください (track() メソッドの更新関係)。
  • データ変数用の追加の有効期限オプションに関するサポート。詳しくは Data Management (データ管理) を参照してください。
  • utag.js 内のセッション カウント ロジックを使わない、ネイティブ セッション カウントに対するサポート。これにより、セッション カウントの正確性が増し、一部のエッジ ケースが修正されました。こうしたエッジ ケースでは、バックグラウンド アクティビティが新たなセッションをトリガーして、セッション カウントが想定よりも高くなっていました。

⚠️重要事項⚠️ Tag Management Module がディパッチ オプションとして使用される場合は、関連する TiQ アカウント/プロファイル内で mobile.html テンプレートも更新する必要があります。このアクションが実施されない場合は、Tealium 内に重複セッションが記録されます。テンプレートを更新する方法、またはについては、担当の Account Manager に問い合わせて支援を求める方法については、[さらに読む] (https://community.tealiumiq.com/t5/iQ-Tag-Management/Managing-Tag-Templates/ta-p/21713#toc-hId--1934787229)。

影響の大きい変更点

  • Swift のベストプラクティスとの整合性を高めるための API の更新
  • パフォーマンス改善のためのモジュール システムの最適化
  • パフォーマンス改善のための個別モジュールの最適化
  • API の簡素化のために、複数のデリゲートオプションを削除
  • 接続性チェックの改善 - NWPathMonitor サポート対象のデバイスで使用できるようになりました
  • プロジェクトから CrashReporter モジュールを削除し、これを使用するための別のリポジトリを追加
  • RemoteCommands モジュール内の NotificationCenter を削除し、Delegate (デリゲート) パターンで置き換え

影響の小さい変更点

  • テスト容易性のためにプロトコルと結果タイプをさらに追加
  • 他の Tealium ライブラリとの一貫性を保つためにクラス名を更新
  • インライン コード ドキュメンテーションをさらに追加
  • ユニット テストを更新し、パフォーマンス テストを追加

2.x より前の旧リリース ノートについては、 Tealium for Swift 1.x を参照してください。
