CrashReporterモジュール

アプリのクラッシュを自動的にトラッキングします。このモジュールが有効になり、付属のフレームワークがインストールされると、アプリでクラッシュが発生した場合にクラッシュイベントがトリガーされます。

使用方法

CrashReporterモジュールは、アプリのクラッシュを自動的にトラッキングします。このモジュールが有効になり、付属のフレームワークがインストールされると、アプリでクラッシュが発生した場合にクラッシュイベントがトリガーされます。

アプリのクラッシュに関する重要な情報を取得するために、このフレームワークの使用を推奨します。

以下のプラットフォームがサポートされています。

  • iOS

インストール

Crash モジュールを Swift Package Manager または CocoaPods によってインストールします。

Swift Package Manager（推奨）

バージョン1.9.0以降でサポートされているSwift Package Managerは、Tealium Swiftライブラリをインストールするために推奨される最も簡単な方法です。

  1. Xcodeプロジェクトで、[File] > [Swift Packages] > [Add Package Dependency]を選択します。
  2. レポジトリの URL として「https://github.com/tealium/tealium-swift-crash-reporter」と入力します
  3. バージョンルールを設定します。通常は"Up to next major"が推奨されます。現在のTealium Swiftライブラリバージョンがリストに表示されない場合は、Swiftパッケージキャッシュをリセットします。
  4. `TealiumCrashModule を選択し、 [Frameworks] > [Libraries & Embedded Content]で、Xcode プロジェクトの各アプリ ターゲットにこれを追加します
  5. この手順 に従って、 tealium-swift ライブラリからの他のモジュールを追加します。

CocoaPods

CrashReporterモジュールをCocoaPodsによってインストールするには：

  1. 次のポッドをPodfileに追加します。

    pod 'TealiumCrashModule'

    Podfile には、既に、以下の行が含まれています。

    pod 'tealium-swift'

  2. 次のコマンドを実行します。

    pod install

Xcode 12 および Apple の M1 チップ のリリースの後は、CrashReporter モジュールによる Carthage のサポートはなくなります。このモジュールでは PLCrashReporter を使用しており、これれにより新たな Apple エコシステムをサポートするための XCFramework が提供されますが、Carthage は、事前構築された XCFramweworkをサポートしません。

構成

CrashReporter モジュールをインスタンス化するためには、SDK の初期化時に TealiumConfig オブジェクトでこれを指定する必要があります。

import TealiumCore
import TealiumCrashModule

let config = TealiumConfig(account: "ACCOUNT",
                               profile: "PROFILE",
                               environment: "ENVIRONMENT",
                               datasource: "DATASOURCE")

// add desired Collectors - no need to include if want compiled Collectors
config.collectors = [Collectors.AppData,
					 Collectors.Crash,] // Instantiates the CrashReporter module
tealium = Tealium(config: config) { _ in }

データレイヤー

モジュールが有効な場合に、トラッキングコールのたびに以下の変数が送信されます。

変数 説明 例
crash_uuid この特定のクラッシュの一意識別子 "CC2DA0E9-E544-429A-AC5E-A268FC62F02A"
device_memory_usage* クラッシュ時にアプリによって使用されているデバイスの現在のメモリ "143.57MB"
app_memory_usage クラッシュ時にアプリによって使用されているデバイスの現在のメモリ "143.57MB"
device_memory_available* クラッシュ時のデバイスの空きメモリ "1068.88MB"
memory_free クラッシュ時のデバイスの空きメモリ "1068.88MB"
device_os_build 現在のOSのビルド番号 "15E217"
crash_process_id クラッシュ時のアプリのPID "84351"
crash_process_path アプリが実行されていたパス "/DemoApp.app/DemoApp"
crash_parent_process アプリを起動した親プロセス "launchd_sim"
crash_parent_process_id 親プロセスのPID "83183"
crash_name わかりやすいクラッシュの名前（該当する場合） "Crash Name"
crash_cause クラッシュの原因 "Crash Reason"
crash_signal_code クラッシュの原因となっている信号コード "#0"
crash_signal_name クラッシュの原因となっている信号名 "SIGABRT"
crash_signal_address クラッシュの信号アドレス "4572945982"
crash_libraries クラッシュ時の読み込み済みのライブラリのリスト [{"baseAddress": "0xa3e0000", "codeType": { "arch": 64, "typeEncoding": "Mach"}, "imageName": "/Applications/Xcode9.3.app/Contents/ Developer/Platforms/iPhoneOS.platform/ Developer/Library/CoreSimulator/Profiles /Runtimes/iOS.simruntime/Contents/ Resources/RuntimeRoot/usr/lib/dyld_sim", "imageSize": 212992, "imageUuid": "4015e9b70bde"}]
crash_threads クラッシュ時のアクティブなスレッドをすべて返します。 {"crashed": 1,"registers": {"cs": "0x07"},"stack": {"instructionPointer": 4572945982,"symbolInfo": {"symbolName": "","symbolStartAddr": 0},"threadId": ""}}
  • この変数は、以前のリリースとの後方互換性のために重複しています。

クラッシュの発生

TealiumCrashReporteriOS.frameworkがプロジェクトに追加されたら、追加の初期化手順は必要ありません。クラッシュモジュールが有効になっていて、Tealium SDKがインストールされている限り、アプリでクラッシュが発生すると、クラッシュデータが自動的にトラッキングコールに追加されます。

APIリファレンス

invokeCrash()

CrashReporterモジュールのクラッシュを発生させます。

TealiumCrashReporter.invokeCrash(name, reason)
パラメータ 型 説明 例
name String クラッシュの名前 "TEST_CRASH_NAME"
reason String クラッシュの理由 "TEST_CRASH_REASON"
