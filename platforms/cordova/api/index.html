CORDOVA

APIリファレンス

Tealium for Cordovaで提供されているクラスとメソッドに関するリファレンスガイド。

クラス：Tealium

以下は、一般的に使用されるCordova Tealiumクラスのメソッドをまとめたものです。

メソッド 説明
addRemoteCommand() リモートコマンドをリモートコマンドマネージャに追加します。
addData() 永続性データレイヤーにデータを追加します。
getData() データレイヤーからの指定値を取得します。
getConsentCategories() ユーザーの同意済みカテゴリを取得します。
getConsentStatus() ユーザーの現在のステータスを取得します。
getVisitorId() 現在の訪問者 ID を取得します。
initialize() 構成パラメータを用いて Tealium を初期化します。
joinTrace() 所与の ID でトレースに参加します。
leaveTrace() アクティブなトレース セッションから退出します。
removeData() addData() を使用して以前設定された永続性データを削除します。
removeRemoteCommand() リモートコマンドをリモートコマンドマネージャから削除します。
setConsentCategories() ユーザーの同意カテゴリを設定します。
setConsentExpiryListener() 同意の有効期限が過ぎたリスナーまたはコールバックを設定します。
setConsentStatus() ユーザーの同意ステータスを設定します。
setVisitorServiceListener() 訪問者サービスのリスナーまたはコールバックを定義します。
terminateInstance() Tealium インスタンスを無効化し破棄します。
track() イベントまたはスクリーン ビューをトラッキングします

addRemoteCommand()

リモートコマンドをリモートコマンドマネージャに追加します。

tealium.addRemoteCommand(id, callback);
パラメータ 型 説明 例
id String タグ構成からのコマンドIDの名前 "test_command"
callback Function リモート コマンドからレスポンスを受信した後に実行されるコールバック関数。コールバックは、タグマッピングからのキーと値のペアのペイロードを返します。 （例を参照）

例:

tealium.addRemoteCommand("firebase", (payload) => {

  var eventName = payload["firebase_event_name"];
  var eventProperties = payload["firebase_event_properties"];

  analytics.logEvent(eventName, eventProperties);
});

addData()

所与の有効期限中、永続性データ ストレージに永続性データを追加します。

tealium.addData(data, expiry);
パラメータ 型 説明 例
data Object キーと値のペアから成るJSONオブジェクト。ここで、キーは文字列で、値は単一文字列または文字列の配列になります {"persistent_key2" : "persistent_val2"}
expiry Expiry データを永続化する対象の期間の長さ Expiry.forever

Collectors

Collectors (コレクター) は、デバイスから補足情報を収集し、データレイヤーにそれを添付したうえで、Tealium Customer Data Hub にそれを転送するモジュールです。コレクターの中にはコア ライブラリに含まれるものもあれば、オプションで、別個のモジュールとしてインストールされるものもあります。

以下の表では、利用可能なコレクターを列挙しています。デフォルト コレクターは、コレクター名に「*」印を付して注記しています。

Collector Name TealiumConfig のリファレンス
AppData* Collectors.AppData
Connectivity* Collectors.Connectivity
Device Collectors.Device
Lifecycle Collectors.Lifecycle

これらのモジュールは、構成オブジェクトの collectors プロパティを使用して有効化または無効化され、定数値は window.tealium.utils.Collectorsから入手できます。

ConsentExpiry

同意設定の有効期限を定義します

パラメータ 型 説明 例
time Number 有効期限が満了するまでの期間 90
unit TimeUnit 有効期限が満了するまでの期間の単位 TimeUnit.days

例:

var ConsentExpiry = tealium.utils.ConsentExpiry
var TimeUnit = tealium.utils.TimeUnit
ConsentExpiry.create(90, TimeUnit.days)

TimeUnit

値 説明
.minutes 分
.hours 時間
.months 月
.days 日

ConsentPolicy

遵守するべき同意ポリシーを定義します。同意ポリシーが config オブジェクトで定義されていない場合は、Consent Manager が無効化されます。定数は window.tealium.utils.ConsentPolicy から入手できます。

例:

var ConsentPolicy = tealium.utils.ConsentPolicy
var gdpr = ConsentPolicy.gdpr
値 説明
.gdpr GDPR
.ccpa CCPA

Dispatchers

Dispatchers (ディスパッチャー) は、データレイヤーからデータを送信させ、これを Tealium エンドポイントに送信するモジュールです。以下のディスパッチャーが現在使用できます。 - 定数は、 window.tealium.utils.Dispatchersから入手できます。

Dispatcher Name TealiumConfig のリファレンス
Collect Dispatchers.Collect
RemoteCommands Dispatchers.RemoteCommands
TagManagement Dispatchers.TagManagement

少なくとも 1 つのディスパッチャーが必要です。ディスパッチャーが 1 つも指定されていない場合は、データはどこにも送信されません。

Expiry (有効期限)

カスタム データの有効期限を定義します。

例:

Expiry.session
値 説明
session 現在のアクティブなセッションのライフタイム。
untilRestart アプリが再起動するまで。
forever アプリのライフタイムについては、削除されるまで。

例:

var Expiry = tealium.utils.Expiry
var session = Expiry.session

getData()

永続性データレイヤーの指定キーの値を取得し、コールバック関数の形式で返します。

tealium.getData(key, callback);
パラメータ 型 説明
key String データレイヤーから取得するべきキー
callback Function 取得されたキーの値を使用するためのコールバック関数

例:

tealium.getData(key, function(data) {
  console.log(data)
});

getConsentCategories()

ユーザーの同意済みカテゴリを取得します。

tealium.getConsentCategories(callback);
パラメータ 型 説明
callback Function 同意カテゴリを使用するためのコールバック関数

例:

tealium.getConsentCategories(categories => {
    console.log("Consent Categories: " + categories)
});

getConsentStatus()

ユーザーの訪問者 ID を取得し、コールバック関数の形式で返します。

tealium.getConsentStatus(callback);
パラメータ 型 説明
callback Function 同意ステータスを使用するためのコールバック関数

例:

tealium.getConsentStatus(function(categories) {
  console.log("Consent Status: " + status)
})

getVisitorId()

ユーザーの訪問者 ID を取得し、コールバック関数の形式で返します。

tealium.getVisitorId(callback);
パラメータ 型 説明 例
callback Function 訪問者 ID を使用するためのコールバック関数 （例を参照）

例:

tealium.getVisitorId(function(visitorId) {
  console.log("VisitorId: " + visitorId)
})

initialize()

他のメソッドを呼び出す前にTealiumを初期化します。

tealium.initialize(config);
パラメータ 型 説明 例
config TealiumConfig Tealium 構成パラメータ （例を参照）
callback Function (オプション) Tealium インスタンスが準備できた後のコールバック関数 （例を参照）

例:

let Environment = tealium.utils.Environment;
let Collectors = tealium.utils.Collectors;
let Dispatchers = tealium.utils.Dispatchers;
let ConsentPolicy = tealium.utils.ConsentPolicy;
let Expiry = tealium.utils.Expiry;

let config = {
    account: 'tealiummobile',
    profile: 'demo',
    environment: Environment.dev,
    dispatchers: [
        Dispatchers.Collect,
        Dispatchers.TagManagement,
        Dispatchers.RemoteCommands
    ],
    collectors: [
        Collectors.AppData,
        Collectors.DeviceData,
        Collectors.Lifecycle,
        Collectors.Connectivity
    ],
    consentLoggingEnabled: true,
    // consentExpiry: {
    //     'time': 10,
    //     'unit': 'days'
    // },
    // consentPolicy: ConsentPolicy.gdpr,
    lifecycleAutotrackingEnabled: true,
    batchingEnabled: false,
    visitorServiceEnabled: true,
    useRemoteLibrarySettings: false
};

window.tealium.initialize(config, function(success) {
    console.log("Init was: " + success)
    if(success) {
        tealium.setVisitorServiceListener(logVisitorUpdated)
        tealium.setConsentExpiryListener(logConsentExpired)
        tealium.addRemoteCommand("hello-world", logRemoteCommand)
    }
})

joinTrace()

指定されたIDでトレースに参加します。Tealium Customer Data Hubでの、 トレース 機能の詳細は、こちらから参照してください。

tealium.joinTrace(id);
パラメータ 型 説明 例
id String CDH から取得されたトレース ID abc123xy

leaveTrace()

トレースは、アプリのセッション中、 leaveTrace() メソッド (以前に参加したトレースから退出し、訪問者セッションを終了) が呼び出されるまでアクティブなままになります。

tealium.leaveTrace();

LogLevel

ログ (記録) されている情報の量を制御するログ レベル プロパティを、以下の値のいずれかに設定します。

値 説明
LogLevel.dev アプリケーションの進捗を表示する情報イベント
LogLevel.qa アプリケーションをデバッグするために用いられるデバッグレベルのイベント
LogLevel.prod 重要なエラーと失敗などのエラー イベント
LogLevel.silent ログがありません (デフォルト)

例:

LogLevel.dev

setConsentCategories()

ユーザーの同意カテゴリを設定します。カテゴリを設定するには、文字列の配列を渡します。デフォルトは、 ConsentStatus が .consentedに設定されるまで空の配列です。同意ステータスが .consented であって、 setConsentCategories() メソッドを使用したカテゴリの指定がなされていない場合は、すべてのカテゴリが設定されます。

tealium.setConsentCategories(categories);
パラメータ 型 説明 例
categories ConsentCategories[] ユーザーの同意カテゴリの配列 [ConsentCategories.email, ConsentCategories.personalization]

例:

var ConsentCategories = tealium.utils.ConsentCategories;

tealium.setConsentCategories([ConsentCategories.analytics, ConsentCategories.email]);

ConsentCategories

値 説明
analytics 分析
affiliates 加盟企業
displayAds ディスプレイ広告
email Eメール
personalization パーソナライゼーション
search 検索
social ソーシャル
bigData ビッグデータ
mobile モバイル
engagement エンゲージメント
monitoring モニタリング
crm CRM
cdp CDP
cookieMatch Cookie照合
misc Misc (雑)

setConsentExpiryListener()

ユーザーの同意設定が ConsentExpiryに従って有効期限が切れた後に実行されるコールバックを定義します。

tealium.setConsentExpiryListener(callback);
パラメータ 型 説明
callback Function 同意の有効期限が切れた後に実行するためのコード

例:

tealium.setConsentExpiryListener(() => {
    print("Consent Expired");
});

setConsentStatus()

ユーザーの同意ステータスを設定します。デフォルトは、変更されるまで .unknown です。

tealium.setConsentStatus(status);
パラメータ 型 説明 例
status String ユーザーの同意ステータス ConsentStatus.consented

例:

var ConsentStatus = window.tealium.utils.ConsentStatus

tealium.setConsentStatus(ConsentStatus.consented);

ConsentStatus

値 説明
.consented ユーザーが同意済みです。
.notConsented ユーザーが同意していません。
.unknown ユーザーが同意済みであるかどうか不明。

setVisitorServiceListener()

訪問者プロファイルが更新されたときに実行されるコールバックを定義します。コールバック レスポンスでは、更新された VisitorProfile が提供されます。

VisitorServiceモジュールは、Tealium Customer Data Hubのデータレイヤーエンリッチメント機能を実装します。

Tealium AudienceStreamのライセンスがある場合に、訪問者プロファイルを使用してモバイル アプリケーションのユーザー エクスペリエンスを強化するには、このモジュールを使用することをお勧めします。AudienceStreamのライセンスがない場合は、訪問者プロファイルが返されないため、このモジュールを使用することは推奨されません。

tealium.setVisitorServiceListener(callback);
パラメータ 型 説明
callback Function 更新された訪問者プロファイルが返された後に実行するためのコード

例:

tealium.setVisitorServiceListener((profile) => {
    console.log(JSON.stringify(profile));
});

Dispatch

イベントやビューを作成するためのユーティリティ オブジェクト。

ビュー

スクリーン ビューをトラッキングするには、キーのタイプが viewName でオプションが data であるオブジェクトを track() メソッドに渡します。これは、手動で、または tealium.utils.Dispatch.view(viewName, data)のユーティリティ メソッドを使用して供給することができます。

例:

var Dispatch = tealium.utils.Dispatch
var tealView = Dispatch.view("VIEW_NAME", {"key": "value"})
tealium.track(tealView)

// the following is equivalent
tealium.track({
  type: "view",
  viewName: "VIEW_NAME",
  data: {"key": "value"}
})

イベント

イベントをトラッキングするには、キーのタイプが eventName でオプションが data であるオブジェクトを track() メソッドに渡します。これは、手動で、または tealium.utils.Dispatch.event(eventName, data)のユーティリティ メソッドを使用して供給することができます。

例:

var Dispatch = tealium.utils.Dispatch
var tealEvent = Dispatch.event("EVENT_NAME", {"key": "value"})
tealium.track(tealEvent)

// the following is equivalent
tealium.track({
  type: "event",
  eventName: "EVENT_NAME",
  data: {"key": "value"}
})

`RemoteCommand

設定された Remote Command を定義するインターフェイス

値 説明
id RemoteCommand の一意識別子名
path (オプション) マッピングに用いるためのローカル ファイル
url (オプション) マッピングに用いるためのリモート ファイル
callback (オプション) RemoteCommand 用のコールバック関数

パスと URL はオプションですが、Tealium iQ Tag Management によってマッピングが処理される場合には、そのうちの一方のみを提供するか、両方を省略します。

React Native アプリの範囲内でコマンドが処理される場合にコールバックを提供します。ハンドラーがネイティブ登録されている場合にコールバックを省略します。

以下の例は、Javascript が処理するコマンドに関するローカル マッピング ファイルです。

let localCommand: RemoteCommand = {
    id: "hello-world",
    path: "hello-mappings.json",
    callback: (payload) => {
        //...
    }
}

以下の例は、ネイティブ処理がされたコマンドに関するリモート マッピング ファイルです。

let remoteNativeCommand: RemoteCommand = {
    id: "hello-world",
    url: "https://you.domain.com/hello-mappings.json"
}
