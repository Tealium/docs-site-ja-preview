XAMARIN

APIリファレンス

Tealium for Xamarinで提供されているクラスとメソッドに関するリファレンスガイド。

クラス：Tealium

Tealium Xamarin ライブラリ用の Tealium クラスのメソッドと定数。

AddRemoteCommand()

リモートコマンドをリモートコマンドマネージャに追加します。

void AddRemoteCommand(IRemoteCommand remoteCommand);
パラメータ 型 説明
remoteCommand IRemoteCommand IRemoteCommand インターフェイスの実装。必要なリモート コマンド プロパティを提供するもの。

例:

public class MyRemoteCommand : IRemoteCommand
    {
        public string CommandId => "my_remote_command";
        public string Description => "description of my remote command";
        public string Path => null;
        public string Url => null;

        public void Dispose()
        {
            //
        }

        public void HandleResponse(IRemoteCommandResponse response)
        {
            var str = response.Payload.GetValueForKey<String>("key");
            if (str == "")
            {
                // etc
            }
        }
    }

tealium.AddRemoteCommand(new MyRemoteCommand());

AddToDataLayer()

所与の有効期限内に、永続性データ ストレージに永続性データを追加します。

void AddToDataLayer(IDictionary<string, object> data, Expiry expiry);
パラメータ 型 説明
data IDictionary<string, object> キーと値のペアの IDictionary。ここで、キーは文字列で、値は、primitives (プリミティブ) または collections (コレクション) になります
expiry Expiry データを永続化する期間 (長さ) 
tealium.AddToDataLayer(new Dictionary<string, object> {
  { "user_language", lang }
}, Expiry.Forever);

ConsentCategory

同意カテゴリーの値。GetConsentCategories および SetConsentCategoriesによって使用するため。

同意カテゴリは、以下のとおりです。

  • ConsentManager.ConsentCategory.Analytics
  • ConsentManager.ConsentCategory.Affiliates
  • ConsentManager.ConsentCategory.BigData
  • ConsentManager.ConsentCategory.Cdp
  • ConsentManager.ConsentCategory.CookieMatch
  • ConsentManager.ConsentCategory.Crm
  • ConsentManager.ConsentCategory.DisplayAds
  • ConsentManager.ConsentCategory.Email
  • ConsentManager.ConsentCategory.Engagement
  • ConsentManager.ConsentCategory.Misc
  • ConsentManager.ConsentCategory.Mobile
  • ConsentManager.ConsentCategory.Monitoring
  • ConsentManager.ConsentCategory.Personalization
  • ConsentManager.ConsentCategory.Search
  • ConsentManager.ConsentCategory.Social

ConsentStatus

同意ステータスの値。SetConsentStatus()によって使用するため。

値 説明
ConsentManager.ConsentStatus.Consented 同意あり
ConsentManager.ConsentStatus.NotConsented 同意なし
ConsentManager.ConsentStatus.Unknown 不明

環境

環境は、3 つのデフォルト環境 (Dev (開発)、QA、Prod (本番)) のいずれか、または公開するカスタム環境です。

値 説明
Environment.dev 開発
Environment.qa QA/UAT
Environment.prod 本番

Expiry (有効期限)

永続性データまたは揮発性データの有効期限を定義します。AddToDataLayer()によって使用するため。

値 説明
Expiry.Session 現在のアクティブなセッションの期間。
Expiry.UntilRestart 次の再起動までの期間。
Expiry.Forever アプリが削除されるまでの期間。

GetFromDataLayer()

永続性データレイヤーから指定キーの値を取得します。

object? GetFromDataLayer(string key);
パラメータ 型 説明
key String データレイヤーから取得するべきキー 
string myString = (string)tealium.GetFromDataLayer("user_language");

GetConsentCategories()

ユーザーが同意したカテゴリのリストを取得します。

List<ConsentCategory>? GetConsentCategories();

例:

tealium.GetConsentCategories().ForEach((c) =>
{
    if (c == ConsentManager.ConsentCategory.Analytics)
    {
        // consented to Analytics
    }
});

GetConsentStatus()

ユーザーの同意の現在ステータスを取得します。ConsentStatus 値を返します。

ConsentStatus GetConsentStatus();

例:

switch (tealium.GetConsentStatus()) {
    case ConsentManager.ConsentStatus.Unknown:
        break;
    case ConsentManager.ConsentStatus.Consented:
        break;
    case ConsentManager.ConsentStatus.NotConsented:
        break;
}

GetVisitorId()

ユーザーの訪問者 ID を取得し、コールバック関数の形式で返します。

string? visitorId = tealium.GetVisitorId();

JoinTrace()

指定されたIDでトレースに参加します。Tealium Customer Data Hubでの、 トレース 機能の詳細については、こちらから参照してください。

tealium.JoinTrace(id);
パラメータ 型 説明 例
id String トレース ID abc123xy

LeaveTrace()

トレースは、アプリのセッション中、 leaveTrace() メソッド (以前に参加したトレースから退出し、訪問者セッションを終了) が呼び出されるまでアクティブなままになります。

tealium.LeaveTrace();

RemoveFromDataLayer()

AddToDataLayer() を使用して以前設定された永続性データを削除します。

void RemoveFromDataLayer(ICollection<string> keys);
パラメータ 型 説明
keys ICollection<String> キー名のコレクション 
tealium.RemoveFromDataLayer(new string[] { "key1", "key2" });

RemoveRemoteCommand()

リモートコマンドをリモートコマンドマネージャから削除します。

void RemoveRemoteCommand(string id);
パラメータ 型 説明
id String 削除するコマンドIDの名前 
tealium.RemoveRemoteCommand("firebase");

SetConsentCategories()

ユーザーが同意したカテゴリのリストを設定します。ConsentCategoryによって使用するため。

void SetConsentCategories(List<ConsentCategory> categories);
パラメータ 型 説明
categories List<ConsentCategory> ユーザーの同意カテゴリのリスト。 
tealium.SetConsentCategories(new List<ConsentManager.ConsentCategory>()
{
    ConsentManager.ConsentCategory.DisplayAds,
    ConsentManager.ConsentCategory.Analytics
});

SetConsentExpiryListener()

同意設定の有効期限が切れたときに実行されるコールバック メソッドを設定します。

void SetConsentExpiryListener(Action callback);
パラメータ 型 説明
callback Action 同意の有効期限が切れたときに実行するためのコード。 
tealium.SetConsentExpiryListener(() =>
{
    System.Diagnostics.Debug.WriteLine("Consent Expired");
});

SetConsentStatus()

ユーザーのための同意ステータスを設定します。デフォルト値は、設定されるまで .Unknown です。

void SetConsentStatus(ConsentStatus status);
パラメータ 型 説明
status ConsentStatus ユーザーの同意ステータス。ConsentManager.ConsentStatusを参照してください。 
tealium.SetConsentStatus(ConsentManager.ConsentStatus.NotConsented);

SetVisitorServiceListener()

訪問者プロファイルが更新されたときに実行されるコールバック関数を設定します。コールバック レスポンスでは、更新された VisitorProfile が提供されます。visitorServiceEnabled が [true] に設定されたときに使用されるため。　

Tealium AudienceStream のライセンスがある場合に、訪問者プロファイルを使用してモバイル アプリケーションのユーザー エクスペリエンスを強化するには、この機能を使用します。

void SetVisitorServiceListener(Action callback);
パラメータ 型 説明
callback Action 訪問者プロファイルが取得されたときに実行されるコード。 
tealium.SetVisitorServiceListener((visitorProfile) =>
{
    System.Diagnostics.Debug.WriteLine("Visitor Updated");
    System.Diagnostics.Debug.WriteLine($"Visitor: {visitorProfile}");
});

TerminateIntance()

Tealium ライブラリを無効化し、すべてのモジュール設定を削除します。

tealium.TerminateIntance();

Track()

TealiumEvent または TealiumView のいずれかのディスパッチ イベントをトラッキングします。

tealium.Track(dispatch);
パラメータ 型 説明
dispatch TealiumView またはTealiumEvent イベント名とデータレイヤーを使用しての Tealium ディスパッチ。

TealiumView

スクリーン ビューをトラッキングするには、 TealiumView(viewName, data) のインスタンスを Track() メソッドに渡します。 TealiumView は、 tealium_event としてトラッキング コールに表示されるビュー名と、オプションのデータ ディクショナリで構成されます。

let tealView = TealiumView("VIEW_NAME", {"key": "value"});
tealium.Track(tealView);

TealiumEvent

非ビュー イベントをトラッキングするには、 TealiumEvent(eventName, data) のインスタンスを Track() メソッドに渡します。 TealiumEvent は、 tealium_event としてトラッキング コールに表示されるイベント名と、オプションのデータ ディクショナリで構成されます。

let tealEvent = TealiumEvent("EVENT_NAME", {"key": "value"});
tealium.Track(tealEvent);

クラス：TealiumConfig

以下は、 TealiumConfig クラスのプロパティをまとめたものです。

パラメータ 型 説明 例
account String (必須) Tealium アカウントの名前 "companyXYZ"
profile String (必須) Tealium プロファイルの名前 "main"
environment Environment (必須) Tealium 環境の名前 Environment.dev
dataSource String CDH データ ソース キー "abc123"
collectors Collectors[] (Required) Sets the list of Collectors 次を用いて Tealium ライブラリを初期化するには [Collectors.AppData]
dispatchers Dispatchers[] (Required) Sets the list of Dispatchers 次を用いて Tealium ライブラリを初期化するには [Dispatchers.Collect]
customVisitorId String カスタム訪問者 ID を設定します "ALK2398LSDKJ3289SLKJ3298SLKJ3"
memoryReportingEnabled Boolean DeviceDataモジュールでメモリレポートを有効または無効にします（デフォルトでは無効）。 true
overrideCollectDomain String Tealium Collect URL のドメインをオーバーライドします。ファーストパーティ ドメインで使用されるため。 "your-domain.com"
overrideCollectURL String データを別のエンドポイントに送信するようにTealium CollectのURLをオーバーライドします。イベント バッチング機能を用いる場合は、 overrideCollectBatchURL プロパティのオーバーライドもします。 "https://custom-domain.com/event"
overrideCollectBatchURL String データを別のエンドポイントに送信するように Tealium Collect のバッチ URL をオーバーライドします。 "https://custom-domain.com/batch-event"
overrideLibrarySettingsURL String 公開設定の URL をオーバーライドします。 "https://custom-domain.com/mobile.html"
overrideTagManagementURL String Tag Managementモジュールで使用されるデフォルトのURLをオーバーライドします。これは、Tealium JavaScriptファイルをセルフホストしている場合に必須です。 "https://custom-domain.com/path/env/utag.js"
deepLinkTrackingEnabled Boolean 標準ディープ リンクの自動トラッキング (Facebook などのソースからアプリへのリンクや、QR トレースなど) を有効化または無効化します。(デフォルト: 有効) false
qrTraceEnabled Boolean QR トレースを有効化または無効化します。(デフォルト: 有効) false
loglevel LogLevel ログ (記録) されている情報の量を制御するログ レベル プロパティを設定します (デフォルト: silent) LogLevel.dev
consentExpiry ConsentExpiry ユーザーの同意設定の有効期限を設定します。(デフォルト : ポリシーによる) ConsentExpiry(90, TimeUnit.days)
consentLoggingEnabled Boolean 同意の記録機能を有効にします。この機能では、監査目的で同意ステータスの変化をすべてTealium Customer Data Hubに送信します。(デフォルト: 有効) true
consentPolicy ConsentPolicy 同意ポリシーを設定します。たとえばCCPA または GDPR。Consent Manager は、このプロパティが設定されている場合のみ有効です。 ConsentPolicy.gdpr
lifecycleAutotrackingEnabled Boolean ライフサイクル自動トラッキングを有効化または無効化します。(デフォルト: 有効) false
useRemoteLibrarySettings Boolean モバイル公開設定 (デフォルト: 有効) を有効化または無効化します。モバイル公開設定を Tealium iQ タグ管理で設定するか、機能を無効化します。 false
visitorServiceEnabled Boolean データレイヤーエンリッチメント API を使用して Visitor Profile (訪問者プロファイル) の自動取得を有効化または無効化します (デフォルト: 無効) true
remoteCommands RemoteCommand[] インスタンスの準備ができたときに追加するための Sets a list of RemoteCommand オブジェクト [{ id: "hello-world", callback: (payload) => { console.log("hello-world: " + JSON.stringify(payload)); } }]

Collectors

コレクター (Collectors) は、デバイスから補足情報を収集し、データレイヤーにそれを添付して、Tealium Customer Data Hub にそれを転送するモジュールです。コレクターの中にはコア ライブラリに含まれるものもあれば、オプションで、別個のモジュールとしてインストールされるものもあります。

以下の表では、利用可能なコレクターを列挙しています。デフォルト コレクターは、コレクター名に「*」印を付して注記しています。

Collector Name TealiumConfig のリファレンス
AppData* Collectors.AppData
Connectivity* Collectors.Connectivity
Device Collectors.Device
Lifecycle Collectors.Lifecycle

このモジュールは、 Collectors プロパティ (TealiumConfigに設定したもの) を用いて有効化または無効化されます。

TealiumConfig tealConfig = new TealiumConfig(
    // ...
    new List<Collectors> {
        Collectors.LifeCycle, Collectors.Device
    }
    // ...
);

ConsentExpiry

ユーザーの同意設定の有効期限を TealiumConfig オブジェクトに設定します。

パラメータ 型 説明
time Number 有効期限が満了するまでの期間。
unit TimeUnit 有効期限が満了するまでの期間の単位。次のうちのいずれか: TimeUnit.Minutes、 TimeUnit.Hours、 TimeUnit.Days、 TimeUnit.Months 
TealiumConfig config = new TealiumConfig(
  //...
  consentExpiry = new TimeUnit(90, TimeUnit.Days)),
  //...
);

ConsentPolicy

同意ポリシーを設定して、 TealiumConfig オブジェクトで使用します。同意ポリシーが設定されていない場合は、consent manager が無効化されます。

TealiumConfig config = new TealiumConfig(
  //...
  consentPolicy = ConsentPolicy.GDPR,
  //...
);

同意値:

  • ConsentPolicy.GDPR
  • ConsentPolicy.CCPA

Dispatchers

Dispatchers (ディスパッチャー) は、データを Tealium エンドポイントに送信するモジュールです。これらは、 TealiumConfig オブジェクトに設定されます。

TealiumConfig tealConfig = new TealiumConfig(
    // ...
    new List<Dispatchers> {
        Collectors.Collect, Collectors.RemoteCommands
    }
    // ...
);

使用可能なディスパッチャーは次のとおりです。

  • Dispatchers.Collect
  • Dispatchers.RemoteCommands
  • Dispatchers.TagManagement

少なくとも 1 つのディスパッチャーが必要です。

LogLevel

ログ (記録) されている情報の量を制御するログ レベル プロパティを、以下の値のいずれかに設定します。

値 説明
LogLevel.dev アプリケーションの進捗を表示する情報イベント
LogLevel.qa アプリケーションをデバッグするために用いられるデバッグ レベルのイベント
LogLevel.prod 重要なエラーと失敗などのエラー イベント
LogLevel.silent ログがありません (デフォルト)
