データソース

EventStreamのデータソースについて説明します。

データソースとは、EventStreamにデータを送るシステムのことです。これらのシステムには、ウェブサイト、ネイティブモバイルアプリやイベントデータをレポートするカスタムアプリケーションなどがあります。データソースはプラットフォーム固有で、必要なコードとインストールを完了する方法を提供します。

データソースのタイプの例を示します。

  • ウェブ（iQ TMS）のためのJavaScript
  • Swift（iOS）
  • Android
  • Apple TV
  • Roku
  • Python
  • Java
  • HTTP API

Tealiumをお客様のウェブサイトかアプリに追加するには、まずデータソースを作成して、コードとインストールを完了する方法を得ます。各データソースには名前、説明と、インストールされたプラットフォームを一意に特定するデータソースキーがあります。

仕組み

データソースは、CDH にデータを送るお客様のサイトとアプリを容易に特定して記録し続けるために使用します。

仕組みは次のとおりです。

  • まず、Swift（iOS）、Android や JavaScript for Web など、Tealium をインストールしたプラットフォームを選び、"mydomain.com"や"MyBrandApp"のような名前を付けます。
  • CDH にデータソースを作成する際に、サンプルコードと_data source key_と呼ばれる一意の値とともにインストール方法が渡されます。データソースキーは、インスタレーションコードの中で、作成したデータソースに一致させるために使用します。
  • データソースキーを使ってコードをインストールしたら、そのデータソースから来るイベントを容易に特定するため、_ライブイベント_を表示させます。

データソースの例：

インスタレーションコードの例：

var tealConfig = TealiumConfig(   
  account: "ACCOUNT",
  profile: "PROFILE",
  environment: "ENVIRONMENT",
  datasource: "DATASOURCE")

let tealium = Tealium(config: tealConfig)
