EVENTSTREAM/GETTING STARTED

イベントフィード

EventStreamのイベントフィードについて説明します。

イベントフィードは、属性に基づく特定の条件に合うイベントの収集です。デフォルトのイベントフィードは、「All Events」と名付けられ、すべての入ってくるイベントを含み、お客様はその属性に基づくイベントのサブセットを特定する追加のフィードを作成します。

仕組み

フィードにより、お客様は最も重要なイベントのタイプを特定し、ベンダーのコネクタに送ることによってアクションを取ることができます。

仕組みは次のとおりです。

  • フィード名 イベントフィードには、コネクタアクションを構成する際など、UI の全体でそれを特定する名前があります。お客様とお客様のチームに意味があるユーザーフレンドリーな名前を選びます。

    例:

    "Purchases with Order Total Over $1,000"

  • 条件 各フィードは、 "All Events" フィードからそれを選別する一つかそれ以上の条件を必要とします。お客様のイベント属性の値（データレイヤー）を使用して、特定タイプのイベントを識別する単純なルールを構築します。

    例:

    tealium_event EQUALS "purchase"
AND
order_total IS GREATER THAN 1000`

  • ライブイベントで見る フィードを作成したら、ライブイベントを調整して、特定のフィードからのイベントだけを表示させます。これは、デバッグ、新しいイベント仕様の作成やフィードの健康状態のモニタリングにも有益です。

  • コネクタの中での使用 イベントフィードを作成したら、コネクタの中で使用してベンダーにデータを送ります。これが作動中の API ハブです。

例

"0" の結果を持つ"search"イベントを特定するイベントフィードの例を示します。このフィードは、お客様のサーチエンジンの健康状態をモニターするために使用するか、お客様の検索能力を改善してくれます。

フィードには2つの条件があることに注意します。

  • tealium_event EQUALS "search"
  • search_results EQUALS 0

イベントフィードの例："Search with No Results"
