イベント仕様: 仕様の追加

EventStreamのイベント仕様の追加について説明します。

イベント仕様は、Live Eventsのイベント詳細ビューから直接追加されます。Live Eventsによってマーケットプレイス仕様と一致するイベントが検出され、仕様のプレビューが表示されるため、イベント仕様は、画面から離れずに容易に追加できます。

また、（マーケットプレイス仕様と一致しない）未知のイベントは、カスタム仕様としてイベント詳細ビューから直接定義されます。これを行うとき、イベントのイベント属性は、_Add Event Specification_モーダルで事前入力されます。

仕組み

カスタムイベントでもマーケットプレイスからのイベントでも、すべてのイベントが、Live Events画面のイベント詳細ビューから作成されます。この例では、「検索」イベントから開始します。

仕組みは次のとおりです。

  1. ビューのイベント詳細では、以下のオプションを利用できます（イベントによって異なります）。
    • 仕様のプレビュー このオプションは、イベントが広く認められたマーケットプレイス仕様の場合に表示されます。このボタンをクリックすると、仕様によって予期される属性が表示され、現在のイベントデータに対して検証されます。
    • イベント仕様の作成 このオプションは、イベントが広く認められていない場合に表示されます。このボタンをクリックするとり、現在のイベントで検出された属性に基づいて、カスタムイベント仕様を作成するダイアログが開きます。
  2. "search"イベントについては、[Preview Spec]をクリックして、その仕様を表示します。現在のフォーマットでは、必須属性であるsearch_keywordとsearch_resultsが見つからないために、送信されたテストイベントは無効ですが、それでも構いません。後でイベントトラッキングを調整します。
  3. [Create Event Specification]をクリックして全体的な仕様ダイアログを開き、この仕様を追加します。メモまたは追加属性を追加します（既存の属性は編集できません）。
  4. [Finish]をクリックします。

これでイベント仕様が作成できました。次は、Live Events画面でデータ品質を確認します。
