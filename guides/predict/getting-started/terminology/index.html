用語集

この記事では、一般的な統計モデリングの用語、Tealium製品固有の用語、およびTealium Predict MLインターフェイスで使用されている用語について定義します。

オーディエンス（Audience）

audience（オーディエンス）は、一連の属性条件を共有する、訪問者プロファイルのグループとして定義されます。オーディエンスを使用すると、ベンダーアクションをリアルタイムでトリガーすることができます。Tealium Predictで、モデルから取得した出力属性を使用すると、会社のマーケティング活動のターゲットになる1つ以上のオーディエンスを作成できます。

混同行列（Confusion Matrix）

Tealum Predictの_confusion matrix_（混同行列）は誤差行列とも呼ばれるもので、実際の値と予測値を比較する形で、Model Explorer（モデルエクスプローラー）内のトレーニング済みモデルに関して報告されるパフォーマンス測定値のことです。業界用語で表現すると、混同行列は、真の値が既知である一連のテストデータを使用し、その後、実際の値と予測値を表形式で表示して、特定のアルゴリズムのパフォーマンスを可視化する目的で使用できます。

データサイエンティスト（Data Scientist）

Data Scientist（データサイエンティスト）は、分析の専門家であり、テクノロジーと社会科学に関するスキルを活用したトレンドの探索、業界に関する知識を使用したデータの管理、コンテキストの理解、既存の前提に対する疑いを抱いてビジネス上の課題に対するソリューションの提示を行います。

デプロイ済みモデル（Deployed Model）

deployed model（デプロイ済みモデル）とは、“trained”（既にトレーニングが完了し）、その後にデプロイされたモデルを指します。このようなモデルを使用すると、予測値を入力して、Tealium AudienceStream内のカスタマープロファイルを形成することができます。

マシンラーニング（Machine Learning）

Machine learning（マシンラーニング）は、人工知能（AI）の一分野であり、パターン認識を通じて、人間によるガイダンスなしでコンピュータが学習できるようになることを重視しています。マシンラーニングはあらかじめ決定された一連のルールを使用し、パターンの記憶、出力の分析、それらのパターンを説明できるモデルの作成、将来の行動のガイドを実行します。どのようなデータを求めているのか開発ユーザーが把握できている場合、マシンラーニングを使用すると、望ましいデータの獲得に要する過程を迅速化できます。どのようなパターンを特定することを望んでいる、またはパターン化しようとしているのか開発ユーザーが正確に把握できていない場合、マシンラーニングを使用して1つのパターンを見つけ、複数の結果を明らかにすることができます。この方法で必要なデータを獲得し、先に進むことができます。

モデル（Model）

Tealum Predict内の_model_（モデル）は、特定の時間枠内で予測しようとしている行動を表します。これに該当するのは、購入、コンバージョン、またはAudienceStream内でトラッキングされているカスタマーの任意の行動です。モデルはアルゴリズムを使用して作成されます。また、その結果を使用して、パターンを説明し、将来の結果を予測することができます。

Model Explorer（モデルエクスプローラー）

Model Explorer（モデルエクスプローラー）は、この製品のインターフェイスのうちインタラクティブなセクションです。各ステージにある各モデルのパフォーマンス測定値を表示できるほか、再トレーニングまたはデプロイなど、インターフェイスで選択できるアクション可能な項目を使用してモデルを微調整することもできます。

モデルの強み（Model Strength）

model strength (モデルの強み) は、指標と、モデルにスコアを割り当てるために使用される受け入れ可能なしきい値を組み合わせたものです。これらのスコアは、モデルの品質とパフォーマンス、また実際にモデルが発揮できる能力を決定するために使用されます。モデルのスコアは、Model Explorerの中で、“Excellent”、“Good”、“Fair”、“Poor”（優秀、良好、妥当、不適切）という形で表示されます。

出力属性（Output Attribute）

Tealium Predictの_output attribute_（出力属性）とは、モデルトレーニングの結果、作成された属性のことです。

予測時間枠（Prediction Timeframe）

日、週、または月を単位とする時間枠です。この時間枠のうち、ターゲット属性に対応するアクションが発生するのはいつなのか予測するために使用されます。たとえば、今後"x"日、週、または月のうちに、あるユーザーが"likelihood to return"（再アクセスする可能性）が該当します。

確率分布（再アクセスする訪問者）（Probability Distribution (Returning Visitor)）

probability distribution（確率分布）とは、トレーニング済みモデルに対して報告されたパフォーマンスグラフのことです。このグラフは、モデルが多数のケースをどれほど良好に区分するかを示します。つまり、訪問者が再アクセスして、関心のある特定のアクションを実施したケースと、訪問者が再アクセスせず、関心のある特定のアクションを実施しなかったケースを比較します。業界用語で表現すると、確率分布は算術関数であり、その結果は、実験でさまざまな結果が発生する確率を表します。したがって、あらかじめ決定したイベントが発生する確率を表しています。

受信者操作特性（ROC）（Receiver Operating Characteristics (ROC)）

ROC/AUC（under the curve、曲線より下の領域）とは、トレーニング済みモデルに対して報告されたパフォーマンス測定値のことです。業界用語で表現すると、_ROC_とは、真陽性の数を、真陽性の数と偽陰性の数の和で割って求めた、真陽性率として知られています。ROCが説明するのは、実際の結果が陽性であるときに、モデルが陽性クラスをどれほど良好に予測できるか、という点です。真陽性率をsensitivity（感度）と呼ぶこともあります。

再トレーニング（Retrain）

時間の経過に伴って、モデルの予測精度は低下します。新しいデータを使用して Tealium Predict モデルの retrain (再トレーニング) を実施すると、予測精度が向上し、より長い期間にわたって、より高い精度にとどまります。

強みのスコア（Strength Score）

モデルの強みのスコアは、各モデルの各バージョンの品質 (/guides/predict/getting-started/model-scores-and-ratings/) を格付けするレーティングを提示します。このレーティングシステムは、4種類のカテゴリレベルによって構成されています。Poor、Fair、Good、Excellent（不適切、妥当、良好、優秀）です。強みのスコアは、各モデルの合計スコアを記録しています。[F1 Score]（F1スコア）、[Recall]（再現率）、[Precision]（適合率）、[Accuracy]（精度）などです。これらのスコアリング要素の詳細な説明については、 モデルのスコアとレーティングを参照してください。

ターゲット属性（Target Attribute）

ターゲット属性とは、モデルを定義するときに選択されたAudienceStream属性のことで、予測しようとするユーザーアクションを表しています。これらの属性は、訪問レベルまたは訪問者レベルのブール値であり、何らかのアクションが既に実行されたことを伝達するために選択されます。たとえば、“Has Purchased”（購入した）という訪問のブール値属性は、ある訪問の間に購入イベントが発生したことを伝達します。

トレーニング（Training）

Tealium Predictの_Training_（トレーニング）とは、予測で使用するためにあらかじめ決定した期間にわたって、モデルがデータの使用と分析を行うステージのことです。このステージで使用するデータのサイズと品質は、モデルをデプロイするときに、結果の精度にとって重要な要因になります。

トレーニング済みバージョン（Trained Version）

Tealium Predictで、モデルの_trained version_（トレーニング済みバージョン）とは、モデルのトレーニングを表す個別のインスタンスのことです。各マシンラーニングモデルには1つのバージョンがあり、予測を高精度で実行するために使用されるデータを活用して、各バージョンのトレーニングを実施します。
