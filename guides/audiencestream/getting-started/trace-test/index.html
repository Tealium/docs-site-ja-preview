AUDIENCESTREAM/GETTING STARTED

トレース: トレースでのテスト

AudienceStreamのトレースを使用したテストについて説明します。

この手順では、トレースを使用して、以前のすべての構成が期待どおりに動作していることを検証します。トレースIDをご自身のアプリケーションに追加するか、Tealium Toolを使用し、簡単なテストを実行して、トレースインターフェイスのログを調べます。

開始する前に、ウェブページでテストする場合は、Tealium Toolsのブラウザ拡張機能を使用します。

トレースの開始

トレースでテストを実行するには、以下の手順に従います。

  1. Tools > Traceに移動し、[New Trace]の[Start]をクリックします。
  2. 提供されたトレースIDをコピーし、[Continue]をクリックします。
  3. 次のいずれかの方法を使用して、トレースIDをアクティベートします。
    • Tealium Tools
      ご自身のテストページを新しいブラウザウィンドウで開きます。Tealium Toolsブラウザ拡張機能をクリックし、[AudienceStream Trace]をクリックします。手順2のトレースIDを入力し、[Start Trace]をクリックします。
    • データ層の属性
      tealium_trace_idをご自身のコードインストールのデータ層に追加し、トレースIDの値に設定します。HTTP APIでは、これは次のクエリ文字列パラメータを追加することを意味します: &tealium_trace_id=012345Swiftアプリでは、たとえば、次のような形式になります: tealium?.volatileData()?.add(data: ["tealium_trace_id": "012345"])
  4. テストケースのワークフローを続行します。これはネイティブアプリでのページの更新、ページ間の移動、購入、または画面の表示を意味することがあります。ここでは500ドルを超える購入が完了すると、作成したオーディエンスがトリガーされます。
  5. トレースインターフェイスに戻り、ログの詳細を調べます。

トレースログ

トレースログは、イベント受信時に自動的に更新されます。UIには、イベントデータの詳細、エンリッチメントされた訪問者属性、参加または退出したオーディエンス、およびトリガーされた任意のコネクタアクションが表示されます。エラーが発生した場合は、そのエラーも問題の詳細情報とともに示されます。

この例では、550ドルの購入が完了し、以下がトレースで検証されました。

  • 更新された訪問者属性
  • 参加したVIPオーディエンス
  • トリガーされたGoogle Sheetsコネクタ
  • 挿入されたGoogle Sheets行

トレースログが期待どおりに表示されなかった場合の確認事項を次に示します。

  • テストで正しいトレースIDを使用しましたか？
  • 訪問者属性、オーディエンス、およびコネクタアクションを追加した後、アカウントを保存して公開しましたか？
  • お使いの属性エンリッチメントルールは正確ですか？
  • お使いのオーディエンスフィルターは適切ですか？

おめでとうございます。このシンプルな入門ガイドはこれで完了です。次の手順では、保存および公開プロセスと、その他の参考情報への役立つリンクをいくつか簡単に紹介します。
